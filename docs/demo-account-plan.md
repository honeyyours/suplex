## 4. 시드 스크립트 설계

### 위치
- `server/scripts/seed-demo-account.js` (이전 `seed-capture-demo.js` 삭제분 재작성, 더 풍부)

### 실행

```powershell
$env:DEMO_OWNER_EMAIL='demo@suplex.kr'
$env:DEMO_OWNER_PASSWORD='<12자_이상_가상_비번>'
railway run --service suplex node scripts/seed-demo-account.js
```

### Idempotent 정책

| 자산 | idempotent 키 | 재실행 시 |
|---|---|---|
| 회사 | name 매칭 | APPROVED·PRO 보장 |
| OWNER 유저 | email 매칭 | 비번 미변경 |
| 멤버십 | userId+companyId | OWNER 보장 |
| 시스템 프리셋 | seed 모드 | 빈 묶음만 채움 |
| 프로젝트 3건 | siteCode 매칭 | wipe+재생성 |
| 라운지 글 | authorId+title | 없는 글만 추가 |

## 9. 위험·짚어둘 점

### A. 데모 계정 데이터가 prod DB에 영구 보관됨
- 영향: 베타 통계·운영 페이지에 데모 회사가 포함될 수 있음
- 완화: 어드민 콘솔에서 데모 회사를 "내부용" 플래그로 표시. 통계에서 제외 (정식 출시 시 별도 작업)

### B. Cloudinary 사진 비용
- 영향: 가상 사진 30~50장 업로드 시 약간의 저장·대역폭 비용
- 완화: Lorem Picsum URL 직접 참조(업로드 X). 사진 변경 시 URL만 갱신

### C. 시연 중 데이터 수정
- 영향: 봉기님이 시연하면서 우발 수정 시 다음 시연에 영향
- 완화: 시드 스크립트를 idempotent로 → 시연 후 1줄 명령으로 데모 초기화

### D. 잠재 고객 공유 시 보안
- 영향: 데모 계정 비번 외부 노출 시 누구나 접근
- 완화: 봉기님 결정 항목 4번대로 read-only 공유 또는 시연만으로 제한
