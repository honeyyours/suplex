## 1. 가비아 (도메인·DNS)

- **접속**: https://my.gabia.com
- **역할**: `suplex.kr` 도메인 등록 + DNS 레코드 관리
- **DNS 레코드 (2026-05-18 기준)**:
  - `MX @ mx1/mx2.hiworks.com` (우선순위 10/20) — 하이웍스 메일 수신
  - `TXT @ v=spf1 include:_spf.hiworks.co.kr ~all` — 하이웍스 SPF
  - `A @ 216.198.79.1` — Vercel 웹사이트
  - `CNAME www → vercel-dns-017.com` — Vercel www 서브
  - `TXT resend._domainkey ...` — Resend DKIM (셋업 중)
  - `MX send → feedback-smtp.amazonses.com` — Resend 발송 (셋업 중)
  - `TXT send v=spf1 include:amazonses.com ~all` — Resend SPF (셋업 중)
  - `TXT _dmarc v=DMARC1; p=none;` — DMARC (셋업 중)
- **갱신 주의**: 도메인 만료되면 사이트 통째로 다운. 자동 갱신 ON 권장 + 결제 카드 만료 알림 확인
- **DNS 전파**: 변경 후 5분~수 시간

## 2. 하이웍스 (메일 수신)

- **역할**: `hello@suplex.kr` 등 apex 도메인 메일 받는 쪽 (가비아 통해 가입한 부가서비스)
- **메일 발송에는 사용 X** — 발송은 Resend (분리 운영)
- **연락처 노출**: 영업·매뉴얼·푸터·법적 문서에 `hello@suplex.kr` 노출 중

## 3. Resend (메일 발송)

- **접속**: https://resend.com
- **역할**: Suplex 앱에서 자동 발송하는 메일 (비번 찾기·초대 등)
- **발신 주소**: `Suplex <noreply@suplex.kr>` (환경변수 `MAIL_FROM`)
- **무료 한도**: 월 3,000건, 일 100건 — 베타 단계엔 충분
- **인증**: suplex.kr 도메인에 SPF/DKIM/DMARC TXT 4개 등록 → Verify
- **API Key 보관**: Railway 환경변수 `RESEND_API_KEY` (평문 노출 금지)
- **셋업 단계 (2026-05-18)**: 도메인 등록 완료 → DNS 레코드 추가 중 → Verify 대기

## 4. Vercel (프론트엔드 호스팅)

- **접속**: https://vercel.com/dashboard
- **역할**: React+Vite 빌드 → `suplex.kr` 정적 호스팅 + `/api/*` rewrite로 Railway 백엔드로 프록시
- **배포 방식**: GitHub `main` 푸시 → 자동 빌드·배포 (1~2분)
- **도메인 연결**: 가비아 A 레코드(`216.198.79.1`) + CNAME(`www`)로 연결
- **로컬 dev**: `vite` (포트 5173). 백엔드 직접 호출은 `VITE_API_URL` 환경변수 또는 Vercel rewrite

## 5. Railway (백엔드 호스팅)

- **접속**: https://railway.app
- **프로젝트**: proud-warmth / production / suplex
- **역할**: Node.js + Express API 서버 + Prisma → Neon
- **배포 방식**: GitHub `main` 푸시 → 자동 빌드·배포
- **start 스크립트**: `prisma db push --skip-generate --accept-data-loss && node scripts/seed-project-members.js && node src/server.js` → **schema 변경 시 별도 마이그레이션 명령 없이 자동 동기화**
- **CLI**: `railway link` 완료, `railway run`으로 prod env 사용해 로컬에서 스크립트 실행 가능
- **환경변수 보관소**: Railway 콘솔의 Variables 탭 (모든 API 키·시크릿·DB 접속 정보)

## 6. Neon (PostgreSQL DB)

- **접속**: https://console.neon.tech
- **역할**: 운영 DB (prod) + 개발 DB (dev) 분리. Railway가 `DATABASE_URL` 환경변수로 접속
- **백업**: Neon이 자동 PITR (Point-in-Time Recovery) 제공 + Suplex 어드민 콘솔에서 회사별 백업 export 가능
- **장애 시 1순위 확인**: Neon 대시보드의 compute 상태 (cold start 가능성 있음)

## 7. Anthropic (Claude API)

- **접속**: https://console.anthropic.com
- **역할**: AI 비서 (지출 분류·자재 동의어·견적 추천 등). SSE 스트리밍 방식
- **모델**: `ANTHROPIC_MODEL` 환경변수 (현재 `claude-sonnet-4-6` 등)
- **요금**: 사용량 종량제. 입력 토큰 + 출력 토큰 기준
- **사용 제어**: Suplex 회사 plan (PRO+) + role(OWNER) 가드 + 회사별 `hideExpenses` 토글
- **API Key**: Railway 환경변수 `ANTHROPIC_API_KEY`

## 8. Cloudinary (사진 저장)

- **접속**: https://console.cloudinary.com
- **역할**: 현장보고 사진·라운지 이미지·프로필 사진 호스팅 + CDN
- **무료 한도**: 25 크레딧/월 (대략 저장 25GB + 대역폭 25GB)
- **어드민 모니터링**: `/admin` 콘솔의 운영 탭에서 사용량 실시간 표시 + 80%+ 빨간 경고
- **API Key**: Railway 환경변수 `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`

## 9. 솔라피(Solapi / CoolSMS) — 대기

- **접속**: https://console.solapi.com
- **역할**: 카카오 알림톡 + SMS 발송 (계획 단계)
- **요금**: 알림톡 건당 7~9원
- **첫 도입 알림**: D-7 / D-3 미확정 체크리스트 (Wedge 전략)
- **진입 시점**: 토스페이먼츠 결제 인프라 완료 후 즉시
- **준비 작업**: 발신번호 등록 + 알림톡 템플릿 심사(채널 등록 필요) + 사업자 정보
- **환경변수 (대기)**: `SOLAPI_API_KEY`, `SOLAPI_API_SECRET`, `SOLAPI_SENDER`, `SOLAPI_PFID`

## 10. 토스페이먼츠 — 미시작

- **접속**: https://tosspayments.com
- **역할**: 구독 결제 게이트웨이 (정식 출시 단계)
- **요금**: 카드 약 2.9%
- **상세 가이드**: [결제준비.md](./결제준비.md) — 5단계 체크리스트·타임라인
- **진입 시점**: 정식 출시 약 2~3개월 전 (사업자등록 + 통장 개설 완료 후)

## 11. GitHub (저장소)

- **저장소**: https://github.com/honeyyours/suplex
- **브랜치 정책**: `main` 단일 브랜치 (베타), 자동 배포 트리거
- **민감 파일 보호**: `.env*` `.gitignore` 등록 — 자격증명이 git에 들어가지 않도록
- **PR 도구**: `gh pr create` CLI 설치됨

## 장애 1순위 의심 순서

prod 다운 보고 시 추측보다 직전 배포 의심:
1. Railway 최근 배포 로그 (5분 내 push 있었는지)
2. Neon DB 상태 (cold start·연결 한도)
3. Vercel 빌드 실패 여부
4. 외부 API 한도 초과 (Cloudinary 크레딧·Anthropic 토큰)
