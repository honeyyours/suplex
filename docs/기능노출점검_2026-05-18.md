# 기능 노출 점검 — 2026-05-18 (1회차)

**4단계 분류**:
- **A** 풀공개 (90%+ 완성도, 라벨 없음)
- **B** 베타 라벨 (70~90%, 베타 배지 + 피드백 버튼)

## 8. 별도 점검 — 데이터 안전 가드 (분류와 무관)

> **봉기님 질문 답 (2026-05-18)**: 전부 어드민만 되는 건 아닙니다. **임포트·사진 ZIP·일괄 삭제는 일반 사용자(OWNER)도 가능**합니다. 그래서 일반 OWNER가 닿을 수 있는 가드는 베타 사용자 첫인상에 직접 영향 — 분류와 별개로 점검 우선순위 높음.

### 8-A. OWNER도 사용 가능 (베타 사용자 영향)

| 작업 | 누가 | 현재 가드 | 가드 우선순위 |
|---|---|---|---|
| 임포트 (회사 자산 9종 / 전체 28종) | OWNER | OWNER + 빈 회사 한정 | OK — 빈 회사 한정으로 사고 차단 |
| 익스포트 (JSON 텍스트) | 모두 | **영구 차단 (모든 등급)** | OK (운영 락인 정책) |
| 익스포트 (사진 ZIP) | OWNER | 만료·등급 무관 항상 가능 | OK |
| 일괄 삭제 (프로젝트·견적 등) | OWNER | **— (코드 점검 필요)** | ⚠️ 확인 필요 — UX 가드(확인 다이얼로그) 있는지 |

### 8-B. 슈퍼어드민 전용 (봉기님만)

| 작업 | 현재 가드 | 가드 우선순위 |
|---|---|---|
| 마이그레이션 (normalize-phases) | Dry Run 옵션 + 어드민 한정 | OK |
| 사용자 / 회사 삭제 | 어드민 + 이메일 확인 + cascade 안내 | OK |
| 어드민 사칭 (impersonate) | READ-ONLY 강제 + 1시간 만료 | OK |

### 8-C. 베타 직전 처리 권장

**일괄 삭제 UX 가드 점검** — OWNER가 자기 프로젝트·견적을 한 번에 잘못 지울 위험이 있는지 코드 확인. 확인 다이얼로그·이름 재입력·cascade 안내 등이 있는지 점검해서 부족하면 보강. (위 §1~7 분류 작업과 무관, 독립 작업)

## 10. 표준 컴포넌트 신설 우선순위 (확정)

### 1순위 — **BetaLabel** (B 11개 처리)

가장 광범위하게 필요. 프로젝트 탭 9개 + 편의 카드 1개 + 홈 카드·시공팀 홈에 베타 배지 일관 부착.

**명세 초안**:
- 작은 배지 (`[BETA]` 또는 ⓑ 같은 미니멀 형태)
- 클릭 시 작은 팝오버: "이 기능은 베타 검증 중입니다. 피드백 보내기" + 피드백 입력창
- 페이지 헤더 / 탭 라벨 옆 / 카드 제목 옆 어디든 배치 가능한 inline 컴포넌트
- 피드백 API는 별도 신설 또는 라운지 운영자 신고 시스템에 연결 검토

### 2순위 — **PlanGuide / ComingSoon 통합** (C 3개 처리)

봉기님 우려 — "사용자가 PRO 기능 인지해야 절여진다" — 의 핵심 레버. 두 컴포넌트가 결국 같은 슬롯·같은 톤이므로 통합 권장:

```jsx
<PlanGuide
  variant="comingSoon" | "planGate"
  title="자재 산출 (PRO)"
  description="..."
  ctaLabel="PRO로 업그레이드"
  features={['타일·도배·마루·페인트 자동 계산', '회사 카탈로그 적용']}
/>
```

- 페이지 전체 placeholder (= 기존 ComingSoon 패턴)
- 슬롯 단위 카드 잠금 (= 가격정책 4-bis 명세)
- "준비 중" / "프로 기능" / "엔터프라이즈 전용" 톤 변주
- 클릭 시 UpgradeModal 발동 (variant=planGate일 때)

**적용 위치**:
- 지출관리 (글로벌) — C 처리. 현재 FeatureGate redirect만 — PlanGuide 페이지 placeholder로 안내 추가
- 공정 현황 (프로젝트) — C 처리. 베타 검증 중 안내 + 미리보기
- 입주민 안내문 (편의 카드) — 이미 인라인 배지로 C 패턴 적용 중. 표준 컴포넌트로 통일

### 3순위 — **UpgradeModal** (가격정책 v8 4-quater)

베타 동안 가격 적용 X이지만, 정식 출시 직전 필수. 6번째 활성 프로젝트 시도 + 견적 작성 시도 시 발동. loss aversion 패턴(기존 5개 리스트 즉시 표시).

### 4순위 — **D 숨김 처리** (상세 견적)

이미 NavLink 미노출 상태. 현 상태 유지가 D 처리 그대로. 별도 작업 없음.
