# 기능 노출 점검 — 2026-05-18 (1회차)

**작업 정의**: 메모리 `수플렉스_운영_기능노출점검.md` 참조

**4단계 분류**:
- **A** 풀공개 (90%+ 완성도, 라벨 없음)
- **B** 베타 라벨 (70~90%, 베타 배지 + 피드백 버튼)
- **C** 티저 (50~70%, "준비 중" 안내 + 미리보기. PRO 기능 미리보기와 통합 가능)
- **D** 숨김 (50% 미만 또는 데이터 위험, UI 제거)

**평가 기준 3개**:
1. 데이터 안전성 (위험 있으면 무조건 C·D)
2. 첫인상 영향 (가입 후 1주일 내 사용 빈도)
3. PRO 어필 가치 (미완성이어도 인지가 결제 동기에 기여하면 C)

**입력 방법**: 각 행의 "분류" 칸에 `A` / `B` / `C` / `D` 중 하나 + 비고 자유 메모

## 9. 표준 컴포넌트 신설 우선순위 (분류 결과 보고 확정)

분류가 결정되면 아래 우선순위로 신설·적용:

1. **UpgradeModal** (가장 가치 높음) — 가격정책 v8 4-quater 명세. 6번째 활성 프로젝트 시도 + 견적 작성 시도 시 발동. loss aversion 패턴.
2. **PlanGuide** (C 티저·슬롯 단위) — 가격정책 v8 4-bis 명세. 잠긴 자리 미리보기 + 등급 안내.
3. **BetaLabel** (B 배지) — 작은 배지 + 피드백 버튼. 인라인 배치.
4. **ComingSoon 활용처 결정** — 이미 구현되어 있으나 import한 곳 없음. 페이지 단위 C 티저로 활용할지, PlanGuide와 통합할지.
