**구조 특징**

- HTML 한 페이지(204줄), 7개 탭 + 입력 패널 + 결과 패널 2-컬럼
- 모든 자재 메타데이터·산식이 `script.js` 상단에 객체 리터럴(`materials`, `tileSpecs`, ...)로 노출

- 첫 스크롤(80px) 이후 카카오 오픈채팅 모달 자동 노출 → localStorage로 dedup

## 3. UX 패턴

### 3.1 화면 구성
```
[ Header — 큰 타이틀 ]
[ 7개 자재 탭 (이모지 + 텍스트) ]
[ 입력 패널 (좌)  |  결과 패널 (우) ]
   ↑ 동적 폼         ↑ 큰 숫자 + 메트릭 4개 + 메모
```

### 3.2 입력 UX의 핵심
- **세그먼티드 컨트롤**: "평수 vs 가로×세로", "바닥 vs 벽", "1회/2회/3회 도장" — 라디오 대신 토글
- **프리셋 버튼**: 로스율 옆에 "일반 7% / 권장 10% / 절단 15% / 복잡 20%" 즉시 적용 버튼
- **conditional fields**: `data-visible-when="airconUse" data-visible-value="commercial"` 같은 선언적 토글
- **자동 의존 동기화**: 마루 종류 바꾸면 박스당 평수가 자동 갱신·disabled
- 입력 즉시 계산(`input` 이벤트), 제출 버튼 없음 → 라이브 결과

### 3.3 결과 UX
- 큰 숫자(발주 권장 매수) + 단위 + 부제(로스 포함 설명)
- 4-셀 메트릭 그리드(시공 면적/평수/순수 매수/박스 수)
- 카톡 친화적 **복사 텍스트** 자동 생성 ("[타일 계산]\n현장: ...\n공간: ...")
- 인쇄 버튼

### 3.4 마케팅 깔때기
- 우하단 떠다니는 카톡 위젯 (실무자 채팅 + 업데이트 요청)
- 스크롤 80px 후 자동 모달 1회 노출
- GA + 자체 트래킹 양쪽으로 `community_chat_click`, `update_request_click` 측정

## 4. 자재별 산식 (★ 핵심)

> 모두 `script.js`에서 1:1 발췌. 단위 일관성·계수 출처는 별도 메모.

### 4.1 공통 변환
```
1평 = 3.3058 ㎡
ceil(x) = Math.ceil(max(0, x))
```

### 4.2 타일 (`calculateTile`)
```
타일 1매 면적 = (W_mm / 1000) × (H_mm / 1000)   [㎡]
순수 매수      = ceil(시공면적 ㎡ / 1매면적)
발주 매수      = ceil(순수 매수 × (1 + 로스율%))
박스 수        = ceil(발주 매수 / 박스당 매수)
```
- 규격 프리셋: 300×300, 300×600, 600×600, 600×1200, 800×800 (+ custom)
- 로스 프리셋: 7/10/15/20%
- "박스당 매수" 모르면 4 입력 안내

### 4.3 도배 (`calculateWallpaper`)
```
벽 면적 (평수 입력)  = pyeong × 천장높이 × 1.2625   ← ★ 마법의 계수
벽 면적 (가로세로)   = 2 × (W + L) × H              ← 둘레 × 높이
총 도배면적         = 벽면적 + (천장 포함 시 바닥면적 ㎡)
롤당 시공량         = 폭(m) × 길이(m)
순수 롤            = 총 도배면적 / 롤당 시공량
발주 롤            = ceil(순수 롤 × (1 + 로스율%))
풀 (kg)            = ceil(발주 롤)  ← 1롤당 1kg 가정
```
- 벽지 프리셋: 실크 1.06×15.6m, 합지 0.93×17.75m, 포인트 1.06×10m
- **계수 1.2625의 정체**: 평수→실 둘레 추정용 경험계수로 추정. (실 형상 가정 없이 평수만으로 벽 면적을 내려는 단순화)
- 풀 = 롤 수 그대로 → 보수적 가정

### 4.4 마루 (`calculateFloor`)
```
순수 박스 = pyeong / 박스당평수
발주 박스 = ceil(순수 박스 × (1 + 로스율%))
발주 평수 = pyeong × (1 + 로스율%)
```
- 박스 프리셋: 강마루 0.7평, 원목 0.5평, 헤링본 0.45평 (+ direct)
- 로스 프리셋: 5/8/10/15%
- 걸레받이는 단순 표시(둘레 m 그대로 노출, 계산 없음)

### 4.5 페인트 (`calculatePaint`)
```
시공면적
  - 직접      = directArea
  - 천장      = 바닥면적 ㎡
  - 벽(평수)  = pyeong × 1.65 × 3.3058   ← ★ 또 다른 계수 1.65
  - 벽(가로세로) = 2 × (W + L) × H
도장면적 = 시공면적 × 도장횟수
순수 L  = 도장면적 / coverage
발주 L  = 순수 L × (1 + 로스율%)
18L 통  = ceil(발주 L / 18)
4L 캔   = ceil(발주 L / 4)
```
- coverage 프리셋: 수성 10㎡/L, 유성 8, 친환경 8, 프라이머 5
- **계수 1.65**: 평수→벽 면적 환산용 (1.2625와 다른 값 → 도배·페인트 계수가 일관되지 않음. 분석상 약점)
- 도장 횟수 1/2/3 세그먼티드, 기본 2

### 4.6 조명 (`calculateLighting`)
```
높이 보정      = 1 + max(0, H - 2.4) × 0.22
총 lumens     = 시공면적 ㎡ × lux × 높이보정
총 W          = 총 lumens / 90        ← LED 90 lm/W
다운라이트 W  = 총 W × (mixed ? 0.65 : 1)
LED 7W 개수   = ceil(다운라이트 W / 7)
LED 10W 개수  = ceil(다운라이트 W / 10)
```
- lux 프리셋: 거실 220, 침실 130, 주방 400, 욕실 160, 공부방 450, 사무실 750, 카페 400, 복도 120 (KS 권장 조도 중간값)
- 메인등 병행 시 다운라이트 35% 감소
- LED 효율 90 lm/W 고정 — 요즘 100~130까지도 나오는데 보수적

### 4.7 에어컨 (`calculateAircon`) ★ 가장 복잡
```
baseKcal = 가정 400 / 사무실 450 / 카페 520 / 음식점 620   [kcal/평·h]
sunlight = 북 0.95 / 동 1.0 / 남 1.08 / 서 1.15
heightFactor = 1 + max(0, H - 2.4) × 0.16
openFactor   = 1.10  (오픈형 체크)
topFloorFactor = 1.12 (탑층 체크)

capacity (kcal/h) = pyeong × baseKcal × sunlight × heightFactor × openFactor × topFloorFactor
capacityKw       = capacity / 860        ← 정의식 1 kW ≈ 860 kcal/h
시판 평형         = ceil(capacity / 400)
실내기 (시스템)   = ceil(pyeong / (상업?6 : 8))
실내기 (스탠드)   = max(1, 1 + ceil(max(0, pyeong-18) / 6))
보조 벽걸이       = 스탠드일 때 max(0, 실내기 - 1)
```
- 가정/상업 토글, 상업이면 업종 선택 노출
- 환경 체크박스 2종(오픈형·탑층)을 가중치로 합성 — 단순하지만 보수적·실용적

### 4.8 가구 (`calculateFurniture`)
```
총 길이    = 본체 + 상부
발주 길이  = 총길이 × (1 + 여유율%)
본체 도어  = ceil(본체길이mm / 도어폭mm)
상부 도어  = (상부 > 0) ? ceil(상부길이mm / 도어폭mm) : 0
면재 참고  = 발주길이 × (높이 mm / 1000)
상판 길이  = 본체길이 × (1 + 여유율%)   ← 체크박스 해제 시 0
```
- 종류별 기본값: 싱크 850/600/450, 붙박이장 2200/600/500, 신발장 2100/350/450 등
- 결과는 "1차 물량 정리용 · 최종 제작 전 실측 필수" 명시 → **방어적**

## 5. 강점·약점

### 5.1 강점 (Suplex가 배울 점)
1. **즉시 입력 → 즉시 결과** — 제출 버튼 없음. 마찰 최소
2. **프리셋 버튼**으로 로스율·도장횟수 등 의사결정 비용 0
3. **결과 카피 텍스트 포맷**이 카톡 친화적 (`[타일 계산]\n현장: ...`)
4. **단위·계수 가시화** — "1평=3.3058㎡", "LED 90 lm/W" 등 hint에 명시
5. **세그먼티드 컨트롤** 일관 사용 — 라디오·드롭다운보다 모바일 친화
6. **PWA + 오프라인** — 현장에서 데이터 끊겨도 동작
7. **방어적 결과 메모** — "현장 메모 · ..." 식 한 줄 안전망

### 5.2 약점 (Suplex가 차별화할 점)
1. **계수 불일치**: 도배는 평수×1.2625, 페인트는 평수×1.65 — 같은 "벽 면적"인데 다른 계수
2. **벽 면적 추정이 단순화 과잉**: 실제 도면·실측·창문 차감 없음. 평수만으로 추정
3. **저장 없음**: 현장명·공간명 입력해도 다음 방문 시 휘발. 프로젝트 단위 누적 X
4. **공정 연계 0**: 자재 계산이 "공정 시드"나 "발주서"로 이어지지 않음. 단발 도구
5. **회사·팀 개념 없음**: 현장 1명이 혼자 쓰는 도구. B2B 협업 X
6. **카탈로그 없음**: 박스당 매수·롤 폭·coverage를 매번 사용자가 입력해야 함
7. **이력·견적 비교 없음**: 다음 견적에 prefill·피드백 사이클 0
8. **마케팅 깔때기가 카톡뿐**: 깔때기→락인 자산이 없음 (그냥 도구)

## 6. Suplex 적용 시사점

> 자세한 적용안은 [본 문서 §7 적용 후보](#7-suplex-적용-후보-별도-검토)에 모았습니다.
> 메모리 연계: [[수플렉스_모듈_간편견적서]], [[수플렉스_기능_견적시스템]], [[수플렉스_모듈_마감재스키마]], [[수플렉스_기능_마감재시스템]]

### 6.1 한 줄
> **"단발 계산기"에 멈춰있는 시장에, Suplex는 _프로젝트·공정·발주_ 와 한 사이클로 묶인 계산기를 얹어 락인을 만든다.**

### 6.2 즉시 이식 가능한 UX (소공사)
- ✅ 로스율·도장횟수 **프리셋 버튼**
- ✅ "평수 ↔ 가로×세로" **세그먼티드 토글**
- ✅ 큰 숫자 + 4-셀 메트릭 + **카톡 카피 텍스트**
- ✅ KS 권장 조도·LED lm/W 같은 **계수 hint 표시** (사용자가 산식을 신뢰하도록)

### 6.3 차별화 — 우리는 *프로젝트의 연속성*으로 이긴다
| 경쟁사 | Suplex 차별화 |
| --- | --- |
| 평수만으로 벽 면적 추정 | **공간 스키마 + 실측값**으로 정확한 면적 (→ [[수플렉스_모듈_마감재스키마]] `spaceGroup`) |
| 매번 사용자가 박스/coverage 입력 | **회사 자재 카탈로그**에서 자동 채움 (회사 자산 → 락인) |
| 결과는 카톡 카피 | **간편견적 → 확정 → contractAmount sync** 사이클로 자동 흘러감 |
| 단발 도구 | **공정 시드 → 발주서 → 정산** 한 사이클 ([[수플렉스_기능_지출정산]]) |
| 회사 개념 없음 | **프로젝트 멤버 + 시공팀 양면** ([[수플렉스_설계_시공팀계정]]) |

### 6.4 무료 wedge 후보
> [[수플렉스_전략_Wedge전략]]에 따라 일정이 wedge지만, **이런 단발 계산기를 _비로그인 무료_ 진입점**으로 추가하는 옵션도 검토 가치 있음:
> - SEO 트래픽 흡수 (경쟁사가 카카오로 모으는 트래픽 일부)
> - "로그인하면 견적·발주로 흘려보낼 수 있어요" CTA
> - 계산값을 회사 프로젝트로 1-클릭 저장 (락인 트리거)
