## 2. 자재별 산식 1:1 비교

### 2.1 타일 — 사실상 동일 (무승부)
```
양쪽 공통:
  타일1매 ㎡ = (W_mm/1000) × (H_mm/1000)
  순수 매수 = 시공㎡ / 타일1매
  발주 매수 = ceil(순수 × (1+로스%))
  박스 수   = ceil(발주 / 박스당매수)
```
- 규격 옵션, 로스 프리셋, custom 입력 모두 일치
- **차이**: A는 로스 프리셋 4단(7/10/15/20), B는 비슷한 수준

### 2.2 도배 — ★★★ B 압승

| 항목 | A | B |
| --- | --- | --- |
| **단위** | 롤 = 폭(m) × 길이(m) → ㎡ 환산 | **롤당 시공평수 직접 (실크 5평/롤, 합지 2평/롤)** |
| **벽 면적 환산** | 평수 × 천장H × **1.2625** | 평수 × **1.65 × (H/2.4)** |
| **천장 환산** | 천장 포함 시 바닥면적 그대로 | 평수 × **0.65** (별도 계수) |
| **풀 (kg)** | 발주 롤 수 그대로 (1:1) | **실크 ㎡/15, 합지 ㎡/8** (합지 1.9배) |
| **검증 기준** | 없음 | **"32평 천장포함 = 실크 15롤"** 주석으로 검증 |
| **자연 로스 가이드** | 사용자가 매번 입력 | "환산식에 포함 → 추가 로스 0% 권장" |

**B가 우위인 이유**
1. 한국 도배사·시공팀이 실제로 **"평/롤"** 단위로 말하지 ㎡로 안 함
2. **합지가 풀을 더 먹는다**는 실무 상식이 산식에 반영됨
3. **천장 높이 보정**이 들어가서 2.7m·3m 천장 정확
4. **자연 로스 사전 포함** → 사용자 의사결정 비용 ↓

**Suplex 채택**: B 산식 그대로 채택. 단, **계수 1.65/0.65/롤당 5평** 출처를 회사별 카탈로그로 override 가능하게 (회사 자산 락인)

### 2.3 마루 — 박스당 평수 카탈로그 진본 확인 필요

| 종류 | A | B |
| --- | --- | --- |
| 강화마루 | (없음) | 0.7평/박스 |
| 강마루 (laminate) | **0.7평/박스** | **0.55평/박스** |
| 원목마루 | 0.5평/박스 | 0.45평/박스 |
| 헤링본 | 0.45평/박스 | (없음) |

```
양쪽 공통:
  순수 박스 = 평 / 박스당평수
  발주 박스 = ceil(순수 × (1+로스%))
```

**평가**: 산식은 같지만 **카탈로그 값이 다름**. 같은 "강마루"가 A=0.7, B=0.55로 30% 차이. 어느쪽이 맞는지는 실제 제조사 카탈로그(동화/한솔/구정 등) 확인 필요.

**Suplex 채택**: **회사 카탈로그를 우선**하고, 빈 회사엔 시스템 프리셋([[수플렉스_설계_시스템프리셋]])으로 시드. 두 사이트 값을 모두 시드 옵션으로 제공.

### 2.4 페인트 — B 약간 우위 (천장 보정)

| 항목 | A | B |
| --- | --- | --- |
| 벽 도장 ㎡ | 평 × **1.65** × 3.3058 | 평 × **1.65 × (H/2.4)** × 3.3058 |
| 천장 도장 | 바닥 ㎡ | 바닥 ㎡ |
| coverage | 수성10/유성8/친환경8/프라이머5 | **완전히 동일** |
| 도장 횟수 | 1/2/3 | 1/2/3 |
| 18L 말통 환산 | ✅ | ✅ |
| 4L 캔 환산 | ✅ | ❌ |

**평가**: 산식 거의 같음. B가 천장 높이 보정 한 단계 더 있고, A가 4L 캔 환산까지 제공. **각각 작은 우위**.

**Suplex 채택**: B 산식 + A의 4L 캔 보조 메트릭.

### 2.5 조명 — 용도 갈림 (학술 vs 실무)

**A — 학술적 (lm/W 기반)**
```
총 lumens = ㎡ × lux × (1 + max(0, H-2.4) × 0.22)
총 W     = 총 lumens / 90               ← LED 90 lm/W 고정
다운라이트 W = 총 W × (mixed ? 0.65 : 1)
```

**B — 실무적 (W/평 직접 테이블)**
```
총 W = ceil(평 × W/평_테이블 × max(1, H/2.4))
```

| 공간 | A lux | B lux / W/평 |
| --- | --- | --- |
| 거실 | 220 | 200 / 6 |
| 침실 | 130 | 120 / 4 |
| 주방 | 400 | 400 / 10 |
| 욕실 | 160 | 150 / 5 |
| 공부방 | 450 | 400 / 10 |
| 사무실 | 750 | 700 / 14 |
| 카페 | 400 | 400 / 10 |
| 복도 | 120 | 120 / 4 |

**A 장점**: LED 효율이 90→120 lm/W로 바뀌면 자동 반영됨. 일반화 가능.
**B 장점**: 시공사가 "10평 거실 = 60W 필요"를 도면 보고 즉시 검증 가능. 빠르다.

**Suplex 채택**: **B 테이블을 디폴트** + 고급 옵션으로 A 산식 노출. 회사가 LED 효율(lm/W)을 회사 설정으로 override 가능.

### 2.6 에어컨 — ★★★ B 압도적 우위

| 항목 | A | B |
| --- | --- | --- |
| **가정용 baseKcal** | 400 | **250** |
| **사무실 baseKcal** | 450 | 350 |
| **카페 baseKcal** | 520 | 450 |
| **음식점 baseKcal** | 620 | 550 |
| **방향 보정** | 북0.95/동1.0/남1.08/서1.15 | **북0.85/동1.0/남1.10/서1.25** (스프레드 ↑) |
| **천장 보정** | 1 + max(0,H-2.4) × 0.16 | max(1, H/2.4) |
| **오픈형** | × 1.10 | × 1.15 |
| **탑층** | × 1.12 | × 1.15 |
| **권장 평형 표현** | ceil(kcal/400) | **시판 평형 enum [6,7,8,10,13,15,17,18,20,22,25,27,30,36]** |
| **HP 환산 (시스템)** | ❌ | **✅ 1HP=2500 kcal/h, 표준 HP enum** |
| **4-조합 분기** | 단순 (스탠드/시스템 2분기) | **가정×스탠드/가정×시스템/상업×스탠드/상업×시스템 4-조합** |
| **방수 추정 (가정)** | ❌ | **평수→방수 (12↓0실, ~20=1실, ~30=2실, ~40=3실, ~↑4실)** |
| **분산 시공 권장** | ❌ | **5HP↑은 자동 분산 권장 (2대 또는 3대)** |

**B 결정적 우위**
1. baseKcal이 A보다 30~40% 낮음 — A는 단열 안 좋던 시절 기준, B는 현 신축 기준
2. **시판 평형 enum**으로 매칭 → 실제 시장에서 살 수 있는 사이즈로 출력
3. **HP 환산** + 표준 HP enum → 상업용 시스템 에어컨 시공자가 그대로 발주 가능
4. **방수 추정**으로 가정용 거실+방 분산 패턴 자동 산출 — 한국 시공 관행 그대로

**Suplex 채택**: **B 산식·테이블 그대로** 채택. baseKcal은 회사별 override 가능하게 (단열 등급별 프리셋).

### 2.7 가구 — A 유일

A에만 존재 (싱크대·붙박이장·신발장·수납장·TV장).
B는 자재만 다루고 가구는 영역 밖.

**Suplex 채택**: 가구는 **별도 모듈**로 처리 (제작가구는 도면·실측이 필수라 평수 기반 빠른 계산기와 결이 다름). A의 산식은 참고용으로만.

## 4. Suplex 적용 후보 — 편의기능 위치 선정

### 4.1 결론: **간편견적서 옆 "자재 산출" 보조탭**

[[수플렉스_모듈_간편견적서]]·[[수플렉스_기능_견적시스템]] 흐름에서:
1. 견적 행에서 공정 종류(타일/도배/마루/페인트/조명/에어컨) 선택
2. 우측 드로어로 **자재 산출 모듈** 슬라이드 인
3. 평수·옵션 입력 → 라이브 계산
4. 결과를 견적 행으로 1-클릭 prefill

### 4.2 산식 채택 최종안
| 자재 | 채택 |
| --- | --- |
| 타일 | B (= A) — 무차별 |
| 도배 | **B 100%** (평/롤 단위 + 풀 종류별 + 천장 보정) |
| 마루 | B 산식 + 회사 카탈로그 우선 |
| 페인트 | B (천장 보정) + A의 4L 캔 메트릭 |
| 조명 | **B 디폴트 (W/평 직접) + 고급 옵션으로 A 산식 (lm/W)** |
| 에어컨 | **B 100%** (4-조합 분기 + 시판 평형 enum + HP) |
| 가구 | 별도 모듈 (즉시 도입 X) |

### 4.3 차별화 (어느 경쟁사도 없는 것)

이건 Suplex만 할 수 있는 것:

1. **회사 카탈로그 override** — 박스당 평수·롤당 시공평수·LED W/평·baseKcal을 회사 자산으로 누적
2. **공간 스키마 실측** ([[수플렉스_모듈_마감재스키마]]) → 평수 추정 X, 실 면적
3. **결과 → 발주서·정산 한 사이클** ([[수플렉스_기능_지출정산]]) — 단발 도구 X
4. **회사 멤버 공유** ([[수플렉스_기능_프로젝트멤버]]) — 한 명이 입력하면 팀 전체 봄
5. **계수 출처 토글** — "왜 1.65인가? → 한국 도배 업계 표준 환산식, 32평=실크 15롤로 검증"을 hint로 노출 → 신뢰성

## 정리 (확정/대기)

### ✅ 확정 (분석 산출물)
- 두 사이트 산식 1:1 비교 완료
- 도배·에어컨은 B 산식 우위 명백, 조명은 용도 분기
- Suplex 채택안: B 베이스 + A의 UX 일부 + Suplex만의 차별화 4가지

### ⏳ 대기 (봉기님 결정)
- 자재 산출 모듈을 **간편견적서 보조탭**으로 넣을지 / **편의기능 탭**([[수플렉스_기능_편의기능]])에 11번째로 넣을지 / **별도 글로벌 메뉴**로 뽑을지
- 시드 카탈로그 값 출처: B 값 + A 값을 모두 시스템 프리셋으로? 아니면 B 단일?
- "회사 카탈로그 override"를 어디서 관리할지: [[수플렉스_모듈_마감재스키마]] 확장 vs 새 카탈로그 모듈
