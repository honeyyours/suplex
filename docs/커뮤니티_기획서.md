# 수플렉스 라운지 기획서

작성일: 2026-05-01 / 갱신: 2026-05-01 (봉기님 피드백 1차 반영) / 상태: 확정 (v1 MVP 작업 진입)

---

## 1. 한 줄 요약

수플렉스에 글로벌 메뉴 8번째로 **"라운지"** 를 추가합니다. 인테리어 회사의 **디자이너·현장팀·운영진**이 모이는 클로즈드 커뮤니티이며, **한 번이라도 수플렉스 회사에 속했던 사람은 퇴사 후에도 평생 접근 가능**합니다. 6개 카테고리(노하우·사용팁·**스케치업 루비**·자유잡담·구인구직·공지) + 직군/주제 `[]` 태그. 클라이언트는 절대 들어오지 못하고, 거래처 실명 비방만 금지합니다. 단가·시세 공유는 허용합니다(같은 업계 종사자 클로즈드의 힘).

| 목적 | 기대 효과 |
| --- | --- |
| **체류시간·리텐션** | Mindbody One은 커뮤니티로 멤버 리텐션 89%. 견적·일정 외 매일 들어올 이유. |
| **사용자 자조 시스템** | dbt 매출 80%+가 커뮤니티 발(發). Procore도 경험자가 신규 답변. CS 부담 자동 분산. |
| **데이터 자산화** | 글이 쌓이면 신규 회사 온보딩·사용자 매뉴얼 보강재. |
| ★ **인테리어 생태계 장악** | **퇴사자도 라운지에 남는다** → 새 회사에서 수플렉스 추천 → 자연 영업. 라운지에서 자연스럽게 구인구직이 일어나면 인테리어 인력시장의 길목을 우리가 쥔다. |
| **최고등급 진입 명분** | 프리셋 커스텀은 최고등급에서만. 라운지에서 "이 프리셋 좋네요" 추천 → 등급 업 동기. |
| **베타 피드백 채널** | 클로즈드 베타의 피드백·기능 요청을 한 곳에서 수집. |

## 4. 이용자 정의

| 직군 | 라운지 활동 | 비고 |
| --- | --- | --- |
| **디자이너** | ★ 핵심 | 견적·일정·마감재 노하우 |

| **운영/대표** | O | 회사 운영·세무·인사 |
| **작업자(목수·타일러 등)** | X | 별도 SaaS 제품 진입 자체가 어려움. 라운지 활동도 비현실적. |
| **클라이언트(인테리어 고객)** | 절대 X | 영구 규칙 |

### 4-1. ★ 라운지 멤버십 (퇴사자도 평생 유지)

- **자격**: 단 1회라도 승인된 회사(`Company.approvalStatus = APPROVED`)에 멤버로 속했던 user.
- **퇴사 시**: 회사 데이터 접근은 차단되지만, **라운지 접근(읽기·쓰기·댓글)은 유지**.
- **표시**: 본인 닉네임 + 본인 선택 라벨(예: "디자이너 / 5년차"). 회사명 노출은 옵션, 기본 OFF.
- **재취업 시**: 새 회사 멤버로 들어가면 자연스럽게 그쪽 데이터 접근 + 라운지는 끊김 없이 이어짐.
- **차단**: 슈퍼어드민이 신고 처리·정책 위반 시에만 라운지 자체 정지.

> **이게 "인테리어 생태계 장악" 의 엔진**: 라운지에 남은 사람이 새 회사에서 수플렉스를 추천하고, 라운지가 곧 인테리어 인력 시장이 된다.

## 5. 카테고리 (6개)

베타 시작 시 6개로 고정. 카테고리는 적게 쪼개고, 세부 분류는 태그(6장)로.

| # | 카테고리 | 예시 글 | 비고 |
| --- | --- | --- | --- |
| 1 | **공정·시공 노하우** | "타일 줄눈 폭 5mm vs 3mm" / "철거 후 평탄화 셀프체크" | 핵심. 직군 태그로 디자이너·현장팀 분리 |
| 2 | **수플렉스 사용 팁** | "견적 가이드 이렇게 짜서 시간 절반" / "이 프리셋 추천합니다" / "어드바이스 D-14 활용" | 프리셋 추천 핵심. 홈 노출 후보 |
| 3 | **스케치업 루비** | "도면 자동 치수 스크립트.rb" / "이런 거 만들 수 있는 분?" / "이 코드 어디서 막히죠" | 디자이너 핵심 도구. 코드 블록·파일 첨부 특수 처리(8-5) |
| 4 | **자유잡담** | "장마철 외부공정 어떻게" / "오늘 점심" | 진입장벽 낮춤, 활성화 마중물 |
| 5 | **구인구직** | "디자이너 1명 채용합니다" / "현장팀 이직 알아봐요" | 봉기님 비전 핵심. v1부터 분리 카테고리. |
| 6 | **공지** *(운영진 전용 글쓰기)* | "v0.9 업데이트" / "베타 피드백 받습니다" | 댓글은 모두 가능 |

> 베타 피드백은 별도 카테고리 만들지 않고 **공지** 카테고리 또는 **자유잡담**의 `[베타피드백]` 태그로 받는다(베타 끝나면 사라질 카테고리는 만들지 않는다).

## 6. 태그 시스템

봉기님이 말씀하신 `[]` 표기 그대로. **글당 1~3개**, 화이트리스트만 허용(자유 태그는 v2 이후).

### 6-1. 직군 태그 (글 작성 시 1개 권장)
- `[디자이너]` `[현장팀]` `[운영]`

### 6-2. 주제 태그 (글 작성 시 0~2개)
- 수플렉스 기능: `[프리셋추천]` `[견적]` `[일정]` `[마감재]` `[지출정산]` `[어드바이스]`
- 시공 영역: `[철거]` `[목공]` `[전기]` `[타일]` `[도장]` `[필름]` `[가전]`
- 스케치업 루비: `[루비공유]` `[루비개발요청]` `[루비질문]` `[튜토리얼]`
- 운영: `[채용공고]` `[구직]` `[베타피드백]` `[세무]`

### 6-3. UX
- 작성 시 드롭다운 멀티셀렉트 (입력은 한국어 그대로 보이게)
- 목록에서 태그 클릭 시 필터링
- 태그별 인기순 정렬도 v2에서

## 7. 핵심 정책

라운지 가이드라인 7항(영구 규칙들과 충돌 없게):

1. **클라이언트 비접근** — 인증된 라운지 멤버(현·전직) 만 접근. 클라이언트 절대 X.
2. **거래처 실명 비방 금지** — 자재상·시공팀·전 직장 실명 비방, 분쟁·소송 프레이밍 금지(분쟁 표현 금지 영구 규칙 연장. 명예훼손 법적 리스크).
3. **단가·시세 공유 OK** — 같은 업계 종사자 클로즈드의 힘. 자유롭게.
4. **클라이언트 식별 정보 차단** — 현장 사진의 주소·동호수·집주인 식별 정보 마스킹 의무.
5. **AI 자문 요청 글 금지** — AI 어드바이스 보류 정책과 충돌(2026-04-26).
6. **신고 기반 모더레이션** — 슈퍼어드민이 처리. 베타엔 즉시 숨김 후 24h 내 검토 SLA.
7. **닉네임 우선** — 회사명 노출은 옵션. 기본 OFF.
8. **루비 스크립트 면책** — 첨부된 `.rb` 스크립트는 **사용자가 만든 것**. 본인 책임 하에 실행. 슈퍼어드민 사전 검토 X(부담 큼). 신고 시 즉시 비공개 + 24h 내 검토. 약관 명시.

### 8-3. 데이터 모델 (Prisma 스케치)

```prisma
model LoungeMembership {        // 퇴사자도 살아남는 멤버십 레코드
  id           String   @id @default(cuid())
  userId       String   @unique
  status       String   @default("active")     // active | suspended | banned
  selfLabel    String?                          // "디자이너 / 5년차" 등
  jobRole      String?                          // designer | site | ops | etc
  grantedAt    DateTime @default(now())
  grantedReason String                          // "회사 OO 승인 시 자동 부여"
  suspendedAt  DateTime?
  suspendedBy  String?
}

model LoungePost {
  id              String   @id @default(cuid())
  authorId        String
  category        String   // knowhow | usage | ruby | free | jobs | notice
  title           String
  body            String   @db.Text                   // 마크다운(코드 블록 포함)
  showCompanyName Boolean  @default(false)
  status          String   @default("active")         // active | hidden | deleted
  pinnedToHome    Boolean  @default(false)            // "오늘의 팁" 큐레이션
  homePinnedAt    DateTime?
  homePinnedBy    String?
  reactionCount   Int      @default(0)
  commentCount    Int      @default(0)
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  comments        LoungeComment[]
  reactions       LoungeReaction[]
  tags            LoungePostTag[]
  attachments     LoungeAttachment[]
  @@index([category, status, createdAt])
  @@index([pinnedToHome, homePinnedAt])
}

model LoungeAttachment {                              // 이미지 + .rb 파일 통합
  id        String   @id @default(cuid())
  postId    String
  kind      String   // image | ruby | other
  fileName  String
  fileSize  Int
  storageKey String
  createdAt DateTime @default(now())
  @@index([postId])
}

model LoungeComment   { id, postId, authorId, body, status, createdAt }
model LoungeReaction  { id, postId, userId, type @unique([postId, userId]) }
model LoungeTag       { id, key, label, kind /* role|topic */, isActive, sortOrder }
model LoungePostTag   { postId, tagId @@id([postId, tagId]) }
model LoungeReport    { id, targetType, targetId, reporterId, reason, status, createdAt }
```

> **격리 원칙**: 기존 도메인 모델과 FK는 `userId` 외 거의 없음. 회사 데이터(Project, Quote 등)와 의도적으로 분리(지출 모듈 격리 정책과 같은 사상). 회사 탈퇴해도 라운지 데이터는 살아남음.

### 8-5. 스케치업 루비 카테고리 특수 처리

| 항목 | 처리 |
| --- | --- |
| **본문 에디터** | 마크다운 + 코드 블록(``` ```). syntax highlight 라이브러리(`prism` 또는 `highlight.js`) ruby 우선. |
| **파일 첨부** | `.rb` 글당 3개, 파일당 **1MB** 상한. 그 외 확장자 거부. |
| **저장** | `LoungeAttachment.kind = "ruby"`. 기존 이미지 업로드 인프라 재사용. |
| **다운로드** | 인증된 라운지 멤버만. 다운로드 카운트(향후 인기도 지표). |
| **작성 폼 가이드** | 카테고리 선택 시 placeholder: "스케치업 버전 / 사용법 / 주의사항을 적어주세요". |
| **면책 안내** | 첨부된 `.rb` 옆 "사용자 제작 스크립트입니다. 본인 책임으로 실행하세요" 고정 문구. |
| **신고 사유 추가** | "악성 코드 의심" 1개 추가. 신고 시 즉시 첨부 비공개. |

## 9. 등급 연계 — "최고등급 진입 명분"

| 등급 | 라운지 접근 | 글쓰기 | 프리셋 추천 글 | 프리셋 커스텀(공유) |
| --- | --- | --- | --- | --- |
| FREE / 미승인 | X | X | X | X |
| BASIC | O | O | 추천만 가능 | X |
| **PRO/최고등급** | O | O | 추천 가능 | **O — 자기 프리셋 공유 가능** |

- **흐름**: 라운지에서 "이 프리셋 좋네요" 추천 글 본 BASIC 회사가 → "우리도 만들고 싶다" → 최고등급 업그레이드.
- **연계 인프라**: `PLAN_FEATURES` 시스템 이미 존재. 라운지에 새 식별자 1~2개 추가만.
- **장기 (v3)**: 프리셋 공유 마켓 — 최고등급이 만든 프리셋을 라운지에서 한 번 클릭으로 import. 다른 시공 SaaS가 따라올 수 없는 자산.

## 10. 홈 화면 "오늘의 팁" 노출

### 10-1. 무엇을 노출하나
- **수플렉스 사용 팁** 카테고리 글 중에서
- 슈퍼어드민이 `pinnedToHome = true` 큐레이션한 글 1개
- 미큐레이션 시 자동 fallback: 최근 7일 내 공감 ≥ 3 인기글 1개

### 10-2. 어디에 노출하나
- 홈 화면 상단(히어로 영역 아래) 또는 사이드 카드 1개
- 형식: 제목 + 직군 라벨 + 첫 80자 + "라운지에서 보기"
- 닫기 버튼: localStorage 24h 숨김

### 10-3. 효과
- 사용자가 매일 1개의 새로운 사용법 학습
- 라운지 트래픽 자연 유입
- 사용 능력 향상 → 활용도·리텐션 동시 상승
