# 챕터 0. 시작하기 전에

> 이 챕터를 읽고 나면 — Suplex가 무엇이고, 무엇이 아닌지를 명확히 알게 됩니다. 이후 챕터들이 훨씬 빠르게 이해됩니다.

---

## 0.1 Suplex가 다루는 것 / 다루지 않는 것

Suplex는 **인테리어 회사 직원만의 도구**입니다. 견적 작성, 마감재 관리, 발주, 일정, 회사 회계까지 인테리어 회사 내부 업무 전반을 한 곳에서 다룹니다.

### Suplex가 다루는 것 ✅

| 영역 | 무엇 |
|---|---|
| 견적 | 간편 견적 (1장짜리), 상세 견적 (한국 표준 양식) — 둘 다 PDF 출력 |
| 마감재 | FINISH (자재) + APPLIANCE (가전) 통합 관리. 6상태로 진행도 추적 |
| 발주 | 거래처 발주서 (PurchaseOrder) 생성 + 마감재 변경 자동 감시 |
| 일정 | 공정 일정 캘린더 + 카톡 친화 일정 복사 + 발주 데드라인 자동 계산 |
| 현장 | 일일 현장보고 (사진 첨부), 체크리스트 |
| 회계 | 지출관리 (계정과목별), AI 비서 (자연어 질문). 회사 단위로 **숨김 토글** 가능 (직원 공유 계정용) |
| 팀 | 회사·역할·협력업체 마스터, 초대 토큰 |
| 변경 추적 | 견적 차수 (1차/2차/...), 마감재 이력, 발주 후 변경 ⚠️ 알림 |
| 공정 자동 인식 | 일정에 "목공" 입력 시 PhaseAdvice 매칭 → 사전 체크리스트 자동 생성 |
| 가전 규격 DB | 366개 모델 자동 검색 + 사용자 입력 자동 학습 (modelCode + brand + size 누적) |

### Suplex가 다루지 않는 것 ❌

| 안 다루는 것 | 이유 |
|---|---|
| 클라이언트(공사 의뢰인) 직접 접근 | 회사 영업 정보 보호. 클라이언트는 PDF만 받음 |
| 토큰 링크 / 고객 폼 / 클라이언트 로그인 | 위와 동일 — 만들지 않을 원칙 |
| 일반 회계 ERP 풀 기능 | 인테리어 업무 흐름에 맞춘 경량 회계만 |
| 실시간 채팅 / 메신저 | 카카오톡 등 이미 잘 작동하는 도구 활용 |
| 도면(CAD) 자체 작성 | 외부 CAD 결과물 첨부만 |
| 마케팅 / 자재 거래소 / 가격 비교 | 본업이 아님 |

> 💡 "Suplex가 안 다루는 것"을 명확히 정해두는 것이 중요합니다. 모든 걸 다 하려는 SaaS는 어느 것도 잘 못합니다. Suplex는 **B2B 인테리어 업무에만** 집중합니다.

## 0.2 핵심 용어 빠른 정리

전체 용어집은 [glossary.md](glossary.md)에 있습니다. 이 페이지에서는 매뉴얼 내내 자주 등장하는 11가지 용어만 빠르게:

| 용어 | 1줄 정의 |
|---|---|
| **마감재** | 공사에 들어가는 모든 자재·가전. FINISH(자재) + APPLIANCE(가전) 통합 |
| **공정** | 작업 단위 (철거·도배·전기 등). 상세 견적의 척추 |
| **spaceGroup** | 마감재를 묶는 공간 단위 (거실·주방·욕실 등). 자유 텍스트 |
| **간편 견적** | 공정별 1장짜리 빠른 견적 |
| **상세 견적** | 한국 표준 양식 갑지 + 18행 원가내역서 |
| **2차 견적** | 변경 후 새로 만든 견적. 1차는 그대로 보존 (SUPERSEDED) |
| **CONFIRMED / CHANGED** | 마감재 6상태 중 가장 자주 보게 될 둘. 확정 / 변경됨 |
| **회사 템플릿** | 평소에 쓰는 마감재·공정 기본 SET. 프로젝트마다 자동 복사 |
| **PhaseAdvice / 사전 체크리스트** | 공정 입력 시 D-3~D-1 사전 액션이 자동 생성되는 표준 어드바이스 |
| **modelCode (품번)** | 가전 모델 자동 학습의 키. `brand`=모델명, `modelCode`=품번으로 분리 |
| **기능 표시 설정 (hideExpenses)** | 회사 단위로 지출관리·AI비서 전부 숨김. 직원 공유 계정에서 영업 데이터 차단용 |

## 0.3 클라이언트는 Suplex에 접근할 수 없습니다 — 우리만의 도구

이 원칙은 매뉴얼 어디에서도 깨지지 않습니다. **반드시 알고 시작해주세요.**

### 무엇이 가능한가

| 작업 | 가능 여부 |
|---|---|
| 회사 직원이 Suplex에 로그인 | ✅ |
| 협력업체(거래처) 마스터 정보 등록 | ✅ |
| 클라이언트에게 PDF 견적서 발송 | ✅ (카카오톡·이메일 등 외부 채널 사용) |
| 클라이언트에게 PDF 발주 명세 발송 | ✅ |
| 클라이언트가 Suplex 화면을 직접 보기 | ❌ |
| 클라이언트에게 "이 링크로 들어와서 확인해주세요" | ❌ |
| 클라이언트가 비밀번호로 로그인하기 | ❌ |
| 클라이언트가 견적서에 직접 코멘트 달기 | ❌ |

### 왜 이렇게 만들었나

1. **영업 정보 보호** — Suplex 안에는 회사 매입가, 마진율, 다른 프로젝트 단가 같은 민감 정보가 가득합니다. 클라이언트가 직접 보면 협상 우위가 깨집니다.
2. **신뢰 기반 업무** — 인테리어 업계는 사장님↔클라이언트의 직접 신뢰가 핵심입니다. 시스템이 끼면 그 신뢰가 약해집니다.
3. **PDF로 충분** — 클라이언트가 알아야 할 정보는 견적서 PDF만으로 다 전달 가능합니다.

### 매뉴얼에서 지켜야 할 표현

- ❌ 안 좋은 예시: "클라이언트가 이 링크로 들어오시면..."
- ❌ 안 좋은 예시: "고객님께 로그인 정보를 보내드립니다."
- ✅ 좋은 예시: "PDF 견적서를 카카오톡으로 보내드리시면 됩니다."
- ✅ 좋은 예시: "고객님께 PDF로 발송한 후, 회신은 카카오톡이나 전화로 받으시면 됩니다."
