# 챕터 8. 발주

> 이 챕터를 읽고 나면 — 거래처에 보낼 발주서를 만들고, 마감재와 연결해 발주 후 변경을 자동 감시하며, 입고·수령을 추적할 수 있게 됩니다.

---

## 8.1 발주란

**발주(PurchaseOrder)**는 회사가 거래처에 자재·인력·서비스를 주문하기 위해 보내는 문서입니다. 클라이언트 견적과는 별개입니다:

| | 견적 | 발주 |
|---|---|---|
| 누가 누구에게 | 회사 → 클라이언트 | 회사 → 거래처 |
| 무엇 | 받을 돈 약속 | 줄 돈 약속 |
| 형식 | 한 장의 견적서 | 거래처별 주문서 |
| 변경 시 위험 | 분쟁 | 위약금·재발주 비용 |

---

## 8.2 발주서 만들기

프로젝트 → **발주** 탭 (6번 탭) → **+ 새 발주**.

### 입력 항목

| 항목 | 입력 |
|---|---|
| 거래처 | 협력업체 마스터에서 선택 |
| 발주 자재 | 마감재 탭에서 선택 (다중 선택 가능) |
| 수량·단위 | 자동 채움 (마감재 기준) |
| 매입 단가 | 거래처 매입가 (회사 매입 기준) |
| 발주일 | 오늘 |
| 입고 예정일 | 거래처 약속일 |
| 결제 조건 | "선금 30% / 잔금 70% (입고 후 30일)" 등 |
| 비고 | 특이사항 |

[스크린샷 자리: 발주 작성 화면]

### 자동 연결

발주 등록 시 해당 마감재 행이 발주와 연결됩니다. 마감재 탭에서 그 항목 옆에 **"발주됨"** 표시.

---

## 8.3 발주 상태

| 상태 | 의미 | 다음 |
|---|---|---|
| `DRAFT` | 작성 중 | 거래처 발송 → ORDERED |
| `ORDERED` | 발주 완료, 거래처 확인 | 입고 시 → DELIVERED |
| `DELIVERED` | 입고 완료 | 검수 후 → ACCEPTED |
| `ACCEPTED` | 수령·검수 완료 | (종료) |
| `CANCELLED` | 취소 | (종료) |

---

## 8.4 ⚠️ 발주 후 마감재 변경 자동 감시

이게 발주 탭의 가장 가치 있는 기능입니다.

### 시나리오

```
1. 욕실 타일 화이트 무광 발주 (ORDERED)
2. 클라이언트가 "그레이로 변경" 요청
3. 마감재 탭에서 itemName 변경
4. 자동으로 ⚠️ 알림 발생
5. 거래처에 즉시 통보 또는 재발주
```

### 자동 감시 메커니즘

`PurchaseOrder.materialChangedAt` 필드가 자동 업데이트됩니다:
- 발주 후 그 마감재가 수정되면 타임스탬프 기록
- 홈 화면, 프로젝트 카드, 발주 탭에 ⚠️ 표시

### 처리 절차

1. ⚠️ 클릭 → 어느 발주의 어떤 자재가 바뀌었는지 확인
2. 거래처에 즉시 연락
3. 옵션:
   - **배송 전 변경 가능** → 거래처에 변경 통보, 새 사양으로
   - **배송 후/생산 후** → 기존 자재 사용 또는 추가 발주 + 비용 정산

---

## 8.5 발주 PDF 출력

거래처에 보낼 발주서를 PDF로 출력 (인쇄 → PDF로 저장).

거래처가 사업자라면 세금계산서 발행 정보가 자동 채워집니다.
