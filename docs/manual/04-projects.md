# 챕터 3. 첫 프로젝트 만들기

> 이 챕터를 읽고 나면 — 새 프로젝트를 생성하고, 10개의 탭이 무엇을 하는지 이해하며, 어느 탭부터 손대야 할지 알게 됩니다.

---

## 3.1 새 프로젝트 만들기

좌측 메뉴 **프로젝트** 클릭 → 우측 상단 **+ 새 프로젝트** 버튼.

### 입력 항목

| 항목 | 입력 예시 | 필수 |
|---|---|---|
| 프로젝트명 | `강남 래미안 304-1502 리모델링` | ✅ |
| 고객명 | `박상철` | ✅ |
| 연락처 | `010-9876-5432` | 권장 |
| 현장 주소 | `서울 강남구 도곡동 123, 강남 래미안 304동 1502호` | ✅ |
| 면적 (평) | `30` | ✅ |
| 시작 예정일 | `2026-05-15` | 권장 |
| 종료 예정일 | `2026-06-26` | 권장 |
| 예상 견적 | `50,000,000` (참고용) | 선택 |
| 비고 | 클라이언트 특이사항 | 선택 |

> 💡 **프로젝트명 명명 규칙 추천**: `[지역] [건물명] [동·호수] [공사 종류]` — 나중에 검색·정렬이 편합니다.

[스크린샷 자리: 새 프로젝트 만들기 모달]

### 만들면 자동으로 일어나는 일

1. 프로젝트가 생성됨
2. **회사 마감재 템플릿 자동 복사** — 회사가 평소에 쓰는 마감재 SET이 이 프로젝트에 자동으로 들어감
3. 견적 라인 템플릿 사용 가능 상태로 됨
4. 견적 기본비율(디자인비·부가세 등)이 회사 기본값으로 미리 채워짐
5. 자동으로 프로젝트 상세 페이지로 진입 (첫 탭 = 견적)

> 💡 회사 템플릿이 잘 정리되어 있을수록 새 프로젝트 진입 시 빈 화면이 아니라 **30~50% 채워진 상태**가 됩니다. 회사 도입 첫 주의 [Playbook #2: 회사 템플릿](playbooks/02-company-template.md) 작업이 그래서 중요합니다.

## 3.3 어느 탭부터 손대야 하나 — 실무 진입 순서

탭 10개를 보고 막막하실 수 있습니다. **실제 업무 흐름은 정해져 있습니다**:

### 첫 미팅 직후 (Day 1)

```
1) 프로젝트 생성                         ← 위에서 마침
2) 메모 탭 (10번)                        ← 미팅 노트 빠르게 카드 메모로 저장
3) 마감재 탭 (4번)                       ← 클라이언트 요청 사양 정리
4) 견적 탭 (1번)                         ← 1차 간편 견적 작성·발송
```

이게 가장 일반적입니다. **메모 → 마감재 → 견적** 순서.

### 견적 합의 후 (Day 7~14)

```
5) 공정 일정 탭 (3번)                    ← 공정별 작업 일정 입력
6) 발주 탭 (6번)                         ← 자재·가전 발주
7) 체크리스트 탭 (7번)                   ← 공종별 점검 항목 셋업
```

### 공사 시작 후 (매일)

```
8) 현장보고 탭 (8번)                     ← 매일 사진 + 진행 상황
9) 지출 탭 (9번)                         ← 영수증·세금계산서 입력
```

### 변경 발생 시 (수시)

```
10) 마감재 탭 → 상태 CHANGED로 전환
11) 견적 탭 → 2차 견적 만들기
12) 발주 후 변경이면 ⚠️ 알림 처리
```

자세한 변경 흐름은 [Playbook #1: 30평 아파트](playbooks/01-30py-apartment-full.md) 참고.

## 3.4 프로젝트 상태

프로젝트는 다음 상태를 거칩니다 (자동 또는 수동):

| 상태 | 의미 | 다음 단계 |
|---|---|---|
| `PLANNING` | 견적 작성 단계 | 견적 발송 후 → ESTIMATING |
| `ESTIMATING` | 견적 발송됨, 클라이언트 검토 중 | 합의 후 → ACTIVE |
| `ACTIVE` | 공사 진행 중 | 마감 후 → COMPLETED |
| `COMPLETED` | 공사 완료, 정산 완료 | 보관 |
| `CANCELLED` | 도중 취소 | 보관 |

> 💡 상태는 **수동으로 직접 바꾸시는 게 권장**됩니다. 예: 1차 견적 발송 후 합의되면 `PLANNING → ACTIVE`. 자동 변경은 분쟁 시 모호함을 만듭니다.

---

## 3.5 프로젝트 검색·필터

좌측 메뉴 **프로젝트**에서:

| 필터 | 사용 |
|---|---|
| 상태별 | PLANNING / ACTIVE / COMPLETED |
| 면적 범위 | 10~20평, 20~30평, 30~50평 등 |
| 진행 기간 | 이번 달, 지난 달, 분기 |
| 검색어 | 프로젝트명, 고객명, 주소 |

---

## 3.6 프로젝트 백업

각 프로젝트는 햄버거 메뉴 → **JSON 백업**으로 전체 데이터를 다운로드 받을 수 있습니다.

| 용도 | 빈도 |
|---|---|
| 정산 완료 후 영구 보관 | 분기 1회 |
| 분쟁 발생 시 증거 확보 | 즉시 |
| 회사 변경(서버 이전 등) 대비 | 월 1회 |

**백업 내용**: 프로젝트 정보, 마감재, 모든 견적 차수, 일정, 발주, 체크리스트, 현장보고, 지출, 메모.
