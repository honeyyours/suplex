## 1. 이 문서가 존재하는 이유

Suplex를 도입한 인테리어 업체 직원이 이 매뉴얼만 보고 — **출근 첫날 혼자서 견적서 1건을 발송**할 수 있도록 한다. 그것이 매뉴얼 라인업 전체의 단일한 목표다.

### 성공 지표

| 지표 | 목표 |
|---|---|
| 도입 업체의 신규 직원 온보딩 시간 | 1주 → **1일** |
| 사용자 → Suplex팀 운영 문의 | **80% 감소** |
| 시범 고객 만족도 | "있어서 도움 됐다" 5/5 |

   0.2 핵심 용어 빠른 정리
   0.3 클라이언트는 Suplex에 접근할 수 없습니다 — 우리만의 도구

   1.1 계정 만들기
   1.2 회사 정보 등록
   1.3 ★ 회사 템플릿 (마감재 기본 SET) — 가장 중요한 초기 작업

2. 홈 화면 사용법

3. 첫 프로젝트 만들기
   3.1 프로젝트 생성
   3.2 10개 탭 한눈에 (간편견적/상세견적/공정일정/마감재/마감재고급/발주/체크리스트/현장보고/지출/메모)
   3.3 어느 탭부터 손대야 하나 — 실무 진입 순서

4. 마감재 관리
   4.1 spaceGroup 만들기
   4.2 FINISH vs APPLIANCE
   4.3 6가지 상태 (UNDECIDED/REVIEWING/CONFIRMED/CHANGED/REUSED/NOT_APPLICABLE)

   4.5 인라인 엑셀 편집 (키보드 네비, 자동 저장)

   4.7 다크모드

5. 간편 견적서
   5.1 표준 구조: 공정 합계 → 디자인비(기본 10%) → 단수조정 → 부가세(기본 0%, 사용자 설정) → 총합계

   5.3 직전 차수와 비교 (AI 자동 요약)

   6.3 마감재와의 연결
   6.4 간편 vs 상세 — 언제 어느 걸 쓰나
   6.5 PDF 출력

7. 변경 관리 (2차 견적)
   7.1 변경 = 견적 2차 원칙 (1차는 수정하지 않음)
   7.2 SimpleQuote SUPERSEDED 상태로 차수 잠그기
   7.3 MaterialHistory — 누가/언제/무엇을 바꿨는지
   7.4 발주 후 마감재 변경 ⚠️ 표시

8. 발주 (이미 구현된 기능)

9. 체크리스트

## 12. 운영 원칙 (작성·다듬기 규칙)

매뉴얼을 직접 만지는 사람이 따라야 할 규칙. 이게 일관성을 결정한다.

1. **클라이언트 비접근 원칙은 절대 위반 금지** — 매뉴얼 어디에도 "고객님이 이 링크로 들어오시면" 같은 문구 X. Suplex는 인테리어 회사 직원만의 도구.
2. **실제 화면과 일치 보장** — 코드 변경 시 해당 챕터 1주 내 업데이트. 안 그러면 매뉴얼 신뢰도 즉시 0.
3. **예시 데이터는 가상**이되 **현실감 유지** — 회사명·프로젝트명·금액 모두 그럴듯한 가상값. "ABC인테리어", "강남 30평", "12,500,000원" 같은 톤.
4. **금액·면적·치수는 한국 인테리어 시장 실제 단가 참고** — "10평 도배 단가 80만원" 처럼 사용자가 보고 "맞네" 할 수 있게.
5. **각 챕터 첫 줄에 "이 챕터를 읽고 나면 무엇을 할 수 있는가" 한 문장** 명시. 안 그러면 사용자가 어디 와 있는지 모름.
6. **각 챕터 끝에 "다음 챕터로" 링크** — 학습 흐름 끊지 않게.
7. **부정문보다 긍정문** — "X하지 마세요" 보다 "Y하세요"가 5배 잘 읽힘.
