# 부록 B. 용어집 (Glossary)

> 이 챕터를 읽고 나면 — 매뉴얼 어디에 등장하는 용어든 의미를 정확히 알게 됩니다.

매뉴얼에서 중요한 용어는 **굵게** 표시되고, 처음 등장할 때 이 용어집의 짧은 정의가 옆에 따라옵니다. 헷갈릴 때마다 이 페이지로 돌아오시면 됩니다.

---

## 가나다 순서

### ㄱ

**간편 견적**
큰 단위로 견적을 빠르게 뽑는 방식. 공정별 합계만 보여주는 한 장짜리 견적. 첫 미팅, 바쁜 고객, 간단한 부분 공사에 적합. → 챕터 5 / [Playbook #5](playbooks/05-simple-vs-detailed.md)

**견적**
공사를 얼마에 해줄지 정리한 문서. Suplex에서는 두 종류가 있다 — 간편 견적과 상세 견적.

**견적 2차**
(→ 2차 견적)

**견적 이력**
한 프로젝트에서 만들어진 모든 견적의 시간순 기록. 1차, 2차, 3차… 차이를 비교할 수 있다. 분쟁 발생 시 결정적 증거가 된다. → 챕터 7 / [Playbook #8](playbooks/08-dispute-evidence.md)

**공정**
공사를 구성하는 작업 단위. 예: 철거 공정, 도배 공정, 전기 공정. **상세 견적의 척추**가 되는 개념. Suplex는 공정 사이에 선후관계를 강제하지 않는다 — 작은 공사부터 큰 공사까지 같은 프레임을 쓸 수 있도록.

### ㄷ

**다크모드**
Suplex 화면을 어두운 배경 + 밝은 글자로 보는 옵션. 마감재처럼 표 데이터를 오래 보는 작업에서 눈 피로를 크게 줄여준다. 설정 메뉴에서 토글. → 챕터 4.5

### ㄹ

(없음)

### ㅁ

**마감재**
공사에 들어가는 모든 자재·가전. Suplex 안에서는 **FINISH**(타일·도배·바닥재 등 자재)와 **APPLIANCE**(가전제품)를 통합해 다룬다. 한 모델에 모든 종류 자재·가전을 같이 등록한다. → 챕터 4

**마감재 6상태**
한 마감재가 가질 수 있는 여섯 가지 상태:

| 상태 | 의미 |
|---|---|
| `UNDECIDED` (미정) | 아직 모델·자재가 정해지지 않음. 견적엔 임시 단가로 들어감 |
| `REVIEWING` (검토 중) | 후보가 좁혀졌으나 최종 결정 전 |
| `CONFIRMED` (확정) | 모델·단가 모두 정해짐. 견적·발주 모두 신뢰 가능 |
| `CHANGED` (변경됨) | 1차 견적 후 변경된 항목. 2차 견적에서 강조 표시 |
| `REUSED` (재사용) | 기존 자재 그대로 사용 (예: 멀쩡한 신발장 유지) |
| `NOT_APPLICABLE` (미적용) | 견적에서 빼는 항목 |

상태에 따라 화면 표시·견적 반영이 달라진다. → 챕터 4.3

### ㅂ

**부가세 (VAT)**
공급가액에 곱해지는 비율. **Suplex 기본값은 0%** (별도 안내). 사용자가 견적별로 % 지정 가능 — 일반적으로 10%를 입력하지만, 면세 사업장이거나 별도 안내 처리할 때는 0%로 둔다. → 챕터 5.1

### ㅅ

**상세 견적**
모든 공정·자재를 한 줄씩 펼쳐서 보여주는 정밀한 견적. 큰 공사, 까다로운 클라이언트, 변경이 잦은 프로젝트에 적합. → 챕터 6 / [Playbook #5](playbooks/05-simple-vs-detailed.md)

**설계감리비**
공정 합계의 **10%**. 인테리어 디자인 + 시공 감리에 들어가는 비용으로 견적서에 별도 라인으로 표시한다. Suplex 간편 견적의 표준 구조에 포함. → 챕터 5.1

**spaceGroup (스페이스 그룹)**
한 프로젝트 안에서 마감재를 묶는 공간 단위. 자유 텍스트라 회사마다 다르게 쓸 수 있다. 예: "거실", "안방", "주방", "공용욕실", "발코니 확장 공간". 입력 형식이 정해져 있지 않은 이유는 — 회사마다 부르는 이름이 다르기 때문.

### ㅇ

**APPLIANCE (어플라이언스)**
가전제품 카테고리의 마감재. 냉장고·식기세척기·인덕션·후드·세탁기 등. (→ FINISH)

**AI 경리**
지출·정산을 AI가 도와주는 기능. **현재 정식 출시 전 보류 상태** — 정식 출시 시 구독 등급 차등용으로 풀린다. 매뉴얼에는 위치만 표시되고, 풀리는 시점에 챕터 10이 채워진다.

**인라인 엑셀 편집**
표 위에서 셀을 클릭해서 그 자리에서 바로 값을 바꾸는 방식. 새 화면 띄우거나 모달 열지 않는다. 마감재처럼 수십 행을 빠르게 다듬을 때 쓴다. → 챕터 4.4

**일정 메뉴 (상위)**
홈 옆에 있는 상위 메뉴. **Phase 17에서 신설 예정** — 회사 전체의 공정 일정을 한 화면에서 보는 용도. (프로젝트 안의 "공정 일정" 탭은 이미 존재 — 헷갈리지 말 것.)

**프로젝트 10탭**
프로젝트 상세 페이지에 있는 10개의 탭. 들어가면 자동으로 첫 번째 탭(간편견적)이 열린다. 순서대로:

| # | 탭 | 역할 |
|---|---|---|
| 1 | 견적 (간편) | 공정별 1장짜리 빠른 견적 |
| 2 | 상세 견적 | 한국 표준 양식 갑지 + 18행 원가내역서 |
| 3 | 공정 일정 | 공정 단위 작업 간트 차트 |
| 4 | 마감재 | 간편 마감재 — FINISH/APPLIANCE 통합 인라인 편집 |
| 5 | 마감재 (고급) | 폼 스키마 기반 정밀 에디터 + 변경 이력 추적 |
| 6 | 발주 | PurchaseOrder 등록·관리 |
| 7 | 체크리스트 | 공종별 점검 항목 |
| 8 | 현장보고 | DailyReport, 사진 첨부 |
| 9 | 지출 | 현장별 영수증·세금계산서 |
| 10 | 메모 | Google Keep 스타일 카드 메모 |

→ 챕터 3.2

**2차 견적**
1차 견적을 보낸 후 변경이 발생했을 때 만드는 새 견적. **변경 = 견적 2차** 원칙 — Suplex에서는 1차를 수정하지 않고 새로 2차를 만든다. 이렇게 해야 "원래 견적이 어땠고, 무엇이 어떻게 바뀌었는지" 추적 가능. → 챕터 7

### ㅈ

**지출관리**
프로젝트별 실제 지출(영수증·세금계산서 등)을 기록하는 메뉴. 견적과 비교해 마진을 본다. → 챕터 13

**직전 차수 비교 (간편 견적)**
간편 견적에서 한 프로젝트의 두 견적 차수를 비교해, 변경 사항을 AI가 자동 요약해 footerNotes(견적서 하단 안내문)에 넣어주는 기능. 2차 견적 보낼 때 "1차 대비 어디가 어떻게 달라졌는지" 클라이언트에게 한 줄로 전달 가능. → 챕터 5.3, 7

### ㅊ

**총합계**
견적의 마지막 줄. **공정 합계 + 디자인비(기본 10%) + 단수조정 + 부가세(기본 0%, 사용자 설정) = 총합계**. 클라이언트에게 보내는 PDF에서 가장 큰 글씨로 표시.

(없음)

### ㅌ

회사가 평소에 쓰는 마감재·공정의 기본 SET. 한 번 만들어두면 새 프로젝트마다 처음부터 입력 안 해도 된다. **Suplex 도입 시 가장 먼저 해야 하는 일.** → 챕터 1.3 / [Playbook #2](playbooks/02-company-template.md)

## 영문·축약어

### F

**FINISH (피니쉬)**
자재 카테고리의 마감재. 타일·도배·바닥재·페인트·문짝 등. (→ APPLIANCE)

**formKey**
마감재의 **표준 형식 키**. 미리 정의된 형식의 마감재를 등록할 때 쓰인다 — 예를 들어 "타일 600×600 화이트 무광"처럼 회사 템플릿에 등록된 표준 마감재. (→ customSpec)

### C

**customSpec**
마감재의 **자유 입력 사양**. formKey로 표현할 수 없는 비표준·일회성 마감재를 위한 메모. 예: "고객이 직접 가져온 수입 타일", "특수 사이즈 주문 제작 가구". (→ formKey)

### P

**PDF 견적서**
간편 견적·상세 견적 모두 PDF로 출력 가능. **클라이언트에게 보내는 형태는 PDF만** — Suplex 자체에는 클라이언트가 접근할 수 없도록 설계되어 있다.

**Phase 17**
Suplex 개발 로드맵에서 일정 탭이 신설되는 단계. v0.5 매뉴얼 시점에 진행 상태 확인 필요.

### S

**SaaS**
Software as a Service. Suplex는 인테리어 회사용 B2B SaaS다.

**Suplex (구 SPLEX)**
이 SaaS의 정식 명칭. 2026-04-23 SPLEX → Suplex로 리네이밍, 2026-04-28 폴더명도 splex → suplex로 정리 완료. 단 `splex_token` 같은 일부 LEGACY 식별자는 호환성을 위해 의도적으로 보존됨.

## 자주 헷갈리는 짝

| 헷갈리는 짝 | 차이 |
|---|---|
| **간편 견적 vs 상세 견적** | 간편 = 공정별 1장짜리, 상세 = 한국 표준 양식 갑지 + 18행 원가내역서 |
| **마감재 vs 마감재 (고급)** | 마감재 = 인라인 편집 빠른 입력, 고급 = 폼 스키마 + 변경 이력 추적 |
| **FINISH vs APPLIANCE** | FINISH = 자재(타일·도배 등), APPLIANCE = 가전제품. 둘 다 "마감재"로 통합 관리 |
| **formKey vs customSpec** | formKey = 표준 정해진 마감재, customSpec = 자유 메모형 마감재 |
| **공정 vs spaceGroup** | 공정 = 작업 단위(철거·도배), spaceGroup = 공간 단위(거실·주방) |
| **1차 견적 vs 2차 견적** | 1차 = 처음 보낸 견적(SUPERSEDED 처리), 2차 = 변경 후 새로 만든 견적 |
| **공정 일정 (탭) vs 일정 (상위 메뉴)** | 탭 = 한 프로젝트 내 공정 간트, 메뉴 = 회사 전체 일정 (Phase 17 신설) |
| **지출관리 (메뉴) vs 지출 (탭)** | 메뉴 = 회사 전체 회계, 탭 = 한 프로젝트의 지출 |
| **발주 (Order) vs 견적 (Quote)** | 발주 = 거래처에 보낼 주문서, 견적 = 클라이언트에게 받을 돈 약속 |

## 클라이언트 비접근 원칙 (모든 챕터 공통 약속)

매뉴얼 어디에서도 이 원칙은 깨지지 않습니다:

> **Suplex는 인테리어 회사 직원만의 도구입니다. 회사의 클라이언트(공사를 의뢰한 분)는 Suplex에 접근하지 않습니다.**
> 클라이언트에게는 **PDF 견적서만** 전달됩니다. 토큰 링크·고객 폼·클라이언트 로그인 같은 기능은 **만들지 않습니다.**

이 원칙이 있는 이유:
- 인테리어 업계는 신뢰 기반의 업무 — 시스템이 노출되면 영업 정보가 새어나갈 수 있다.
- 클라이언트가 Suplex 화면을 직접 보면 "이 자재 단가가 왜 이래?" 같은 협상 우위가 깨진다.
- PDF만으로도 클라이언트는 충분한 정보를 얻을 수 있다.
