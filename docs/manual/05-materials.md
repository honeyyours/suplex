# 챕터 4. 마감재 관리

> 이 챕터를 읽고 나면 — 마감재 탭에서 자재(FINISH)와 가전(APPLIANCE)을 통합 관리하고, 6가지 상태로 진행도를 추적하며, 인라인 엑셀 편집으로 빠르게 입력할 수 있게 됩니다.

마감재 관리는 **Suplex의 가장 핵심적인 일상 업무**입니다. 매일 보게 됩니다.

---

## 4.1 spaceGroup — 공간 단위로 묶기

마감재의 가장 기본적인 분류 축은 **spaceGroup**입니다. 자유 텍스트라 회사가 평소에 부르는 이름을 그대로 쓰시면 됩니다.

### 권장 spaceGroup

| spaceGroup 예시 | 사용 빈도 |
|---|---|
| 거실 | 매우 자주 |
| 안방 / 주방 / 공용욕실 / 안방욕실 | 매우 자주 |
| 작은방-1, 작은방-2 (방 개수만큼) | 자주 |
| 현관 / 베란다 / 창호 / 발코니 | 자주 |
| 다용도실 / 보일러실 / 펜트리 | 가끔 |
| 옥상 / 마당 (주택) | 드물게 |

> 💡 **자유 텍스트의 장점**: 회사마다 부르는 이름이 다르거나 (예: "주방" vs "키친"), 한 프로젝트의 특수 공간(예: "옷장방", "와인 셀러")도 쉽게 추가 가능. 단점은 **오타 위험** — `주방`과 `주 방`(공백 들어감)이 다른 그룹으로 인식됩니다. 회사 내에서 표기 통일 권장.

### subgroup (선택)

spaceGroup 안에서 더 세분화하고 싶을 때 `subgroup` 컬럼을 사용:

| spaceGroup | subgroup | itemName |
|---|---|---|
| 거실 | 바닥·벽·천장 | 강마루 |
| 거실 | 바닥·벽·천장 | 도배 |
| 거실 | 조명·전기 | LED 다운라이트 |
| 거실 | 조명·전기 | 콘센트 추가 |

subgroup은 비워두셔도 됩니다. 화면 정렬에만 영향.

## 4.2 FINISH vs APPLIANCE

마감재는 두 종류로 나뉘지만 **같은 테이블에서 통합 관리**됩니다:

| kind | 의미 | 예시 |
|---|---|---|
| **FINISH** | 자재 (시공·설치되는 모든 비가전) | 타일, 도배, 강마루, 등기구, 양변기, 수전, 가구 |
| **APPLIANCE** | 가전제품 | 식기세척기, 인덕션, 후드, 냉장고, 세탁기, 에어컨 |

### APPLIANCE 전용 필드

가전은 자재와 다르게 추가 정보가 필요합니다:

| 필드 | 설명 | 예시 |
|---|---|---|
| `installed` | 설치 여부 | true (빌트인) / false (스탠드형) |
| `size` | 외형 사이즈 (W × D × H mm) | "598 × 567 × 815" |
| `remarks` | 특이사항 | "걸레받이 150mm용", "다리 60mm 조절 가능" |

> 💡 **가전 규격 마스터**가 단계적으로 시드되고 있습니다 (설정 → 가전 규격). 모델명을 입력하면 사이즈·걸레받이가 자동 채워질 수 있습니다 — 단, 매칭 안 되는 모델은 직접 입력해주셔야 합니다. 자세한 흐름은 [appliance-db-plan.md](../../appliance-db-plan.md) 참고.

### 가전 모델명 표기 권장

| 패턴 | 예시 |
|---|---|
| `[브랜드] [모델명]` | `LG DUE6BGL2E`, `삼성 DW80F75K1USWS` |
| `[제품명] ([모델명])` | `LG 디오스 식기세척기 (DUE6BGL2E)` |

매뉴얼 내 표기는 자유지만 **회사 내에서 한 가지로 통일** 권장 — 검색·필터 정확도에 영향.

## 4.3 6가지 상태 (MaterialStatus)

각 마감재는 다음 6가지 상태 중 하나를 가집니다. **상태에 따라 화면 표시·견적 반영이 달라집니다**.

| 상태 | 한글 | 색상 (기본) | 견적 반영 | 언제 |
|---|---|---|---|---|
| `UNDECIDED` | 미정 | 회색 | 임시 단가 | 모델·자재가 아직 안 정해짐 |
| `REVIEWING` | 검토 중 | 노랑 | 임시 단가 | 후보가 좁혀졌으나 결정 전 |
| `CONFIRMED` | 확정 | 녹색 | 실제 단가 | 모델·단가 모두 정해짐. **가장 자주 보게 될 상태** |
| `CHANGED` | 변경됨 | 주황 | 새 단가 | 1차 견적 후 변경된 항목. 2차 견적에서 강조 표시 |
| `REUSED` | 재사용 | 파랑 | 0 또는 일부 | 기존 자재 그대로 사용 (예: 멀쩡한 신발장 유지) |
| `NOT_APPLICABLE` | 미적용 | 흰색 | 제외 | 견적에서 빼는 항목 |

### 상태 전환 흐름

전형적인 상태 흐름:

```
[프로젝트 생성]
       ↓
   UNDECIDED ─┐
       ↓     │
   REVIEWING ┘← 클라이언트와 협의 중
       ↓
   CONFIRMED ← 결정! 견적·발주 모두 신뢰 가능
       ↓
   ┌─────────┴─────────┐
   ↓                   ↓
CHANGED            REUSED
(변경 발생)        (안 바꾸기로 결정)
   ↓
[2차 견적 발행]
```

### 상태별 활용 팁

- 미팅 직후엔 모든 항목을 **UNDECIDED**로 시작.
- 클라이언트 합의된 항목부터 차례로 **CONFIRMED**로.
- 합의 못 한 항목은 계속 UNDECIDED — 클라이언트가 잊으셨을 때 "X, Y, Z 자재 결정 부탁드립니다" 한 번에 알릴 수 있음.
- 1차 견적 발송 후 변경 요청 들어오면 해당 항목만 **CHANGED**로 바꾸기. 다른 항목은 그대로 둠.
- **NOT_APPLICABLE** 활용: 회사 템플릿에 들어있는데 이번 프로젝트에서 안 쓰는 항목 (예: 30평인데 옥상 자재). 삭제하지 말고 NOT_APPLICABLE로.

## 4.4 formKey vs customSpec — 표준 vs 자유 입력

마감재의 사양 입력은 두 가지 방식이 있습니다:

### formKey — 표준 폼 사용

회사 템플릿에 등록된 마감재 형식 키를 사용합니다. 미리 정해진 필드들이 폼으로 뜹니다.

예: `formKey = "wallpaper"` (도배지) 선택 시
- 색상
- 패턴
- 폭 (cm)
- 도배지 종류 (실크 / 합지 / 천연)
- 단가 (원/m²)

이미 정해진 형식이라 누락 필드가 없고 통계 가능.

### customSpec — 자유 입력

formKey로 표현 안 되는 비표준·일회성 마감재용. JSON으로 자유롭게 키-값 저장.

예시:
```json
{
  "재질": "고객 직접 가져온 수입 타일",
  "원산지": "이탈리아",
  "사이즈": "300×600×10mm",
  "특이사항": "납기 6주, 사전 발주 필수"
}
```

### 어느 걸 쓸지 — 70/30 규칙

- 회사가 **자주 쓰는 표준 자재** → `formKey` (전체의 약 70%)
- 일회성·특수·고객 직접 가져온 자재 → `customSpec` (약 30%)

> 💡 회사 템플릿이 잘 정리되어 있을수록 formKey 비중이 높아져 작업이 빨라집니다.

## 4.5 인라인 엑셀 편집 — 핵심 작업 방식

마감재 탭의 진짜 매력은 **엑셀처럼 빠른 인라인 편집**입니다.

### 기본 동작

1. 행을 **클릭** → 펼쳐짐 (편집 모드)
2. 필드 입력
3. **자동 저장** (1초 뒤)
4. 저장 완료 시 행이 **초록색으로 한 번 깜박임**

> 💡 **"저장" 버튼을 누르지 마세요** — 1초 디바운스로 자동 저장됩니다. 누르려고 마우스 옮길 필요 없음.

### 키보드 단축키 (필수 암기)

| 키 | 동작 |
|---|---|
| `↑` `↓` | 행 위·아래로 이동 |
| `←` `→` | 필드 왼쪽·오른쪽 이동 (입력 끝/처음일 때만) |
| `Tab` | 다음 필드로 이동 |
| `Shift+Tab` | 이전 필드로 이동 |
| `Enter` | 다음 필드 (단, textarea는 줄바꿈) |
| `Esc` | 펼친 행 닫기 |
| `F2` | 행 펼치기 (또는 클릭) |
| **마지막 필드에서 Tab** | **다음 행 자동 펼침** |

### 빠른 입력 워크플로우 — 30개 행 5분에 끝내기

```
1. 첫 행 클릭 → 펼침
2. Tab으로 필드 옮기며 입력
3. 마지막 필드에서 Tab → 다음 행 자동 펼침
4. 반복
5. 끝나면 Esc
```

마우스는 **거의 안 씁니다.**

[스크린샷 자리: 인라인 편집 — 펼친 행 + 단축키 표시]

### 자주 쓰는 패턴

#### 같은 자재 여러 spaceGroup에 동시 적용

```
거실 — 강마루 메이플 — 단가 75,000
   ↓ Ctrl+D (행 복사 — 있으면)
안방 — 강마루 메이플 — 단가 75,000  (spaceGroup만 변경)
   ↓
작은방-1 — 강마루 메이플 — 단가 75,000
```

> 💡 행 복사 단축키가 미구현일 수 있습니다. 그 경우 회사 템플릿에 미리 SET으로 등록해두는 게 더 빠릅니다.

#### CSV 업로드 (대량 입력)

마감재 탭에 **CSV 임포트** 기능이 있는 경우, 엑셀에서 미리 정리한 표를 통째로 올릴 수 있습니다.

CSV 양식:
```csv
spaceGroup,kind,itemName,unitPrice,quantity,unit,status,remarks
거실,FINISH,강마루 메이플,75000,18,m²,CONFIRMED,
주방,APPLIANCE,LG DUE6BGL2E 식세기,1432200,1,대,CONFIRMED,빌트인
...
```

## 4.6 마감재 vs 마감재 (고급) — 둘 중 어느 걸 쓰나

프로젝트 탭에는 마감재 관련 두 개 탭이 있습니다. 헷갈리시면 안 됩니다:

| | **마감재** (4번 탭) | **마감재 (고급)** (5번 탭) |
|---|---|---|
| URL | `/materials` | `/materials-advanced` |
| 컴포넌트 | ProjectMaterialsSimple.jsx | ProjectMaterials.jsx |
| 입력 방식 | **인라인 엑셀 편집** | **폼 스키마 기반 모달** |
| 변경 이력 | 기본 추적 | **상세 추적 (MaterialHistory)** |
| 속도 | 빠름 ⚡ | 중간 |
| 정확도 | 보통 | 높음 |
| 권장 사용 | **일상 업무** | 분쟁 가능성 큰 자재, 고급 사양 |

### 추천 사용법

- **일상의 99%** → 마감재 (간편) 탭. 인라인 편집으로 빠르게.
- **변경이 잦은 항목 / 고급 사양 자재** → 마감재 (고급)에서 한 번 더 정리. 변경 이력이 자세히 남음.
- 두 탭 데이터는 **같은 테이블 공유** — 어느 쪽에서 수정하든 양쪽에 반영됩니다.

## 4.7 다크모드

마감재처럼 표 데이터를 오래 보는 작업은 다크모드가 눈을 훨씬 편하게 합니다.

**좌측 사이드바 하단** 또는 **설정**에서 토글:

| 모드 | 동작 |
|---|---|
| `light` | 항상 밝은 배경 |
| `dark` | 항상 어두운 배경 |
| `system` (기본) | OS 설정 따라 자동 |

> 💡 회사 사무실이 밝은 곳이면 `light`, 야근이 잦거나 모니터가 밝으면 `dark`. 잘 모르겠으면 `system`으로 두세요.

## 4.8 변경 이력 — MaterialHistory

마감재의 모든 의미 있는 변경은 자동으로 기록됩니다:

| 기록되는 것 | 예시 |
|---|---|
| 누가 | 김미영 사원 |
| 언제 | 2026-05-13 14:30 |
| 어떤 필드 | itemName, unitPrice, status |
| 이전 값 → 새 값 | "화이트 무광" → "그레이 무광" |

**조회 방법**:
- **마감재 (고급) 탭** 안에서 행 클릭 → 변경 이력 패널
- **프로젝트 햄버거 메뉴** → "변동 로그"

이 이력은 **분쟁 시 결정적 증거**입니다. 자세한 활용은 [Playbook #8: 분쟁 증빙](playbooks/08-dispute-evidence.md) 참고.
