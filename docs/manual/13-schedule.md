# 챕터 12. 일정 관리

> 이 챕터를 읽고 나면 — 프로젝트별 공정 일정을 간트 차트로 관리하고, 회사 전체 일정 메뉴(Phase 17)에서 모든 현장의 진행 상황을 한눈에 볼 수 있게 됩니다.

---

## 12.1 일정의 두 가지 단위

Suplex의 일정 관리는 두 곳에서 일어납니다:

| 위치 | 단위 | 용도 |
|---|---|---|
| **프로젝트 → 공정 일정 탭** (3번 탭) | 한 프로젝트 안의 공정 단위 | 일별·공정별 작업 계획 |
| **상위 메뉴 → 일정** (Phase 17 신설) | 회사 전체의 모든 현장 | 회사 자원 배분, 거래처 일정 |

> ⚠️ Phase 17 일정 메뉴는 신설 예정 — v0.5 매뉴얼 시점에 구현 상태에 따라 본 챕터 보완.

---

## 12.2 프로젝트 공정 일정 입력

프로젝트 → **공정 일정** 탭 → **+ 새 공정**.

### 공정 한 행

| 필드 | 예시 |
|---|---|
| 공정명 | "철거 공사" |
| 공종 (workType) | DEMOLITION |
| 시작일 | 2026-05-15 |
| 종료일 | 2026-05-16 |
| 담당 거래처 | 협력업체 마스터에서 선택 |
| 예정 시간 | (선택) "오전 8시" 등 |
| 의존성 | 없음 (Suplex는 선후관계 강제 X) |
| 상태 | 예정 / 진행 / 완료 / 지연 |
| 비고 | 특이사항 |

### Suplex의 일정 철학

> **공정 사이에 선후관계를 강제하지 않습니다.** 작은 공사·큰 공사 모두 같은 프레임을 쓸 수 있도록.

전통적인 간트 차트 도구는 "철거 → 목공 → 도배" 같은 의존성을 강제하지만, Suplex는 그렇지 않습니다. 이유는:

- 작은 공사(부분 도배 등)는 의존성이 없거나 단순함
- 큰 공사도 현장 상황에 따라 순서 자주 바뀜
- 과도한 의존성 강제는 데이터 입력 부담만 늘림

대신 **시각적으로 시간 순서가 보이도록 간트 차트로 표현**합니다.

## 12.3 데드라인 룰

설정 → 공정 데드라인 룰에 회사 표준 룰을 등록 가능:

| 공종 | 평균 소요 기간 (30평 기준) | 데드라인 알림 |
|---|---|---|
| 철거 | 2일 | 시작 전 3일 |
| 목공 | 5일 | 시작 전 5일 |
| 도배 | 3일 | 시작 전 2일 |
| 바닥 | 2일 | 시작 전 1일 |
| ... | ... | ... |

이 룰이 등록되어 있으면 새 프로젝트 만들 때 공정 일정이 자동 시드됩니다.

---

## 12.4 회사 전체 일정 (상위 메뉴, Phase 17)

좌측 메뉴 → **일정**.

### 화면 구성 (예정)

- 월간/주간 캘린더
- 모든 현장의 공정이 한 화면에
- 거래처별 일정 (한 거래처가 여러 현장에 가는지 충돌 확인)
- 휴일·공휴일 자동 표시

### 활용

| 시나리오 | 가치 |
|---|---|
| 거래처 일정 충돌 발견 | "김씨 도배팀이 5/20 두 현장에 동시에 잡혔네" |
| 회사 자원 배분 | 다음 주 어느 공종에 인력 집중 |
| 클라이언트 응대 | "5/25에 우리 집 와주세요" 가능 여부 즉시 확인 |
