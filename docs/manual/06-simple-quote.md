# 챕터 5. 간편 견적서

> 이 챕터를 읽고 나면 — 공정별 1장짜리 간편 견적을 작성하고, 회사 양식을 적용하며, 직전 차수와 비교하여 PDF로 출력할 수 있게 됩니다.

간편 견적은 **Suplex의 가장 대표적인 산출물**입니다. 신규 도입 회사가 가장 먼저 만들어보는 것도 이것입니다.

---

## 5.1 표준 구조

간편 견적의 합계 공식은 다음과 같이 고정되어 있습니다 (Suplex 표준):

```
┌─────────────────────────────────────────────┐
│                                             │
│  공정 합계                ?,???,???   (자동) │
│  + 디자인비 (10%)         ?,???,???   (자동) │
│  + 단수조정                  -25,000  (수동) │
│                          ───────────         │
│  공급가액                ?,???,???   (자동) │
│  + 부가세 (사용자 설정%)  ?,???,???   (자동) │
│                          ───────────         │
│  ▶ 총합계               ??,???,???   (자동) │
│                                             │
└─────────────────────────────────────────────┘
```

### 각 줄의 의미

| 줄 | 무엇 | 기본값 | 변경 가능? |
|---|---|---|---|
| 공정 합계 | 견적 표의 모든 라인 금액 합 | 자동 계산 | ❌ 표 수정으로 변경 |
| 디자인비 | 공정 합계 × 비율 | **10%** | ✅ 0~30% 권장 |
| 단수조정 | 매끄러운 숫자로 만들기 위한 수동 조정 | 0 | ✅ 음수·양수 모두 가능 |
| 공급가액 | 위 셋을 합한 부가세 전 금액 | 자동 | ❌ |
| 부가세 | 공급가액 × 비율 | **0%** (별도 안내) | ✅ 0% 또는 10% |
| **총합계** | 클라이언트가 결제할 최종 금액 | 자동 | ❌ |

> 💡 **부가세 기본값이 0%인 이유**: 사업자 클라이언트면 10%로, 일반 개인 클라이언트면 "VAT 별도" 안내로 처리하시는 회사가 많기 때문. 견적별로 매번 결정하는 것이 안전합니다.

## 5.2 새 견적 만들기

프로젝트 → **견적** 탭 (1번 탭) → **+ 새 견적 만들기**.

### 빈 견적 생성

자동으로 빈 견적이 만들어지고 표가 뜹니다. 표는 다음 컬럼으로 구성:

| 컬럼 | 의미 | 입력 |
|---|---|---|
| 품명 (itemName) | 공정 또는 항목 이름 | "철거 공사", "도배 공사" 등 |
| 규격 (spec) | 사양·범위 | "30평 일괄", "실크 벽지" 등 |
| 수량 (quantity) | 숫자 | 1, 30, 50 등 |
| 단위 (unit) | m², 식, 개, 평 등 | 자유 텍스트 |
| 단가 (unitPrice) | 1단위 가격 | 원 단위 |
| 금액 (=수량×단가) | 자동 계산 | ❌ |
| 비고 (notes) | 특이사항 | 자유 텍스트 |

### 인라인 편집 (마감재 탭과 동일)

키보드 단축키도 동일:
- `↑` `↓` 행 이동
- `Tab` 다음 필드 (마지막 필드에서 Tab → 새 행 추가)
- `Esc` 닫기

### 견적 라인 템플릿 활용

회사 템플릿에 등록된 견적 라인을 재사용할 수 있습니다. **+ 템플릿에서 추가** (또는 비슷한) 버튼으로 회사가 자주 쓰는 항목을 선택하면 자재비·노무비·경비 자동 채움.

## 5.3 견적 옵션

상단 또는 측면 옵션 패널에서:

### 회사 양식 (templateKey)

견적서 PDF의 디자인을 결정합니다:

| 양식 | 특징 | 적합한 회사 |
|---|---|---|
| `classic` | 빽빽하고 정통적인 한국 견적서 스타일 | 중소·중견 인테리어 회사 |
| `modern` | 여백 넓고 깔끔한 모던 스타일 | 디자인 중시 회사, 프리미엄 |

회사 정체성에 맞는 것을 회사 기본값으로 두시면 매번 바꿀 필요 없음.

### 디자인 및 감리비 (designFeeRate)

기본 10%. **0~30%** 범위 권장.

- 단순 도배·바닥만 할 때: 5~8%
- 표준 리모델링: 10% (Suplex 기본)
- 고급 디자인 개입 큰 프로젝트: 15~20%

### 단수조정 (roundAdjustment)

매끄러운 숫자로 만들기 위한 수동 조정. 음수·양수 모두 가능.

예시:
- 자동 계산이 36,423,750원 → -23,750원으로 단수조정 → 36,400,000원

### 부가세 (vatRate)

기본 0%. 보통 0% 또는 10%.

| 부가세 | 언제 |
|---|---|
| 0% | 일반 개인 클라이언트, "VAT 별도" 안내 |
| 10% | 사업자 클라이언트, 세금계산서 발행 |
| 기타 | 면세 사업장 등 (드물게) |

### footerNotes (하단 안내문)

견적서 PDF 가장 아래 출력될 안내문. 자유 텍스트.

권장 내용:
```
※ 본 견적서는 [날짜] 현장 미팅 기준으로 작성되었습니다.
※ 가구류 및 가전 일부는 별도 구매 사항입니다.
※ 자재 등급 변경 시 단가가 변동될 수 있습니다.
※ 본 견적의 유효기간은 발행일로부터 30일입니다.
```

## 5.4 직전 차수와 비교 — AI 자동 요약

이 기능이 Suplex의 진짜 매력입니다.

### 언제 쓰나

2차 견적을 만들 때.

### 사용법

2차 견적 화면에서 **"직전 차수와 비교"** 버튼 클릭.

### 자동으로 일어나는 일

1. 1차 견적과 2차 견적을 비교
2. AI가 차이를 한국어로 자동 요약
3. 요약문이 2차 견적의 **footerNotes에 자동 추가**

자동 추가 예시:

```
※ 본 견적은 2차(변경) 견적입니다. 1차 대비 변경 사항:
  - 강마루 색상 변경: 메이플 → 오크 (전체)
  - 욕실 타일 색상 변경: 화이트 무광 → 그레이 무광
  - 거실등 펜던트 추가 (320,000원)
  - 합계 차이: +536,250원

※ 1차 견적(2026-05-10 발행)은 SUPERSEDED 처리되었습니다.
```

> 💡 클라이언트가 PDF만 받아도 어떤 변경이 있었는지 한눈에 압니다. "왜 53만 원이 늘었나요?" 같은 질문이 안 나옵니다.

자세한 변경 흐름은 [챕터 7 — 변경 관리](08-changes.md) 참고.

## 5.5 PDF 출력 (PrintModal)

### 절차

1. 견적 우측 상단 **PDF 인쇄** 버튼 클릭
2. **PrintModal** 미리보기가 뜸
3. 미리보기에서 양식·내용 확인
4. **인쇄** 버튼 → 대상 프린터 선택 → **"PDF로 저장"**
5. 파일명 입력 → 저장

### 권장 파일명 규칙

```
[프로젝트번호]_[고객명]_견적_v[차수]_[YYYY-MM-DD].pdf
```

예시:
- `강남래미안304-1502_박상철_견적_v1_2026-05-10.pdf`
- `강남래미안304-1502_박상철_견적_v2_2026-05-14.pdf`

### 발송

PDF 저장 후 **외부 채널**(카카오톡, 이메일)로 클라이언트에게 직접 발송. Suplex 안에서는 발송 기능 없음 — 클라이언트 비접근 원칙입니다.

## 5.6 상태 흐름

견적은 다음 상태를 가집니다:

| 상태 | 의미 | 다음 단계 |
|---|---|---|
| `DRAFT` | 작성 중. 아직 클라이언트에게 안 보냄 | 발송 후 → SENT |
| `SENT` | 클라이언트에게 발송 완료. 응답 대기 | 합의 후 → ACCEPTED, 변경 발생 → SUPERSEDED |
| `ACCEPTED` | 클라이언트가 합의함 | 공사 진행 |
| `SUPERSEDED` | 변경 발생으로 폐기. **새 차수가 유효** | (영구 보관) |

### 상태 전환은 수동

- 발송 직후 → 직접 **DRAFT → SENT**로 전환
- 합의 직후 → **SENT → ACCEPTED**
- 2차 견적 만들 때 → 1차를 **SENT/ACCEPTED → SUPERSEDED**

> ⚠️ **자동 전환은 권장 안 합니다.** 자동이면 분쟁 시 모호함이 생깁니다 ("Suplex가 자동으로 바꿨다"). 수동 전환 = 명확한 의사결정 시점.

## 5.7 자주 막히는 곳

**Q. 디자인비가 너무 비싸 보여요. 클라이언트가 거부감 있을 텐데요.**
A. 디자인비를 5~8%로 낮추거나, 견적 표의 첫 줄에 "디자인 및 감리 일괄"을 식 단위로 넣고 디자인비 비율은 0%로 두는 방법도 있습니다. 회사 영업 전략에 따라 결정.

**Q. 한 견적서에 두 명의 고객 (집주인+세입자) 정보를 동시에 넣고 싶어요.**
A. 프로젝트의 비고 필드 또는 견적의 footerNotes에 두 명 정보를 모두 기재. PDF 헤더는 1명까지만 가능.

**Q. 부가세 5%로 하는 건 가능한가요?**
A. 가능합니다 (소수점 입력 가능). 다만 한국 세법상 부가세는 0% 또는 10%만 정상 — 별도 사정 있을 때만 사용.

**Q. 견적 발송 후 단가만 살짝 바꾸고 싶어요. 새 견적 만들기 귀찮아요.**
A. **그래도 새 견적(2차)을 만드세요.** 1차를 수정하면 분쟁 시 증거가 망가집니다. 자세한 흐름은 [챕터 7 — 변경 관리](08-changes.md) 참고. 시간은 1차 복제 → 단가만 수정 → 2분이면 끝납니다.

**Q. PDF가 깨져 나와요 (한글 폰트 문제 등).**
A. 브라우저 인쇄 다이얼로그에서 "배경 그래픽" 옵션을 켜고, 한글 폰트가 시스템에 설치되어 있는지 확인. 권장 브라우저는 Chrome / Edge.
