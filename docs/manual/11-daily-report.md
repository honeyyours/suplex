# 챕터 10. 현장보고

> 이 챕터를 읽고 나면 — 공사 현장의 일일 진행 상황을 사진과 함께 기록하고, 클라이언트 응대·분쟁 시 증빙으로 활용할 수 있게 됩니다.

---

## 10.1 현장보고란

공사 진행 매일 또는 일정 주기로 작성하는 보고서. 사진을 첨부할 수 있는 것이 핵심.

### 왜 작성해야 하나

| 이유 | 효과 |
|---|---|
| 클라이언트 응대 | "지금 어디까지 됐나요?" 질문에 사진으로 즉시 답변 |
| 분쟁 예방 | "재시공해주세요" → 사진으로 시공 상태 증명 |
| 거래처 관리 | 작업 인수인계 사진으로 책임 명확화 |
| 회사 자산 | 향후 유사 프로젝트 견적 산출 시 참고 |

---

## 10.2 작성 절차

프로젝트 → **현장보고** 탭 (8번 탭) → **+ 새 보고**.

### 입력 항목

| 항목 | 입력 |
|---|---|
| 날짜 | 보고일 |
| 작성자 | 자동 (로그인한 직원) |
| 진행 공종 | 다중 선택 (목공·전기·타일 등) |
| 사진 | 다중 업로드 (휴대폰에서 직접 가능) |
| 현장 메모 | 자유 텍스트 (문제·이슈·잘된 점) |
| 다음 일정 | 내일·이번 주 예정 작업 |
| 클라이언트 공유 여부 | YES면 별도 PDF 생성 |

[스크린샷 자리: 현장보고 작성 화면 — 사진 업로드 UI]

---

## 10.3 사진 첨부 팁

| 팁 | 효과 |
|---|---|
| 같은 위치를 매일 같은 각도에서 촬영 | 진행 비교 가능 |
| 자재 입고 시 라벨·박스 사진 | 거래처 자재 검증 |
| 시공 직전·직후 비교 사진 | 분쟁 시 결정적 증거 |
| 문제 발생 시 즉시 사진 | "이미 그랬다"는 주장 차단 |
| 휴대폰 시간 자동 동기화 켜두기 | 메타데이터에 정확한 시간 |

---

## 10.4 클라이언트 공유

클라이언트에게 진행 상황을 공유하고 싶으면:

1. 보고 화면 우측 **"클라이언트용 PDF"** 버튼
2. 자동으로 클라이언트용 형태(매입가·내부 메모 제거)로 PDF 생성
3. 카카오톡으로 발송

> ⚠️ Suplex의 클라이언트 비접근 원칙: PDF만 발송하고, 클라이언트가 Suplex 화면에 접근하지 않습니다.

---

## 10.5 모바일 권장

현장에서 휴대폰으로 사진 찍으며 즉시 입력하는 게 가장 효율적입니다. 데스크톱 권장하지 않음.
