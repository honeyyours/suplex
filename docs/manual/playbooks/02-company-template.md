# Playbook #2 — 회사 템플릿 처음 만들기

**우리 회사 마감재·견적 라인 표준 SET을 한 번에 정리해서 평생 편한 상태 만들기**

> 이 Playbook은 Suplex 도입 첫 주에 반드시 한 번 해두셔야 하는 작업입니다. 한 번 잘 해두면 이후 모든 새 프로젝트가 자동으로 혜택을 봅니다.

소요 시간: 2~4시간 (한 번에 다 하기보다 2~3일 나눠서 하는 게 효율적)

---

## 시나리오 배경

```
회사: ABC인테리어 (직원 4명, 강남 일대 아파트 리모델링 전문)
사장님 김갑수: "Suplex 도입했으니 이제 우리 회사 표준을 정리하자.
              지금까지 직원마다 다르게 입력해서 매번 헷갈렸잖아."
견적 담당 김미영: 사장님과 함께 작업 시작
```

이 작업의 목표:
- 회사가 평소 자주 쓰는 마감재 30~50개를 **마감재 템플릿**에 등록
- 자주 쓰는 공정 단위 견적 라인 20~30개를 **견적 라인 템플릿**에 등록
- 공종 키워드 룰, 데드라인 룰 같은 회사 표준 정리

---

## 단계 ① — 회사가 평소 쓰는 마감재 정리 (30분)

### 워크시트 작성

엑셀 또는 종이에 평소 진행한 5~10개 프로젝트의 마감재를 다 모아봅니다:

| spaceGroup | itemName | kind | 회사 평균 단가 | 평소 사용 빈도 |
|---|---|---|---|---|
| 거실 | 강마루 메이플 | FINISH | 75,000/m² | 매우 자주 |
| 거실 | 강마루 오크 | FINISH | 78,000/m² | 자주 |
| 거실 | 도배 실크 화이트 | FINISH | 12,000/m² | 매우 자주 |
| 거실 | LED 다운라이트 | FINISH | 15,000/개 | 매우 자주 |
| ... | ... | ... | ... | ... |

> 💡 **사용 빈도 별로 분류**: 매우 자주 (essential = true) / 자주 (active = true) / 가끔 / 안 씀 (active = false). 너무 많이 등록하면 신규 프로젝트 시 정리가 어려워요.

## 단계 ⑤ — 회사 정보·로고·도장 등록 (15분)

**설정 → 회사 정보**.

- 회사 정식 명칭, 사업자번호, 주소, 연락처 ← 견적서 PDF에 들어감
- **회사 로고** (가로 약 200px PNG)
- **도장 이미지** (정사각형 약 80×80px, 배경 투명 PNG)

> 💡 로고와 도장은 견적서 외관의 95%를 좌우합니다. 어설픈 로고 보다 글자 로고라도 깔끔한 게 낫습니다.

---

## 단계 ⑥ — 가전 규격 마스터 시드 (선택, 30분)

**설정 → 가전 규격 마스터**.

평소 자주 쓰는 가전 모델 5~20개를 미리 등록:

| 모델 코드 | 브랜드 | 카테고리 | W | H | D | 걸레받이 |
|---|---|---|---|---|---|---|
| LG DUE6BGL2E | LG | 식기세척기 | 598 | 815 | 567 | 150 |
| LG DEE6BGE | LG | 식기세척기 | 598 | 815 | 567 | 100 |
| 삼성 DW80F75K1USWS | 삼성 | 식기세척기 | 595 | 815 | 572 | (현장) |
| LG BEY3GST | LG | 인덕션 | 760 | 60 | 510 | - |
| ... | ... | ... | ... | ... | ... | ... |

마감재 입력 시 모델 코드 입력하면 자동 채움 → 작업 시간 단축.

---

## 단계 ⑦ — 첫 새 프로젝트로 검증 (30분)

새 프로젝트 만들고 5분 안에 1차 견적이 나오는지 테스트:

1. 새 프로젝트 → 마감재 탭이 자동 시드됨 (✓)
2. 빠진 자재만 추가 입력 (5~10개)
3. 견적 탭 → + 템플릿에서 추가로 공정 라인 즉시 채움
4. 옵션 자동 채워짐 (디자인비 10%, 부가세 10% 등) (✓)
5. PDF 출력 → 회사 정보·로고·도장 정상 표시 (✓)

**여기까지 5분 안에 끝나면 성공.** 안 되면 어디서 막히는지 단계 ②~④로 돌아가 보강.

---

## 끝났습니다 — 이제 평생 편합니다

이 작업을 한 번 해두면:

| 결과 | 효과 |
|---|---|
| 새 프로젝트 만들기 | 빈 화면 → 70% 채워진 상태 시작 |
| 견적 작성 시간 | 30분 → 5분 |
| 직원 간 입력 일관성 | 매번 다름 → 회사 표준 통일 |
| 신입 사원 온보딩 | 1주 → 1일 |
| 분기·연간 분석 | 데이터 일관 → 통계 가능 |

**분기에 한 번씩 갱신**: 자재 단가 변동, 신모델 추가, 안 쓰게 된 항목 비활성화. 한 번 만들고 끝이 아니라 살아있는 회사 자산.
