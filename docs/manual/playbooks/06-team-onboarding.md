# Playbook #6 — 신입 사원 들어왔을 때

**견적 담당 신입 직원이 출근 첫날 → 1주일 안에 혼자서 견적 만들 수 있는 상태로**

> 이 Playbook을 읽고 따라하시면 — 신입 사원의 Suplex 온보딩을 1주일 안에 끝낼 수 있게 됩니다 (전통적인 1개월 → 1주).

소요 시간: 첫날 1시간 + 매일 30분 × 5일

---

## 시나리오

```
ABC인테리어, 신입 견적 담당자 박지영 (5/27 입사)
사장님 김갑수: "지영씨, 김미영 사원이 결혼 휴직이라
첫 주에 모든 견적 업무 인수받아야 해요. 매뉴얼 잘 읽고 따라해보세요."
박지영: (긴장) "네 알겠습니다."
```

---

## 출근 첫날 (1시간)

### 단계 ① — 계정 발급 (10분, OWNER가 함)

OWNER 김갑수:
1. 좌측 메뉴 → 팀관리 → + 팀원 추가
2. 박지영 입력: 이름·이메일·역할(DESIGNER)·연락처
3. 임시 비밀번호 발급됨 → 박지영에게 카카오톡 전달

### 단계 ② — 첫 로그인 + 비밀번호 변경 (10분, 박지영)

1. Suplex 사이트 → 로그인 → 임시 비번 입력
2. 자동으로 비밀번호 변경 화면
3. 본인이 기억할 수 있는 비밀번호로 변경
4. 홈 화면 진입

### 단계 ③ — Quick Start 따라하기 (40분, 박지영)

[Quick Start Guide](../00-quick-start.md)를 그대로 따라하기:
- 가상의 프로젝트 만들기
- 마감재 입력 (인라인 편집 키보드 단축키 익히기)
- 간편 견적 작성
- PDF 출력

→ 첫날 끝까지 **PDF 한 장 출력**까지 도달하면 성공.

---

## 둘째 날 (30분)

### 단계 ④ — 회사 템플릿 살펴보기 (10분)

OWNER가 신입에게 회사 템플릿 안내:
- 마감재 템플릿 30~50개 (회사 표준 자재)
- 견적 라인 템플릿 20~30개 (자주 쓰는 공정 라인)
- 견적 기본비율 (디자인비 10%, 부가세 10% 등)

> "지영씨, 우리 회사가 평소 쓰는 자재·공정이 여기 다 정리돼있어요.
>  새 프로젝트 만들면 이게 자동 복사돼서 70% 채워진 상태로 시작합니다."

### 단계 ⑤ — Playbook #1 따라하기 (20분)

[Playbook #1: 30평 아파트](01-30py-apartment-full.md) 시나리오를 가상 프로젝트로 따라하기.

→ 모든 핵심 기능을 한 번 경험.

---

## 셋째 날 (30분)

### 단계 ⑥ — 진행 중 프로젝트 둘러보기

OWNER 김갑수와 함께:
1. ACTIVE 상태 프로젝트 5개 차례로 열어보기
2. 각 프로젝트의 견적·마감재·일정 확인
3. 김미영이 작성하던 스타일 학습
4. 지난 1차/2차 견적 비교해보기 (변경 흐름 이해)

---

## 넷째 날 (30분)

### 단계 ⑦ — 변경 처리 시뮬레이션

[Playbook #3: 공사 중 변경](03-mid-project-change.md) 시나리오를 가상으로 따라하기:
- 마감재 CHANGED 상태 전환
- 1차 SUPERSEDED → 2차 견적 만들기
- 직전 차수 비교 → AI 자동 요약 확인

→ 실제 변경 요청이 들어왔을 때 당황하지 않을 준비.

---

## 다섯째 날 (30분)

### 단계 ⑧ — 실전 1건 처리 (OWNER 옆에서)

신입이 실제 클라이언트 1건의 견적을 작성. OWNER가 옆에서 봐주며:
- 마감재 정확히 입력했나
- 공정 라인 누락 없나
- 옵션 회사 표준 따랐나
- footerNotes 적절한가

PDF 발송 직전 OWNER 검토 → 발송.

→ **"혼자서 견적 1건 발송"** 성공. 온보딩 완료.

---

## 1주 차 끝 — 체크리스트

| ☐ | 항목 |
|---|---|
| ☐ | 본인 비밀번호로 로그인 가능 |
| ☐ | 새 프로젝트 만들 줄 안다 |
| ☐ | 마감재 인라인 편집 키보드 단축키 안다 (Tab/Esc/F2) |
| ☐ | 6상태 (UNDECIDED/CONFIRMED/CHANGED 등) 의미 안다 |
| ☐ | 간편 견적 작성·PDF 출력 안다 |
| ☐ | 변경 = 2차 견적 원칙 안다 |
| ☐ | 직전 차수 비교 (AI 요약) 사용해봤다 |
| ☐ | 클라이언트 비접근 원칙 이해한다 |
| ☐ | 회사 템플릿 활용한다 |
| ☐ | 분쟁 시 증빙 (MaterialHistory 등) 안다 |

10개 모두 체크되면 온보딩 성공. 미체크 항목은 해당 챕터·Playbook 다시 보기.

## 자주 막히는 곳

**Q. 신입이 매뉴얼만 읽고는 이해 못 하는 부분이 있어요.**
A. **Playbook #1**부터 따라하게 하세요. 매뉴얼은 사전(레퍼런스)에 가깝고, Playbook은 실전 시나리오라 학습 효율이 다릅니다.

**Q. 실전 첫 견적이 너무 늦어집니다.**
A. 단계 ⑧에서 OWNER가 옆에 있으면 시간이 늘어납니다. 이게 정상이에요. 5번째 견적부터 속도가 붙습니다.

**Q. 비밀번호 자주 잊어버립니다.**
A. OWNER가 비밀번호 초기화 발급해주세요 (팀관리 → 비밀번호 초기화). 신입 시기엔 자주 발생.

**Q. 신입이 회사 외부에서 작업하고 싶어합니다.**
A. Suplex는 웹 기반이라 어디서든 접속 가능합니다. 단 **공용 PC에서 로그인 후 로그아웃 필수**, 사무실 외부에서 마감재 변경은 OWNER가 알 수 있도록 메모 남기게 하세요.
