# Playbook #3 — 공사 중 클라이언트 변경 요청 처리

**욕실 타일 변경 요청을 받고 → 마감재 변경 → 2차 견적 발행 + 발주 후 변경 ⚠️ 처리까지**

> 이 Playbook을 읽고 따라하시면 — 공사 진행 중 발생하는 변경 요청을 분쟁 없이 깔끔하게 처리할 수 있게 됩니다.

소요 시간: 익숙해지면 30분 안에 처리 가능

---

## 시나리오

```
프로젝트: 강남 래미안 304-1502 (5/15 철거 시작, 6/26 입주 예정)
현재 시점: 5/22 (철거 완료, 목공 진행 중)
발주 상황: 욕실 타일 (XYZ타일에 화이트 무광 발주 완료, 5/26 입고 예정)

5/22 오후 박상철 클라이언트 카카오톡:
"미영님, 와이프가 욕실 타일을 그레이로 바꾸고 싶다고 합니다.
가능할까요?"
```

---

## 단계 ① — 거래처에 즉시 변경 가능 여부 확인 (10분)

**가장 먼저 할 일**: 마감재 수정이 아니라 **거래처에 전화**.

```
김미영: "안녕하세요 XYZ타일 김 부장님, ABC인테리어 김미영입니다.
        지난주 발주한 600×300 화이트 무광 타일 25m²,
        혹시 그레이 무광으로 변경 가능할까요?"

XYZ김부장: "5/26 입고 예정이라 출하 전이면 가능해요.
            오늘 5/22니까 4일 여유 있어요. 그레이 무광 단가가 36,000원/m²이니
            500원/m² 인상됩니다. 총 12,500원 추가."

김미영: "감사합니다. 클라이언트에게 확인 후 다시 연락드리겠습니다."
```

> 💡 **거래처 확인이 첫 번째**입니다. Suplex 작업은 그 다음. 변경 불가능한데 견적부터 만들면 헛수고.

---

## 단계 ② — 클라이언트에게 변경 가능 여부 + 비용 차이 안내 (5분)

박상철님께 카카오톡:

> 박상철님, 욕실 타일 변경 가능하다고 거래처에서 확인받았습니다.
> 화이트 무광 → 그레이 무광 변경 시 단가 차이가 있어 약 32,000원 추가됩니다 (공용+안방 욕실 합).
> 진행할까요?

박상철님: "네, 32,000원이면 진행해주세요."

이 답변(카카오톡 메시지)은 **분쟁 시 증거**입니다. 캡처하셔서 메모 탭에 저장하시거나 적어도 카카오톡에 영구 보관.

---

## 단계 ③ — 마감재 탭에서 변경 (3분)

프로젝트 → **마감재** 탭.

해당 행 펼치기:

| 변경 전 | 변경 후 |
|---|---|
| spaceGroup: 공용욕실 | spaceGroup: 공용욕실 |
| itemName: **벽 타일 — 600×300 화이트 무광** | itemName: **벽 타일 — 600×300 그레이 무광** |
| unitPrice: 35,000 | unitPrice: 36,000 |
| status: CONFIRMED | **status: CHANGED** |

같은 방식으로 **안방욕실 벽 타일도** 변경.

> 💡 **상태를 CHANGED로**: 이게 핵심입니다. 화면에서 시각적으로 강조되고, 2차 견적 만들 때 자동 인식됩니다.

### 자동으로 일어나는 일

1. **MaterialHistory 자동 기록**: 누가, 언제, 어떤 필드, 이전 값 → 새 값
2. **PurchaseOrder.materialChangedAt 갱신**: 발주 후 변경됐으니 ⚠️ 알림 표시
3. **자동 저장**: 1초 뒤

---

## 단계 ④ — ⚠️ 알림 처리 (5분)

홈 화면 또는 프로젝트 카드에 **⚠️ 발주 후 마감재 변경** 알림이 뜸.

이 알림은 "거래처에 통보 필요" 표시입니다. 이미 단계 ①에서 확인했으니:

1. 알림 클릭 → 어느 발주인지 확인
2. 발주 탭으로 이동 → 해당 발주 메모에 기록:
   ```
   2026-05-22 14:30: 화이트 → 그레이 무광 변경 협의 완료 (XYZ김부장).
   추가 비용 12,500원 (자재비), 5/26 그대로 입고 예정.
   ```
3. 발주 상태는 그대로 ORDERED 유지 (취소 아님, 사양만 변경)

> 💡 발주 표 안에는 "변경 메모"를 충분히 자세히 적어두세요. 나중에 거래처와 정산할 때 근거가 됩니다.

---

## 단계 ⑤ — 1차 견적 SUPERSEDED 처리 (1분)

견적 탭으로 이동 → 1차 견적 → 상태를 **SENT → SUPERSEDED**로 변경.

이렇게 하면 1차는 영구 보관되지만 "유효한 견적이 아님"으로 마킹됩니다.

---

## 단계 ⑥ — 2차 견적 작성 (10분)

**+ 새 견적** 또는 **1차 복제** (있다면 더 빠름).

라인은 1차와 거의 같지만 욕실 공사만 단가 조정:

| 품명 | 1차 단가 (식) | 2차 단가 (식) | 변경 사유 |
|---|---|---|---|
| 욕실 공사 (공용 + 안방) | 1,800,000 × 2 = 3,600,000 | 1,816,000 × 2 = 3,632,000 | 그레이 타일 단가 차이 |

다른 라인은 그대로. 옵션도 그대로 (디자인비 10%, 부가세 10% 등).

**자동 계산** 결과:

```
공정 합계      :  30,157,000  (+32,000)
디자인비 (10%) :   3,015,700
단수조정       :     -22,700  (총합계 0원 단위 맞춤)
공급가액       :  33,150,000
부가세 (10%)   :   3,315,000
─────────────────────────────
총합계         :  36,465,000  (1차 36,423,750 대비 +41,250)
```

---

## 단계 ⑦ — 직전 차수와 비교 (AI 자동 요약) (1분)

2차 견적 화면에서 **"직전 차수와 비교"** 클릭.

자동으로 footerNotes에 추가됨:

```
※ 본 견적은 2차(변경) 견적입니다. 1차 대비 변경 사항:
  - 욕실 타일 색상 변경: 화이트 무광 → 그레이 무광 (공용·안방 모두)
  - 합계 차이: +41,250원 (자재비 차이 +32,000원 + 부가세 +9,250원)

※ 1차 견적(2026-05-10 발행)은 SUPERSEDED 처리되었으며, 본 2차 견적이 유효합니다.
※ 변경 합의일: 2026-05-22 (카카오톡 확인)
```

---

## 단계 ⑧ — 2차 PDF 발송 (3분)

PDF 인쇄 → PDF로 저장 → 카카오톡:

> 박상철님, 욕실 타일 변경 반영해서 2차 견적서 보내드립니다.
> 1차 대비 41,250원 추가됐습니다 (자재 단가 차이 + 부가세).
> 검토 후 합의해주시면 진행하겠습니다.

박상철님: "네 확인했습니다. 진행해주세요."

→ 2차 견적 상태를 **SENT → ACCEPTED**로 전환.

---

## 끝났습니다 — 이 시나리오의 가치

이 한 번의 처리에서 다음 모두를 깔끔히 했습니다:

| 처리 | 효과 |
|---|---|
| 거래처 사전 확인 | 변경 불가능했으면 헛수고 안 함 |
| 클라이언트 합의 (카카오톡) | 분쟁 시 증거 |
| 마감재 CHANGED 상태 | 화면 강조 + 2차 견적 자동 인식 |
| MaterialHistory | 누가/언제/무엇을 자동 기록 |
| 발주 메모에 변경 내역 | 거래처 정산 근거 |
| 1차 SUPERSEDED | 1차 무력화 + 영구 보관 |
| 2차 견적 + AI 요약 | 클라이언트가 PDF만 봐도 변경 이해 |
| 2차 ACCEPTED | 의사결정 시점 명확 |

**총 소요 시간**: 약 30~40분.

---

## 자주 막히는 곳

**Q. 거래처가 변경 불가하다고 하면?**
A. 클라이언트에게 "이미 발주 완료된 자재라 변경이 어렵습니다"고 안내. 옵션 두 가지:
- (1) 그대로 화이트로 진행 (변경 포기)
- (2) 화이트 자재 그대로 받고, 그레이 추가 발주 (위약금·재발주 비용 발생, 클라이언트 부담)

**Q. 2차 견적 후 또 변경 요청이 오면?**
A. 그대로 같은 절차. 2차 SUPERSEDED, 3차 새로 만들기. **변경이 자잘하면 일주일 단위로 묶어서 한 차수**로 처리하시면 효율적.

**Q. 카카오톡 합의가 분쟁 시 증거가 되나요?**
A. 네, 한국 민사 분쟁에서 **카카오톡 메시지는 유효한 증거**입니다. 단, 캡처해서 메모 탭에 보관하시면 더욱 안전합니다 (카카오톡 계정 분실·해킹 대비).
