# Playbook #5 — 간편 견적 vs 상세 견적: 고객 유형별로 어느 걸 쓸까

**같은 프로젝트를 두 방식으로 작성해보고 어떤 차이가 있는지 비교**

> 이 Playbook을 읽고 따라하시면 — 클라이언트·공사 규모·발주처 유형에 따라 어느 견적을 쓸지 즉시 판단할 수 있게 됩니다.

소요 시간: 둘 다 작성하면 1시간

---

## 시나리오 — 같은 프로젝트, 두 클라이언트

```
프로젝트: 30평 아파트 리모델링, 공사비 약 3,600만 원

시나리오 A: 일반 가정 클라이언트 박상철 (이전 Playbook #1)
시나리오 B: 사업자 클라이언트 (주)강남상사 (사옥 사무실 리모델링)
```

같은 규모인데 클라이언트 유형이 달라 어느 견적이 적합한지 다릅니다.

---

## 시나리오 A: 일반 가정 클라이언트 — 간편 견적

### 권장 이유

| 이유 | 설명 |
|---|---|
| 클라이언트가 회계·세무 잘 모름 | 18행 원가내역서 보면 "이게 뭐죠" 질문만 늘어남 |
| 빠른 의사결정 필요 | 첫 미팅 후 1~2일 안에 결정해야 영업 종결 |
| 변경 빈번 가능성 | 변경 시 1차 → 2차 만들기가 간편하면 빠름 |
| 결제 단순 | 부가세 별도 안내 또는 10% 단순 적용 |

### 견적 모습 (간편)

```
┌──────────────────────────────────────────┐
│  강남 래미안 304-1502 견적서                │
│  발행: 2026-05-10  수신: 박상철님           │
├──────────────────────────────────────────┤
│  품명          수량   단가      금액        │
│  철거 공사      1식  2,500,000  2,500,000  │
│  목공+가벽     1식  4,800,000  4,800,000  │
│  도배 공사    132m²    25,000  3,300,000  │
│  바닥 공사     45m²    95,000  4,275,000  │
│  욕실 공사     2식  1,800,000  3,600,000  │
│  주방 공사     1식  6,800,000  6,800,000  │
│  전기 공사     1식  1,200,000  1,200,000  │
│  창호·필름     1식  1,800,000  1,800,000  │
│  베란다 확장    1식  1,500,000  1,500,000  │
│  현장 정리      1식    350,000    350,000  │
├──────────────────────────────────────────┤
│  공정 합계              30,125,000          │
│  디자인비 (10%)          3,012,500          │
│  단수조정                  -25,000          │
│  공급가액               33,112,500          │
│  부가세 (10%)            3,311,250          │
│  ─────────────                              │
│  ▶ 총합계               36,423,750          │
└──────────────────────────────────────────┘
```

10줄 견적 표 + 합계 패널 = **클라이언트가 30초 안에 이해**.

## 시나리오 B: 사업자 클라이언트 — 상세 견적

### 권장 이유

| 이유 | 설명 |
|---|---|
| 회계 부서가 정밀한 원가 분해를 요구 | 18행 원가내역서 표준 |
| 세금계산서·매입증빙 필요 | 자재비/노무비/경비 분리 명확 |
| 입찰·관급공사 가능성 | 표준 양식 필수 |
| 사장님 결재 라인 (총무 → 재무 → 대표) | 형식 갖춰진 상세 견적이 통과 잘 됨 |

## 비교표

| 항목 | 간편 견적 | 상세 견적 |
|---|---|---|
| 작성 시간 | 10분 | 30~60분 |
| PDF 분량 | 1~2 페이지 | 5~10 페이지 |
| 클라이언트 이해 | 즉시 | 회계·세무 지식 필요 |
| 변경 처리 | 빠름 (2차 만들기 5분) | 느림 (모든 라인 재검토) |
| 발주처 만족도 | 일반 가정 ✅ / 사업자 ⚠️ | 일반 가정 ⚠️ / 사업자 ✅ |
| 입찰 적합 | ❌ | ✅ 필수 |

---

## 두 가지 같이 쓰는 패턴 (큰 프로젝트)

5,000만원 이상 큰 프로젝트는 **둘 다 운용**:

```
[첫 미팅 — 5/10]
   ↓
[간편 견적 v1 발송]            ← 빠른 합의용
   ↓
[합의 — 5/13]
   ↓
[상세 견적 v1 발송]            ← 정식 계약용
   ↓
[중간 변경 — 5/22]
   ↓
[간편 견적 v2 발송]            ← 변경 빠른 합의
   ↓
[합의 — 5/23]
   ↓
[상세 견적 v2 발송]            ← 변경 정식판
```

간편으로 빠른 합의 → 상세로 형식 갖추기. 같은 데이터를 두 형식으로 표현할 뿐.

## 결정 트리

```
                    [클라이언트 유형은?]
                          /          \
                  사업자             일반 가정
                  /                    \
           [공사 규모는?]          [공사 규모는?]
            /        \              /         \
         5천만 미만  5천만+      1천만 미만  1천만+
            ↓         ↓             ↓         ↓
         간편      상세          간편       간편
                 (필수)         (단순)    (변경 잦으면 둘다)
```

## 끝났습니다

| 클라이언트 | 추천 |
|---|---|
| 일반 가정 / 부분 공사 / 빠른 의사결정 | **간편 견적만** |
| 일반 가정 / 큰 공사 / 변경 잦음 | **간편 (운영) + 상세 (계약 마무리)** |
| 사업자 / 사옥·매장 | **상세 견적 필수** |
| 사업자 / 입찰 참여 | **상세 견적 필수**, 형식 엄격 준수 |
