## 3. 페이지 한 장 표준 양식

타사 SaaS(ServiceTitan·Procore·Asana Guide·Notion Help)의 페이지 가이드 공통 패턴을 차용한 **6단 구조**. 모든 페이지가 이 양식을 따른다.

```markdown
## [페이지명]

> **이 페이지는** 한 줄 정의 — "무엇을 하는 곳"인가.

### 화면 한눈에
[전체 스크린샷 — 영역 ①~⑤ 번호 라벨이 빨간 원으로 박힌 상태]

| 번호 | 영역 | 설명 |
|---|---|---|
| ① | 헤더 | … |
| ② | 좌측 사이드 | … |
| ③ | 메인 컨텐츠 | … |
| ④ | 우측 패널 | … |
| ⑤ | 액션 바 | … |

### 이 페이지에서 할 수 있는 것
- 작업 A — 한 줄 설명
- 작업 B
- 작업 C

### 이럴 때 옵니다 (시나리오)
- 시나리오 1: 신규 견적 차수를 만들 때
- 시나리오 2: 1차 발송 후 클라이언트 변경 요청을 받았을 때

### 인접 페이지로
- → [공정 일정] — 일정에서 이 공정의 마감재를 확인할 때
- → [발주] — 마감재 확정 후 발주서를 만들 때

### 자주 묻는 질문 (선택)
- Q. … / A. …
```

**원칙:**
- 한 페이지 가이드는 **A4 인쇄 시 2장 이내**. 길어지면 시나리오로 분리 → `playbooks/`로 이동.
- 영역 라벨은 **최대 7개**. 그 이상은 인지 부하.
- "기능 나열"이 아닌 **"무엇을 위해 오는 페이지인가"**를 가장 먼저 답한다 (JTBD 캔버스 톤과 일치).

## 4. 캡처 파이프라인 (기존 인프라 확장)

### 4.1 현재 상태 — `scripts/screens/`
- Puppeteer 기반, prod 자동 로그인 → 9페이지 순회 (전역 메뉴 한정)
- 데스크톱 1440×900 / 모바일 393×852 두 모드
- 출력: `~/Desktop/suplex_screens_<mode>_<date>/01_xxx.png`

### 4.2 확장 작업 (3단계)

| 단계 | 작업 | 산출 |
|---|---|---|
| 1 | 페이지 목록 9 → 25개 확장 (프로젝트 13탭 + 라운지 글/글쓰기) | `export-mobile-screens.mjs` 페이지 배열 수정 |
| 2 | "첫 프로젝트 자동 선택" 로직을 13탭 순회로 확장 | `for tab in [...] await page.goto(/projects/:id/tab)` |
| 3 | 영역 라벨 자동 오버레이 (PNG 후처리) — 사양 JSON 기반 | 각 페이지마다 `{ "areas": [{ "x":..., "y":..., "label":"①" }] }` |

**3번 영역 라벨 자동 오버레이**는 선택사항. 처음에는 **수동 라벨링**(Figma·Skitch 등)으로 빠르게 시작하고, 페이지가 안정되면 자동화로 옮길 수 있음. 매뉴얼 v0.7 단계에서는 **수동 라벨링 우선** 권고.

### 4.3 시드 데이터
캡처 품질의 90%는 데이터에서 옴. **베타 회사 한 곳을 "캡처 전용 데모 회사"로 픽스**하고 다음을 채워둠:

- 프로젝트 3개 (진행 중·발주 후 변경·완료) — 각각 다른 상태 시연
- 마감재 spaceGroup 8개·아이템 50개 — 인라인 엑셀 보여주기
- 견적 1차 SENT + 2차 DRAFT — 차수 비교 시연
- 체크리스트 시스템 룰 시드된 상태 — D-14/D-7 노출
- 라운지 게시글 6 카테고리당 2~3개 + 인기 글 — 카드·핫포스트
- 일정 이번 주 + 다음 주 빽빽하게 — 캘린더 채우기

> ⚠️ 데모 회사 데이터에 실명·실거래처가 들어가지 않게 가상값으로 채움 — README.md 운영 원칙 3번과 일치.

## 5. 상태별 캡처 — 페이지당 평균 3장

같은 페이지라도 "빈 상태 / 데이터 있음 / 모달 열림"은 다른 화면이다. 매뉴얼에 모두 필요.

| 상태 | 언제 캡처하나 | 예시 |
|---|---|---|
| **빈 상태 (Empty)** | 새 회사·새 프로젝트 직후 | "프로젝트가 없습니다 — 첫 프로젝트 만들기" CTA |
| **메인 상태 (Main)** | 데이터가 충분히 채워진 상태 | 마감재 50줄·견적 차수 2개 등 |
| **상호작용 상태 (Modal/Drawer)** | 주요 모달이 열린 순간 | 견적 차수 비교·자재 요청·인건비 정산 시트 |

페이지당 평균 **2~3장**, 총 약 **60~75장** 예상.
