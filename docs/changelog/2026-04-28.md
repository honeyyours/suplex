2. **로그인 자체가 막힘**. `/auth/login`의 `user.findUnique include: { memberships: { include: { company: true } } }`가 회사 모든 컬럼 SELECT 시도 → 같은 P2022.

### 14-2. 회복 (3단계 push)

**`326f12f` fix(phases): graceful fallback**

**`f5e7376` fix(auth+company)**

**`aaaf27c` chore(server)**: 임시 마이그레이션 자동화
- server/package.json의 start를 `prisma migrate deploy && node ...` → `prisma db push --skip-generate && node ...`로 변경.

- db push는 idempotent라 다음 재배포에도 안전. 정식 마이그레이션 패턴 도입 시 원복 예정.

### 14-3. 영구 규칙 추가
