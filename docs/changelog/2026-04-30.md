## 2. JTBD 캔버스 P1 작성 (`docs/sales/JTBD캔버스_P1.md`)

**페르소나**: 직원 3~10명 인테리어 회사 사장. 4~10개 현장 동시 운영. 95% 엑셀+카톡+종이노트.

**Pain 8개**:
1. 프로젝트 피드백·노하우 휘발 — "어디 남겼는지 까먹어서"
2. 외부 = 사무실 전화 의존 — "마감재 뭐였지? 타일 언제? 마감재 확정?"
3. 일정·메모 위치 망각
4. 발주 데드라인 망실 — "오늘까지 발주했어야 했는데"
5. 작업자·자재상 섭외 십수번 반복 타이핑
6. 직원 머리속 노하우 증발
7. 변경 1건당 3~5곳 동기화
8. 모바일 가독성 X

**도입 망설임 → Risk Reversal**:
- "엑셀 익숙해서" → 카톡·엑셀 그대로 + 그 사이 흩어진 정보만 모아줌
- "변화 귀찮아서" → 가입 첫날 30분 셋업 (시스템 프리셋)
- "데이터 이전 귀찮아서" → 옛날 프로젝트 안 옮겨도 됨, 다음 신규부터
- 잠재 비용·직원 따라옴 거부감 등

## 3. 헤드라인 표준 3종 (모든 자료 일관)

| 자리 | 워딩 |
|---|---|
| **메인 (표지·홈페이지·원페이저 Hero)** | 한 사람의 능력이 아니라, 회사 전체의 능력이 올라갑니다. |
| **서브** | 효율로 더 많은 매출을 만드세요. 같은 구성원으로 프로젝트 운영 효율을 극대화하고, 담당자가 바뀌어도 업무의 본질과 노하우는 회사의 자산으로 남습니다. |
| **Problem 슬라이드** | 현장에서 사무실에 전화하지 마세요. 마감재·일정·발주, 까먹지도 잃지도 않습니다. |

## 4. 폐기된 표현 (영구 금지)

봉기님 톤 시그널: **결핍 프레이밍 X / 확장·성장 프레이밍 ✓**.

| 폐기 | 대체 |
|---|---|
| 신입도 사장처럼 | **회사 전 직원의 능력이 같이 올라감** |
| 직원이 떠나도 / 그만두면 | **담당자가 바뀌어도** |
| 정보가 새다 | **기록이 한곳에 정렬되지 않으면 놓침** |
| 현장을 굴리다 / 같은 인력으로 | **프로젝트 운영** / 구성원 |
| 엑셀 95% (직접 명명) | 우리만 알고 활용 |
| 매출 늘었는데 사장 시간 줄어듦 | (결핍 프레이밍 폐기) |

## 7. 제외 확정 (소개서·원페이저에서 빠짐)

코드는 살아있되 사용 계획 없는 메뉴 — 영업 자료에서 제거:

- **상세 견적 탭** (`quotes-detail`) — 라우팅만 남고 ProjectDetail 탭에서 제외
- **마감재 고급 페이지** (`/materials-advanced`) — App.jsx 라우트 미등록
- **AI 회계·자동분류 도구** (F.AI_ADVISOR / F.AI_BILLING_TOOLS) — 보류
- **2차 견적 자동 비교** — 차별점 약함
- **지출관리 별도 컷** — 현재 큰 기능 X (여름 이후 통장 자동연동 시 다시 어필)
- **AI비서 별도 컷** — 조회 도구 10종만 살아있음, 우선순위 낮음
