# 2026-05-15 — UI 다듬기·견적 그룹 합계·라운지 라벨 단축

오늘은 ★ 새 기능 3개(견적 그룹 합계, 견적 비고 키보드, 모바일 프로젝트 필터 시트)와 다수의 UI 다듬기. 인테리어 현장 운영자가 모바일에서 들고 다니며 쓰는 경험을 정렬하는 데 집중.

## 1. ★ 새 기능

### 1-A. 견적 — 그룹별 합계 표시
- `linesWithMeta`에 `_groupTotal`(그룹 내 라인 `quantity*unitPrice` 합) 부착
- **그룹 헤더 우측**: `합계 N원` 작은 배지 (총합 > 0 일 때만)
- **그룹 종료 마커**: 기존 `↩ 그룹 종료` → `↩ 그룹 합계 N원`으로 교체
- 그룹 아래 항목 추가/수정·수량·단가 변경 시 자동 재계산
- 종료 마커가 없는 그룹도 헤더에선 합계 노출

### 1-B. 견적 — 비고 셀 Enter 동작 분리
- **Enter** → 다음 행 같은 컬럼으로 이동 (마지막이면 새 행 추가) — 기존
- **Ctrl/⌘ + Enter** → 셀 안에서 줄바꿈 (네이티브 textarea 동작 허용)
- placeholder "(Ctrl+Enter 줄바꿈)"으로 안내 변경

### 1-C. 프로젝트 모바일 — 진행중 상단 영역 압축 (A안)
- 🧪 샘플 시연용 생성 버튼 모바일 hide (sm+에서만)
- 모바일 1행 = `[🔍 검색] + [⚙️ 필터]`
- ⚙️ 필터 탭 → 바텀시트로 연도·상태·정렬 한 번에 (초기화/적용 버튼)
- 디폴트(올해/전체/최근순)와 다르면 ● 점 표시
- 데스크탑은 기존 필터 카드 그대로

## 오후 세션 — 홈 재설계 + 체크리스트 분류 + 모바일 네비 + 그림 이모지 일괄 제거

오전 작업에 이어, 디자이너 시안 검토 결과를 결정론 규칙으로 옮기고, 모바일 메인 메뉴를 앱 스타일 하단 네비게이션 바로 이동. 체크리스트에 분류(team) 필드를 신설해 홈 카드와 연결. 핵심 화면 전반에서 그림 이모지를 텍스트·SVG 라인 아이콘으로 일괄 교체.

### 8. ★ 새 기능 — 홈 "3일 안에 할 일" 카드 (결정론, 역할 기반)

홈 최상단(인사말 바로 다음)에 액션 카드 추가. **AI 브랜딩은 일체 사용하지 않음** — 메모리 정책 "AI 어드바이스 보류" 유지(정식 출시 시점에 sparkle·"AI 비서가 정리" 카피만 옵션 부착 가능).

**백엔드 `GET /api/dashboard/today-actions`**
- 윈도우 `[today-∞, today+3]` — 지남 항목 계속 노출(긴급도 최상)
- 항목 종류:
  - `DailyScheduleEntry`(category 있음, 3일내) → team `FIELD`
  - `PurchaseOrder` PENDING(지남·3일내) → team `BOTH`
  - `Material` `UNDECIDED`/`REVIEWING`(진행중·시공 14일 내 프로젝트만) → team `DESIGN`
  - `ProjectChecklist`(3일내, 미완료, team≠OTHER) — `checklist.team` 직접 사용
- 응답 `{ level, dayLabel, kind, title, meta, href, team }`

**역할 노출 라우팅 (`canSeeTeam`)**
- `OWNER`: 전부
- `FIELD`: team ∈ {FIELD, ORDER, BOTH}
- `DESIGNER`: team ∈ {DESIGN, ORDER, BOTH}
- `OTHER` 팀은 누구에게도 노출 X

**프론트 `HomeTodayActions`**
- 4개 노출 + "더 보기" 토글
- 4단계 컬러 태그: 지남(rose)·오늘(amber)·D-N(orange)·검토(sky)
- 빈 상태 "3일 안에 챙길 일이 없어요"
- kind별 SVG 아이콘(schedule·order·material·checklist·project)

### 9. ★ 새 기능 — 체크리스트 분류(team) 필드

`ProjectChecklist`에 `team` 컬럼 + `ChecklistTeam` enum 신설.

```prisma
enum ChecklistTeam { FIELD | DESIGN | ORDER | OTHER }
team  ChecklistTeam  @default(OTHER)
```

**자동 추론 `services/checklistTeam.inferChecklistTeam`**
- `phase='발주'` 또는 title에 `발주|자재|주문|매입|배송|입고` → `ORDER`
- `phase='마감재'` 또는 title에 `마감재|샘플|모델|시안|컬러칩|선정|확정` → `DESIGN`
- `phase`가 표준 25공정 → `FIELD`
- 그 외 → `OTHER`

**적용**
- POST/PATCH `/api/projects/:id/checklists` — team 명시 가능, 미지정 시 추론
- 시스템 룰 자동 시드(`systemAdvicesSync`)도 추론 결과 반영
- 대시보드는 `checklist.team` 직접 사용(regex 매칭 제거)

**UI**
- 체크리스트 추가 폼 "분류" 셀렉트(기본 "자동") — 자동/현장/디자인/발주(양쪽)/기타(홈 미노출)
- 항목에 분류 chip 표시(현장 sky · 디자인 violet · 발주 amber)

**기존 행** 기본값 `OTHER` → 홈 카드 미노출 (옵션 a). 사용자 편집 시 활성화. 일괄 백필은 보류.

### 10. ★ 새 기능 — 모바일 하단 네비게이션 바 (앱 스타일)

상단 헤더 안 가로 스크롤 메뉴를 페이지 하단 fixed nav로 이전. 데스크톱(sm+) 헤더 메뉴는 변경 없음.

- **4개 primary 탭**: 홈·일정·프로젝트·발주 (등분 그리드)
- **더보기 바텀시트**: 지출·AI비서·팀·라운지·설정 (슬라이드업·딤·grabber·4열 아이콘)
- 단색 라인 SVG 아이콘(currentColor) + 라벨 세로 스택
- 배경 `bg-navy-800`, 비활성 `text-navy-200` / 활성 `text-white` — 상단 헤더와 톤 통일

### 12. 체크리스트 기한 배지 — D-N 방식

`📅 12/17` 배지의 이모지 글리프 안 "17" 숫자가 실제 날짜와 충돌. 다음 형식으로 교체:

- 지남: `3일 지남 · 12/17` (rose)
- 오늘 마감: `오늘 마감 · 12/17` (rose, bold)
- D-1~D-7: `D-3 · 12/17` (amber, medium)
- 그 외: `D-N · M/D` (navy)
- 완료된 항목: `마감 12/17` (회색, D-N 생략)

### 13. 그림 이모지 일괄 제거

홈, 발주(전역/프로젝트), 지출관리, 팀관리, 프로젝트 햄버거, 체크리스트 — 콘텐츠 표시용 그림 이모지를 텍스트·SVG 라인 아이콘으로 교체. ✓/✕/‹/› 같은 텍스트 기호와 ✓(확정 체크)는 UI 컨트롤로 유지.

- 카드/탭/배지/버튼에서 🏗️·📝·📅·🔥·📦·✅·🏷️·🔗·🏠·💰·👥·✉️·📋·📨·📥·📤·✏️·🗑️ 전수 제거
- `PO_STATUS_META.icon` 필드 자체 제거 → 의존 라벨 정리(Orders·ProjectExpenses·Admin·HomeActivityFeed)
- 체크리스트 입력 폼·섹션 헤더·항목 배지 이모지도 제거(📅 기한·📌 일정 연결·📷 사진 첨부 필수·⬜·📅·✅)

### 34. 부수 발견 — "이름만 백업"이었던 4가지 사고

검증 사이클에서 드러난 그동안 작동 안 했던 코드들 (모두 같이 수정):

**(1) Vercel rewrite 함정** — `client/vercel.json` 의 `/(.*) → /index.html` 이 `/api/...` 도 SPA 로 잡음. `downloadFrom()` 이 axios 안 거치고 fetch 로 상대 경로 호출했더니 다운로드 폴더에 `.html` 떨어진 사고. `axios.defaults.baseURL` 재사용 + Content-Type 검증으로 수정. 어드민 백업도 동일 버그 함께 수정.

**(2) Prisma `quotes.lineItems` 명명 오류** — 실제 관계는 `Quote.lines`. `admin.routes.js` 어드민 백업·`fullCompanyExport.js` 양쪽에서 같은 오류. 즉 **어드민 백업이 그동안 한 번도 실제 작동한 적 없었음** 확인.

**(3) Prisma `Project.memos` 명명 오류** — 실제 관계는 `Project.projectMemos`. `admin.routes.js` · `fullCompanyExport.js` · `backup.routes.js`(프로젝트/회사 백업) 4곳 모두 동일 버그. 사용자 백업도 그동안 메모 못 가져왔음 확인.

**(4) Content-Disposition 헤더 한글 에러** — `ERR_INVALID_CHAR`. HTTP 헤더는 ASCII 만 허용해 한글 회사명("리플레이스 디자인") 직접 박으면 깨짐. RFC 5987 `filename="ASCII"; filename*=UTF-8''<encoded>` 패턴으로 수정 (`companyAssets.routes.js` + `admin.routes.js` 둘 다).

**즉시 안전화**: BackupMenu OWNER 한정 + 복원 임시 비활성 (현재 일정·체크리스트만 복원이라 데이터 손실 위험) + `backup.routes.js` 에 `requireRole('OWNER')` 가드. 이전엔 `authRequired` 만이라 FIELD 까지 회사 데이터 export 가능했던 보안 이슈도 같이 잡힘.

## 37. 데이터 입출력 — 회사 OWNER 권한 회수 → 슈퍼어드민 전용 통로화

회사가 자기 데이터를 마음대로 빼가거나 외부 JSON 을 막 주입하는 통로를 닫고, 봉기님(어드민)에게 카톡·메일로 요청 → 어드민 콘솔에서 처리하는 운영 모델로 전환.

### 37-A. 서버 라우트 이전 + 슈퍼어드민 가드
- `/api/company-assets/*` (4종) 폐기 → `/api/admin/companies/:companyId/data-transfer/*` 로 이전
- `requireRole('OWNER')` → `requireSuperAdmin` 으로 가드 교체
- 대상 회사 검증 미들웨어(`resolveTargetCompany`) 추가 — `:companyId` path 파라미터로 받아 존재 검증
- `import-full` 은 대상 회사의 OWNER user id 자동 픽업(`pickOwnerUserId`) — 기존 `req.user.id`(어드민) 매핑 불가
- audit action 명도 `company-assets.*` → `admin.company-data.*` 로 일관화 + `companyId: 대상회사` 명시(어드민 본인 컬럼으로 새지 않게)
- routes/index.js 등록 순서: `/admin/companies/:companyId/data-transfer` 를 `/admin` 보다 먼저 마운트(더 구체적 경로 우선)

### 37-B. 클라이언트 OWNER 진입점 제거
