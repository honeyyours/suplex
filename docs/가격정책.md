**Wedge·Lock·Moat 전략 정합** (2026-05-17, v8 정교화):
- **FREE** = wedge — **일정 모듈 완전 풀공개**(단일·통합 캘린더·일괄 복사·확정·자동 인식 전부) + 시공팀 무제한 연결 + 동시 활성 프로젝트 5건. 진입 마찰 0
- **STARTER → PRO → ENT** = lock — 회사 자산 누적·자동화·표준 코드화 사다리
- **CREW** = moat — 시공팀 양면 플랫폼, 풀 무료(paying side 아님), 다중 회사 일정 통합으로 네트워크 효과

**트리거 직렬화 (v8 신설)** — 페르소나 분리·매출 안정성:
- **FREE → STARTER**: 6번째 활성 프로젝트 시도 시 데이터 보존 마찰(loss aversion)로 자연 전환. 회사 성장 = 결제 시점.
- **STARTER → PRO**: "견적·자재·정산까지 한 화면" 욕망. 운영 통합·자동 피드백 사이클이 차별 가치.
- **PRO → ENT**: 회사 표준 코드화 — 시스템 프리셋 자유 편집 + AI 어드바이스 + 알림톡·OCR.
- **이중 트리거의 의도**: 일정만 깊이 굴리고 싶은 회사(STARTER) + 운영 전체 통합하고 싶은 회사(PRO) 두 페르소나 모두 결제로 안착. 한 트리거에만 의존 X.

## 3. 등급별 기능

### 3-0. FREE (신설, 2026-05-17 / v8 일정 풀공개 정합) — 일정 wedge
- **일정 모듈 완전 풀공개 (v8)** — 단일 프로젝트 일정 + **회사 전체 통합 캘린더** + 일정 픽스(계획/확정) + **일정 일괄 복사**(프로젝트 간 다건) + 자동 인식 키워드 + 공정 칩 + 모바일 시트 + D-N 표시. 일정 모듈은 단 하나도 PRO에 가두지 않음
- **수동 체크리스트** — 자유 항목·완료 체크·**공정 태그** + **수동 카톡 복사** (작업자 전달)
- **시공팀(CrewAccount) 연결 무제한** — 시공자 메인화면 노출 가능 (양면 플랫폼 진입)
- **동시 활성 프로젝트 5건** (v8 명확화) — 프로젝트 등록 자체는 무제한 누적 보존, 단 동시 진행 중(`PLANNED`/`IN_PROGRESS`/`ON_HOLD`)이 5건. 완료·취소 처리 시 카운트에서 제외되어 자유롭게 다음 활성 진입 가능
- 견적·마감재·발주·정산·메모는 **읽기만 가능** (작성 X — 데이터 누적 가치 보호)
- 시스템 룰 D-7 미확정 알림 1종 (맛보기)
- 출력물에 **수플렉스 워터마크** (PDF + 카톡 복사 텍스트)
- **카톡 복사 자동 푸터** — 일정·체크리스트·인건비 정산·발주서를 카톡 복사 시 끝에 `📋 수플렉스로 정리 — suplex.kr` 자동 추가. **바이럴 진입로** (Dropbox·Calendly 패턴). STARTER+ 업그레이드 시 자동 제거
- 라운지 ❌ (STARTER+ 활성)

### 3-1. STARTER (v8 재정의) — 일정 강화 트랙
**포지셔닝**: "일정만 깊이 굴리고 싶은 회사"가 안착할 등급. 견적은 엑셀로 계속 쓰지만 일정·체크리스트·시공팀 운영을 도구로 묶고 싶은 페르소나.

**v8 핵심 변경**:
- **동시 활성 프로젝트 무제한** ★ (v7 10건 → v8 무제한). 영업 매력 강화
- 작성 권한(견적·자재·발주·정산)은 **여전히 차단** (PRO 트리거 보존)

**일정 고급 기능 (FREE 대비 추가)**:
- **일정 일괄 복사 다건 + 양식 자동 출고** (v7 기준 STARTER 명시 보강 필요 항목 — 추후 논의)
- **카톡 양식 자동 출고** (체크리스트·정산·발주서 카톡 양식 자동) — 추후 논의
- **워터마크 제거** (PDF + 카톡 복사 텍스트)
- ※ STARTER 일정 강화 기능 세부 추가 항목은 **v8.x에서 보강 예정** (봉기님 2026-05-18 확정)

**체크리스트·운영 자동화 (v7에서 유지)**:
- **시스템 프리셋 자동 적용 + 활성/비활성 토글**

### 7-bis. 구독 만료 후 정책 상세 (v6 확정)

**락인 전략**: 데이터 인질형 lock-in 폐기 → **운영 락인 (data portability lock-in)** 채택. 사용자는 데이터 언제든 열람 가능, 단 다른 도구로의 일괄 이동(텍스트 portability)은 차단.

**작동 방식**:
1. 구독 만료 즉시 → 모든 추가/수정/삭제 버튼 비활성. 텍스트 데이터(메모·견적·일정·체크리스트·연락처·메모 등) 영구 화면 열람.
2. 만료 후 6개월 → 사진·첨부 자동 삭제 (Cloudinary 비용 동인 차단). 텍스트는 영구 보관.
3. 만료 후에도 사진 ZIP 부분 보관 항상 가능 — 사진 자가 백업 안전장치. 6개월 자동 삭제 전 사용자가 직접 받아갈 수 있음.
4. 재구독 즉시 정상 모드 복귀. 사진 만료 전이면 자동 복원.

**JSON 텍스트 데이터 export 영구 차단**: 모든 등급·구독 상태와 무관하게 텍스트 일괄 export 불가. 사진은 ZIP 부분 보관으로만 외부 이전 가능. 텍스트를 옮기려면 화면 보고 수동 베끼기만 가능 — 5년치 누적 시 사실상 불가능 → 운영 락인 효과.

**핵심 인사이트 — 접근(access) ≠ 이동(portability)**:
- 사용자: "내 데이터 언제든 볼 수 있다" → 신뢰·평판 보호
- 우리: "옮길 수가 없으니 결국 안 떠남" → 매출·LTV 보호
- 기존 SaaS의 hard lock-in(인질형)은 사용자가 "갇혔다"고 느껴 분노 → 평판 깎임. 우리는 "자유롭다"고 느끼게 하면서 실제로 못 떠나게 하는 구조.

**윤리적 정당성·법적 안전**:
- 데이터 자체 차단 X (열람은 무기한)
- 사진 셀프 백업 안전장치 보장
- 가입 약관에 명확 사전 고지 + 동의 필수 ("텍스트 일괄 export 영구 불가")
- 한국 개인정보보호법상 "내 데이터 영구 삭제 요청권"은 별도 보장 (Suplex 보유분 영구 파기, 사용자 자체 보관본은 사용자 책임)

## 10. 정식 출시 후 검토 사항 (의견 — v5 신규)

> v5에서 추가된 **미확정 정책 검토안**. 베타 기간에는 적용 X. 정식 출시 후 실사용 데이터 확보 후 v6에서 확정 또는 폐기.

### 10.1 베타 가치 가시화 (도입 권장)

베타는 무료지만 **사용자가 어느 정도 가치를 받는 중인지 명시**해야 함 — 정식 출시 시 결제 결정의 정보 비대칭 방지.

**반영안 (in-product UI)**:
- 설정 페이지에 "사용량 패널" 추가:
  - 현재 활성 프로젝트 수 (등급 캡 대비)
  - 현재 저장 사용량 (GB)
  - v4 가격표 기준 환산 안내: "지금 사용 패턴 = 프로 등급 / 정식 출시 시 ₩99,000"
- 오버리지 단가는 미확정 → 환산에 표시 X (추측 가격 노출 금지)
- 표시 톤: 압박이 아닌 **"체감 가치 안내"** — 베타 종료 시 사용자가 결제 결정을 정보 기반으로 할 수 있게

### 10.2 장기 누적 데이터 관리 (부수 검토)

> **v6 업데이트**: 핵심 정책(JSON export 영구 차단, 사진 6개월 만료, 운영 락인)은 섹션 7·7-bis·9로 이동·확정. 사진 부분 보관 기능도 2026-04-30 구현 완료. 본 섹션은 12개월 미접근 자동 알림·콜드 스토리지 등 부수 검토안만 남김.

**12개월 미접근 자동 알림 (UX 제안)**:
- 12개월 이상 미접근 프로젝트 자동 알림: "사진 N장 / X GB — 외부 백업 후 삭제 권장"
- 헤비 유저(100GB+) 회사가 자발적으로 사진 부분 보관에 진입하도록 유도하는 핵심 동선
- 수동으로 한 건씩 처리하게 두면 사용자가 절대 안 함

**콜드 스토리지 (장기 검토, 자동 절감)**:
사용자가 직접 안 옮겨도 자동 절감되는 옵션 — 1년+ 미접근 프로젝트의 사진을 콜드 티어로 자동 이전 (조회 시 수 초 지연, 단가 1/5 수준). 정식 출시 후 실사용 분포 보고 결정.

### 10.3 사용량 기반 과금 (의견, 정식 출시 후 결정)

**배경**: 저장 비용은 사용량에 선형 증가하나 매출은 등급별 평탄(flat) → 헤비 유저가 라이트 유저에 보조받는 구조. 인테리어 업계는 업체당 프로젝트·사진·영수증 누적 편차가 매우 큼. "일 많이 하는 업체가 더 낸다"가 SaaS 공정성·지속성 측면에서 자연스러움.

**2가지 레버**:

| 레버 | 제안 | 우선순위 |
|---|---|---|
| **저장 오버리지** | 기본 용량 초과 시 add-on (예: 프로 50GB 초과분 ₩X / 10GB·월, **단가 미정**) | 1순위 |
| **프로젝트 수 캡** | 프로 = 무제한 → 30건으로 조정 (엔터프라이즈만 무제한) | 보류 |

**저장 오버리지를 1순위로 선택한 이유**:
- "기본료 ₩99,000 그대로" — 영업 라인 보존
- 한국 SaaS에 익숙한 패턴 (NHN·네이버클라우드 등)
- "쓴 만큼" 납득 쉬움 — 실제 비용과 직결되어 분쟁 적음
- 헤비 유저는 자연히 사진을 많이 쌓으므로 "일 많이 하는 업체가 더 낸다"가 자동 달성

**프로젝트 수 캡 보류 이유**:
- "무제한"이 프로 등급의 강한 셀링 포인트 — 잃으면 엔터프라이즈가 그 차액 가치를 정말 입증해야 함
- 저장 오버리지만 도입해도 헤비 유저는 자동으로 더 부담 (동일 효과)
- 단순함 유지 — 한 번에 한 가지 변수만 조정

**단가 결정 시점**:
- 베타 기간에는 도입 X (가격 변동 없음 — 약속 신뢰 유지)
- 정식 출시 후 6~12개월 실사용 분포 측정 → v6에서 단가 확정
- 지금 추측으로 정하면 너무 비싸거나 너무 싸게 잡힐 위험 → 데이터 기반 결정 원칙

**베타 사용자 보호**:
오버리지 도입 시 베타 회원도 **30일 사전 통보 + 평생 20% 할인이 오버리지 단가에도 동일 적용**.

```
  ① 현재 등급 유지 (FREE)
     ↓ 클릭 시 기존 5개 프로젝트 리스트 즉시 표시
     ↓ "5개 중 하나를 완료 처리하거나 삭제하셔야 합니다"
     ↓ 사용자가 직접 선택해서 마무리 → 새 프로젝트 활성화 가능
     ↓ 마음의 부담(loss aversion) 발동 시점
  
  ② STARTER로 업그레이드 (프로젝트 무제한 + 일정 고급 기능)
     → 월 5만 9천원, 베타 평생 20% 할인 4만 7천원
     → "데이터 정리 안 하셔도 됩니다"

[취소]  [① 기존 정리]  [② STARTER 보기]
```

**의도**:
- ①번 선택지를 클릭하면 **기존 5개 프로젝트 리스트를 바로 띄움**. 사용자가 "이거 마음 아프다, 어떤 걸 날리지?" 감정 직면 → 자연스럽게 ② 결제 결정으로 회귀.
- 강제 모달 X (사용자 자율). 단 손실 직시화로 결제 동기 극대화.
- ②번 항상 명확히 노출 — 위 모달, 프로젝트 리스트 상단·하단 모두에 STARTER 제안 카드 고정.

**메커니즘**:
- 프로젝트 상태(`status`)가 `PLANNED`/`IN_PROGRESS`/`ON_HOLD` 중 하나로 5건 도달 시 신규 생성 게이트 발동.
- 상태 변경(`COMPLETED`/`CANCELLED`)으로 한 건 빠지면 즉시 신규 생성 가능 — 강제력 부드러움.
- 완료·취소된 프로젝트는 영구 데이터 보존 + 열람 가능 (운영 락인 정책 정합) — "삭제"는 실제 데이터 영구 파기가 아니라 활성 상태 해제로 안내.

### 패턴 2: STARTER → PRO 트리거 (견적·자재·정산 작성 시도)

```
[견적 작성 시도]
─────────────────────────────────────────────
견적·자재·발주·정산 작성은 PRO 등급 기능입니다.

선택지:
  ① 지금 일정·체크리스트만 더 잘 활용 (현재 등급 유지)
     → STARTER에서 일정·체크리스트·시공팀 운영 깊이 활용 안내
  
  ② PRO로 업그레이드 (견적·자재·정산 + 자동 피드백 사이클)
     → 월 9만 9천원, 베타 평생 20% 할인 7만 9천원
     → "공정별 견적 가이드가 다음 견적에 자동으로 채워지는 사이클"

[취소]  [② PRO 보기]
```

**의도**: 견적 자체는 강제 X (엑셀로 계속 쓸 수 있는 회사 보호). 단 PRO에서 작동하는 **자동 피드백 사이클** — 봉기님 차별화 코어 — 을 명확히 미리보기로 전달해 욕망 형성.

### 패턴 3: PRO → ENTERPRISE 트리거

ENT 트리거는 회사가 자체 시공 표준을 시스템에 새기고 싶을 때 발생. 모달 패턴은 위와 동일하되, **"시스템 프리셋 자유 편집·AI 어드바이스·알림톡·OCR" 4종 묶음** 미리보기로 정의.

### 운영 규칙
- 모달은 행동 시점에만 표시 — 페이지 진입·헤더 등 무관한 자리에 띄우지 않음 (영업 피로 회피).
- ① 선택지(현재 등급 유지)는 항상 가능해야 함 — 강제 결제 X.
- ② 선택지는 **베타 할인가 항상 명시** — 정가만 노출하면 영업 약함.

## 4-ter. CREW (시공팀) 무료 트랙 (v7 신설)

**위치**: 인테리어 회사 paying side와 분리된 별도 계정 타입. 자세히 → [[수플렉스_설계_시공팀계정]]

**가격**: 영원 무료 (paying side 아님, 빠른 확산이 핵심)

**제공 기능**:
- 다중 인테리어 회사가 잡은 일정 통합 캘린더 (회사별 색상 분리)
- 더블 부킹 자동 알림
- 인테리어 회사가 추가 요청한 체크리스트 메인화면 자동 노출 (예: "거실 콘센트 2개 추가" 같은 다음 방문 시 처리 사항)
- 자기에게 온 발주서 모음
- 정산 영수증 보관

**제공 안 하는 기능** (격리):
- 인테리어 회사의 견적·마감재·정산 본문 데이터 접근 X
- 다른 회사 일정·고객 정보 노출 X (자기에게 잡힌 일정만)

**왜 무료인가**:
- 시공팀은 paying side가 아님 — 결제 마찰이 곧 확산 저해
- 양면 락인 효과 = 시공팀 1팀이 거래 인테리어 회사 5개를 묶어버림
- 운영 비용은 인테리어 회사 PRO/ENT 매출로 cover
- 정식 출시 후 자재 광고 검토 가능 (별도 매출 라인)

**영업 메시지**:
> "시공팀과 사무실이 한 화면에서 일정을 맞추는 도구."
> "거래하는 시공팀 5팀 일정을 카톡 스크롤 없이 한 화면에서."
> "김목수가 다른 일정 잡으면 우리 회사가 즉시 압니다."
