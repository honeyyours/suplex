### 7-bis. 구독 만료 후 정책 상세 (v6 확정)

**락인 전략**: 데이터 인질형 lock-in 폐기 → **운영 락인 (data portability lock-in)** 채택. 사용자는 데이터 언제든 열람 가능, 단 다른 도구로의 일괄 이동(텍스트 portability)은 차단.

**작동 방식**:
1. 구독 만료 즉시 → 모든 추가/수정/삭제 버튼 비활성. 텍스트 데이터(메모·견적·일정·체크리스트·연락처·메모 등) 영구 화면 열람.
2. 만료 후 6개월 → 사진·첨부 자동 삭제 (Cloudinary 비용 동인 차단). 텍스트는 영구 보관.
3. 만료 후에도 사진 ZIP 부분 보관 항상 가능 — 사진 자가 백업 안전장치. 6개월 자동 삭제 전 사용자가 직접 받아갈 수 있음.
4. 재구독 즉시 정상 모드 복귀. 사진 만료 전이면 자동 복원.

**JSON 텍스트 데이터 export 영구 차단**: 모든 등급·구독 상태와 무관하게 텍스트 일괄 export 불가. 사진은 ZIP 부분 보관으로만 외부 이전 가능. 텍스트를 옮기려면 화면 보고 수동 베끼기만 가능 — 5년치 누적 시 사실상 불가능 → 운영 락인 효과.

**핵심 인사이트 — 접근(access) ≠ 이동(portability)**:
- 사용자: "내 데이터 언제든 볼 수 있다" → 신뢰·평판 보호
- 우리: "옮길 수가 없으니 결국 안 떠남" → 매출·LTV 보호
- 기존 SaaS의 hard lock-in(인질형)은 사용자가 "갇혔다"고 느껴 분노 → 평판 깎임. 우리는 "자유롭다"고 느끼게 하면서 실제로 못 떠나게 하는 구조.

**윤리적 정당성·법적 안전**:
- 데이터 자체 차단 X (열람은 무기한)
- 사진 셀프 백업 안전장치 보장
- 가입 약관에 명확 사전 고지 + 동의 필수 ("텍스트 일괄 export 영구 불가")
- 한국 개인정보보호법상 "내 데이터 영구 삭제 요청권"은 별도 보장 (Suplex 보유분 영구 파기, 사용자 자체 보관본은 사용자 책임)

## 9. 영구 제외된 기능 (정식 출시에서도 안 만듦)

회계 영역 진입 절대 금지 정책에 따라 다음은 **정식 출시에서도 구현하지 않음**:
- ❌ **AI 회계 도구** (지출 분석·PnL 자동·계정과목 자동 매칭) — 복식부기·부가세·원천세·4대보험 영역 진입 X 정책 강화
- ❌ 다중 지점·법인 통합·BI 대시보드·결재 라인·API 액세스 — 인테리어 1~10인 SaaS 영역 밖, 별도 커스텀 솔루션 영역
- ❌ **JSON 텍스트 데이터 export** (v6에서 영구 제외) — 운영 락인 정책. 텍스트 일괄 export·portability는 모든 등급·구독 상태와 무관하게 차단. 사진은 ZIP 부분 보관으로만 외부 이전 가능. 자세히 → 섹션 7-bis

### 10.1 베타 가치 가시화 (도입 권장)

베타는 무료지만 **사용자가 어느 정도 가치를 받는 중인지 명시**해야 함 — 정식 출시 시 결제 결정의 정보 비대칭 방지.

**반영안 (in-product UI)**:
- 설정 페이지에 "사용량 패널" 추가:
  - 현재 활성 프로젝트 수 (등급 캡 대비)
  - 현재 저장 사용량 (GB)
  - v4 가격표 기준 환산 안내: "지금 사용 패턴 = 프로 등급 / 정식 출시 시 ₩99,000"
- 오버리지 단가는 미확정 → 환산에 표시 X (추측 가격 노출 금지)
- 표시 톤: 압박이 아닌 **"체감 가치 안내"** — 베타 종료 시 사용자가 결제 결정을 정보 기반으로 할 수 있게

### 10.2 장기 누적 데이터 관리 (부수 검토)

> **v6 업데이트**: 핵심 정책(JSON export 영구 차단, 사진 6개월 만료, 운영 락인)은 섹션 7·7-bis·9로 이동·확정. 사진 부분 보관 기능도 2026-04-30 구현 완료. 본 섹션은 12개월 미접근 자동 알림·콜드 스토리지 등 부수 검토안만 남김.

**12개월 미접근 자동 알림 (UX 제안)**:
- 12개월 이상 미접근 프로젝트 자동 알림: "사진 N장 / X GB — 외부 백업 후 삭제 권장"
- 헤비 유저(100GB+) 회사가 자발적으로 사진 부분 보관에 진입하도록 유도하는 핵심 동선
- 수동으로 한 건씩 처리하게 두면 사용자가 절대 안 함

**콜드 스토리지 (장기 검토, 자동 절감)**:
사용자가 직접 안 옮겨도 자동 절감되는 옵션 — 1년+ 미접근 프로젝트의 사진을 콜드 티어로 자동 이전 (조회 시 수 초 지연, 단가 1/5 수준). 정식 출시 후 실사용 분포 보고 결정.

### 10.3 사용량 기반 과금 (의견, 정식 출시 후 결정)

**배경**: 저장 비용은 사용량에 선형 증가하나 매출은 등급별 평탄(flat) → 헤비 유저가 라이트 유저에 보조받는 구조. 인테리어 업계는 업체당 프로젝트·사진·영수증 누적 편차가 매우 큼. "일 많이 하는 업체가 더 낸다"가 SaaS 공정성·지속성 측면에서 자연스러움.

**2가지 레버**:

| 레버 | 제안 | 우선순위 |
|---|---|---|
| **저장 오버리지** | 기본 용량 초과 시 add-on (예: 프로 50GB 초과분 ₩X / 10GB·월, **단가 미정**) | 1순위 |
| **프로젝트 수 캡** | 프로 = 무제한 → 30건으로 조정 (엔터프라이즈만 무제한) | 보류 |

**저장 오버리지를 1순위로 선택한 이유**:
- "기본료 ₩99,000 그대로" — 영업 라인 보존
- 한국 SaaS에 익숙한 패턴 (NHN·네이버클라우드 등)
- "쓴 만큼" 납득 쉬움 — 실제 비용과 직결되어 분쟁 적음
- 헤비 유저는 자연히 사진을 많이 쌓으므로 "일 많이 하는 업체가 더 낸다"가 자동 달성

**프로젝트 수 캡 보류 이유**:
- "무제한"이 프로 등급의 강한 셀링 포인트 — 잃으면 엔터프라이즈가 그 차액 가치를 정말 입증해야 함
- 저장 오버리지만 도입해도 헤비 유저는 자동으로 더 부담 (동일 효과)
- 단순함 유지 — 한 번에 한 가지 변수만 조정

**단가 결정 시점**:
- 베타 기간에는 도입 X (가격 변동 없음 — 약속 신뢰 유지)
- 정식 출시 후 6~12개월 실사용 분포 측정 → v6에서 단가 확정
- 지금 추측으로 정하면 너무 비싸거나 너무 싸게 잡힐 위험 → 데이터 기반 결정 원칙

**베타 사용자 보호**:
오버리지 도입 시 베타 회원도 **30일 사전 통보 + 평생 20% 할인이 오버리지 단가에도 동일 적용**.
