# 수플렉스 라운지 — 운영 가이드 & 부작용 시뮬레이션

작성일: 2026-05-01 / 대상: 슈퍼어드민(봉기님) + 미래의 운영 담당자
포지셔닝: `docs/커뮤니티_기획서.md`(SSOT)의 **자매 문서 — 운영·리스크 매뉴얼**

---

## 0. 이 문서를 읽어야 하는 이유

라운지는 코드 측면에선 v1 출시가 끝났지만, **커뮤니티는 코드가 아니라 운영이 만든다.** 일주일 운영 미숙으로 평판 회복에 6개월 걸리는 게 클로즈드 커뮤니티의 현실입니다. 이 문서는 다음 3가지를 정리합니다.

1. **사용 방법** — 사용자·슈퍼어드민·퇴사자 각 역할의 일상 동선
2. **부작용 시뮬레이션** — 15개 시나리오 × (가능성·심각도·조기 신호·대응책)
3. **사례 연구** — 다른 회사·업계가 이미 겪은 실패와 성공에서 배우기

⚠️ 이 문서에는 **봉기님이 v1 단계에서 내린 "단가·시세 게시 OK" 정책의 법적 재검토 권장**이 9장에 담겨 있습니다. 운영 시작 전 반드시 한 번 보세요.

## 1. 라운지 사용 방법 (역할별)

### 1-1. 일반 사용자 (디자이너 · 현장팀 · 운영진)

**진입**: 상위 글로벌 메뉴 8번째 "라운지" 클릭. 회사가 APPROVED이면 자동으로 라운지 멤버.

**첫 진입 시 권장 동선**:
1. 우측 상단 "내 프로필" 버튼 → 직군(디자이너/현장팀/운영) 선택 + 표시 라벨(예: "디자이너 / 5년차") 작성
2. 6개 카테고리 둘러보기 (전체 탭부터)
3. 관심 직군 태그(`[디자이너]` 등)로 필터링
4. 첫 글 작성: 가벼운 자유잡담부터 권장

**글쓰기 규칙** (10초 안에 외울 것):
- ❌ 거래처·시공팀·전 직장 **실명 비방** 금지 (분쟁 표현 영구 규칙 연장)
- ❌ 클라이언트 **주소·동호수·집주인 식별** 가능 정보 게시 금지
- ❌ "AI에게 분석해달라" 류 자동 자문 요청 글 금지 (AI 어드바이스 보류 정책)
- ⚠️ **단가·시세 정보** — 기획서 v1 단계 "허용". 그러나 9장 법적 검토 결과에 따라 변경 가능
- ✅ 익명 닉네임 우선, 회사명 표시는 옵션
- ✅ 스케치업 루비 카테고리는 코드 블록(```ruby ... ```) + .rb 파일 첨부 OK (1MB×3개)

**신고 동선**: 글/댓글 우하단 🚩 → 사유 5개 중 선택 → 즉시 큐로 들어감. 슈퍼어드민이 24시간 내 처리.

### 1-2. 슈퍼어드민 (봉기님)

**일일 점검 (3분)**: 어드민 콘솔 → "💬 라운지" 탭 → 처리 대기 신고 0건 유지

**주간 점검 (10분)**:
1. 신고 큐 클리어 (숨김/기각 + 메모)
2. "수플렉스 사용 팁" 카테고리에서 좋은 글 1개 → "📌 홈 큐레이션" 클릭
3. 활성도 낮으면 시드 글 1~2개 직접 작성 (5장 시드 게시 가이드 참고)

**월간 점검 (30분)**:
1. 카테고리별 글 수 추이 (6장 측정 지표)
2. "💬 라운지 → 멤버십 보강" 1회 클릭 (멱등, 안전)
3. 신고 패턴 분석 — 특정 사용자 반복 위반 시 멤버십 정지 검토
4. 로드맵 우선순위 재검토 (10장 v2 백로그)

**모더 의사결정 매트릭스**:

| 위반 수위 | 1차 액션 | 2차 (재발) | 3차 (반복) |
| --- | --- | --- | --- |
| 가벼운 표현 (오해의 소지) | 댓글 코칭 (DM 대신 답글) | 글 숨김 + 안내 | 멤버십 정지 |
| 명백한 거래처 비방 | 즉시 글 숨김 | 멤버십 7일 정지 | 영구 정지 |
| 클라이언트 정보 노출 | 즉시 글 숨김 + 작성자 통보 | 멤버십 정지 | 영구 정지 |
| 악성 .rb 의심 | 첨부 즉시 비공개 | 작성자 활동 동결 | 영구 정지 + 신고서 보관 |
| 사기 채용 | 즉시 글 삭제 | 영구 정지 | 영구 정지 + 가능 시 형사 신고 자료 |

### 1-3. 퇴사자 (라운지 멤버십 유지자) ★ 비전 핵심

**원칙**: 회사를 떠나도 LoungeMembership.status='active'면 평생 접근.

**시나리오 A — 다른 회사로 이직**: 새 회사가 APPROVED면 그쪽 데이터 + 라운지 자연스럽게 이어짐. 본인은 변화 거의 못 느낌.

**시나리오 B — 무소속 기간**: 회사 데이터 접근은 차단. 라운지는 유지. 단, v1엔 메뉴 진입 경로 일부 제약 — **정식 출시 전 v1 + 핫픽스로 "회사 없는 멤버십 보유자" 전용 진입 페이지 추가 검토**(11장 백로그).

**시나리오 C — 새 회사 입사 후 수플렉스 추천**: 라운지에서 본 좋은 글들이 추천 동기 → 신규 회사 가입. **이 사이클이 바로 영업 무료화의 정수.**

## 2. 라운지의 5가지 목적 + 측정 지표

| # | 목적 | 측정 지표 | v1 목표 |
| --- | --- | --- | --- |
| 1 | 체류시간·리텐션 | DAU 대비 라운지 방문율 | ≥ 20% |
| 2 | 자조 시스템 | 사용자 작성 Q&A에 사용자 답변 비율 | ≥ 50% |
| 3 | 데이터 자산화 | 누적 글 수 (3개월) | ≥ 200 |
| 4 | 인테리어 생태계 장악 | 회사당 평균 활동 사용자 + 퇴사 후 활동 유지 비율 | 회사당 ≥ 1, 퇴사자 30일 활동 ≥ 30% |
| 5 | 등급 진입 명분 | `[프리셋추천]` 태그 글에 노출된 BASIC 회사의 PRO 전환율 | 측정 인프라 v2에서 |

이 지표는 **3개월 후 첫 리뷰**, **6개월 후 v2/v3 GTM 결정**.

## 3. 부작용 시뮬레이션 — 15가지 시나리오

각 시나리오 형식: **[가능성·심각도] / 조기 신호 / 1차 대응 / 사후 정리**

### 3-A. 콘텐츠 위험 (5개)

#### S1. 거래처 실명 비방 누적
- **[높음 · 매우 심각]** 분쟁 표현 영구 규칙 직접 위반 + 명예훼손 법적 리스크
- **신호**: "○○자재상 사기쳤어요" 류 글 등장
- **대응**: 즉시 글 숨김 → 작성자에 정책 안내 → 재발 시 7일 정지
- **사후**: 정책 가이드라인을 글쓰기 모달 placeholder에 명시화 + 부정 키워드 작성 단계 경고 (v2)

#### S2. 클라이언트 식별 정보 노출 (현장 사진)
- **[중간 · 매우 심각]** 클라이언트 비접근 영구 규칙 위반, 개인정보보호법 저촉 가능
- **신호**: 현장 사진에 우편함·문패·창밖 풍경(주소 추정 가능)
- **대응**: 즉시 첨부 비공개 → 작성자에게 마스킹 후 재업로드 안내
- **사후**: 사진 업로드 시 안내 모달 추가(v2). 약관에 책임 사용자 귀속 명시

#### S3. 단가·시세 게시 누적 → 카르텔 의심 ★
- **[중간~높음 · 매우 심각]** 한국 공정거래법 제40조 위반 가능. 9장 별도 분석 참조.
- **신호**: "X아파트 30평 전기공사 평당 얼마 받으세요?" 류 게시물 + 회사들 응답
- **대응**: **9장 정책 재검토 결정에 따라**. 만약 정책을 "허용→가이드라인" 변경한다면, 단가 직접 명시 글에 안내 코멘트 + 작성자 코칭
- **사후**: 9장 결정에 따라 카테고리별 단가 정책 명시 (예: 일반 지역·범위 OK, 직접 단가 X)

#### S4. 악성 루비 스크립트 첨부
- **[낮음 · 심각]** 스케치업 외부 시스템 접근, 파일 삭제, 키로깅 등
- **신호**: 다운로드 후 사용자 신고 ("실행하니 이상한 동작") + 파일명/내용 의심 패턴
- **대응**: 첨부 즉시 비공개 → 작성자 활동 동결(글쓰기 차단) → 정확한 분석 후 영구 정지 + 다른 다운로더에게 공지
- **사후**: 악성 코드 신고 사유 명시(이미 v1 포함). v3+에서 자동 정적 분석(키워드 매칭) 검토

#### S5. AI 자문 요청 글 우회
- **[낮음 · 중간]** 정책 충돌
- **신호**: "AI 어드바이스 보고 어떻게 생각하세요?" 류 + 화면 캡처
- **대응**: 글 숨김 + 작성자 안내 — 이 정책은 보류 결정이며 AI 시점은 정식 출시 후
- **사후**: 작성 단계 키워드 감지(v2) — "AI" "분석해줘" 등

### 3-B. 활성화 위험 (3개)

#### S6. 텅 빈 광장 (Bettermode·실패 사례 다수)
- **[매우 높음 · 중간]** 베타 회사 적어 글 X → 더 안 옴 → 죽음
- **신호**: 출시 D+7 전체 카테고리 합쳐 글 ≤ 5개, 댓글 0개
- **대응**: 슈퍼어드민이 매주 1~2개 시드 (5장 시드 가이드). 사전 섭외 회사 5~10개 OWNER에 직접 부탁("첫 글 1개만 부탁드려요"). 자유잡담 카테고리로 진입장벽 ↓
- **사후**: 30-50 cornerstone posts 패턴 적용 (첫 30일 슈퍼어드민이 매일 1개 시드 목표)

#### S7. 소수 발언 독점 (1~2명이 80% 글)
- **[중간 · 중간]** 다른 멤버가 위축 → 침묵 동조 → 결국 그 1~2명만 남음
- **신호**: 주간 글 작성자 분포 — 상위 1명이 50%+ 차지
- **대응**: 슈퍼어드민이 다른 회사 OWNER에 직접 답글 부탁. 인기 글 큐레이션을 다양한 작성자로 분산
- **사후**: 작성자 1인당 일일 글 수 가이드(권장, 강제 X). 댓글이 더 많은 글 인기순 노출(v2)

#### S8. 잡담 비율 폭주 → 노하우가 묻힘
- **[높음 · 중간]** 자유잡담만 활성, 본업(사용 팁·노하우) 사라짐
- **신호**: 자유잡담 카테고리가 전체 글의 60%+ 차지
- **대응**: 카테고리별 분리 노출 강화. 사용 팁 카테고리에 슈퍼어드민이 좋은 시드 적극 투입
- **사후**: 자유잡담 카테고리 한도 검토. 또는 자유잡담을 별도 화면으로 분리 (v3)

### 3-C. 사회·법적 위험 (3개)

#### S9. 구인구직 사기
- **[중간 · 심각]** 가짜 채용 공고 → 실제 구직자 피해 → 수플렉스 책임 추궁
- **신호**: 너무 좋은 조건 ("월 800 디자이너 모집"), 외부 연락처(카톡 ID) 의심
- **대응**: 즉시 글 삭제 + 작성자 영구 정지. 이미 연락한 사용자 있으면 알림 시도
- **사후**: 구인구직 카테고리 약관 강화 — "수플렉스는 채용 중개 X, 외부 연락 시 본인 책임" 고정 안내. v3에서 회사 인증 배지

#### S10. 잘못된 세무·법률 정보 확산
- **[높음 · 중간]** "사장님 이거 비용처리 가능해요"가 사실 위반 — 사용자 따라하다 추징
- **신호**: `[세무]` 태그에 단정적 진술이 다수 좋아요
- **대응**: 댓글로 "이건 세무사 확인 필요"라는 운영 코멘트. 글 자체는 유지(자유 발언)
- **사후**: `[세무]` 카테고리 진입 시 면책 안내 모달 (v2)

#### S11. 노조성·집단 행동 형성 (예: 인건비 인상 단합)
- **[낮음 · 매우 심각]** 한국 공정위는 노동·서비스 가격 담합도 처벌. 라운지가 "단합 채널"로 의심받으면 수플렉스 자체 리스크
- **신호**: 다수 회사가 "○○공정 평당 인건비 X원 미만 거절합시다" 식 게시 + 동조 댓글
- **대응**: 글 숨김 + 작성자 통보 — 이런 형태의 가격 협의는 라운지 정책 위반 (담합 의심). 9장의 단가 정책 재검토 결과에 따라 가이드라인 강화
- **사후**: 9장의 단가 정책이 "허용 → 제한"으로 바뀌면 자연스럽게 차단됨

### 3-D. 운영 위험 (2개)

#### S12. 신고 폭주 → 슈퍼어드민 번아웃
- **[중간 · 중간]** 모더가 따라가지 못함 → 24h SLA 깨짐 → 신뢰 하락
- **신호**: 신고 큐 ≥ 20개 + 24시간 미처리
- **대응**: 즉시 모든 큐 처리 (1차 액션이 단순 — 숨김 또는 기각). 임시 카테고리 잠금
- **사후**: 정식 출시 전 모더 알림 자동 메일(v2). 베타 회사 수 제한으로 자연 통제. 정식 출시 시 보조 모더 1명 위임 검토

#### S13. 스팸봇·외부 침투
- **[낮음 (클로즈드라) · 중간]** 라운지 멤버십 가진 user 계정이 탈취되어 스팸 광고 게시
- **신호**: 동일 사용자가 짧은 시간에 10개+ 글, 외부 링크 다수
- **대응**: 즉시 user `tokenVersion++` (강제 로그아웃) + 멤버십 정지 + 글 일괄 삭제
- **사후**: 짧은 시간 다중 게시 자동 감지(v2). 라운지에 외부 도메인 링크 화이트리스트(v3)

### 3-E. 비즈니스 위험 (2개)

#### S14. 경쟁사 첩보 활동
- **[중간 · 낮음]** 경쟁 SaaS 직원이 공정위 위장 가입해서 라운지 활동 분석
- **신호**: 식별 어려움 (정상 활동과 구분 안 됨)
- **대응**: 가입 단계의 회사 검증을 강화 (현재 베타 진입 통제로 봉기님 직접 승인 → OK). 첩보 자체는 막을 수 없으나, 라운지가 "공개" 자료가 아니라는 점은 이미 클로즈드라 자연 차단
- **사후**: 정식 출시 후 회사 사업자번호·세금계산서 검증 추가(v3)

#### S15. 퇴사자가 전 직장 비방·정보 유출
- **[중간 · 심각]** 퇴사자 라운지 멤버십이 비전이지만 양면 칼날
- **신호**: 회사명·전 직장 OWNER 실명 비방, 내부 견적·고객 정보 폭로
- **대응**: 거래처 실명 비방 정책 그대로 적용 — 즉시 숨김 + 멤버십 정지. 단, "회사 떠난 후 본인 경험 공유"는 가능 (실명·구체 정보 X 한정)
- **사후**: "전·현 직장 관련 글 가이드라인" 부록 추가(v2)

| **B. 가이드라인 제한 ★ 권장** | 일반론 OK (지역·범위·트렌드), 직접 단가 명시 X | 클로즈드 가치 80% 보존 + 법적 리스크 대폭 감소 | "이 자재 평당 X원" 같은 구체 글은 제한 |
| C. 전면 금지 | 모든 단가 정보 게시 차단 | 법적 리스크 0 | 클로즈드 가치 약화, 라운지 매력 ↓ |

### 9-4. 옵션 B 가이드라인 (권장 시 적용)

라운지에서 OK:
- "타일 평당 단가가 작년보다 많이 올랐어요" (트렌드)
- "수도권은 인건비 상승 체감되네요" (지역적 일반)
- "○○ 자재가 합리적이에요" (브랜드 추천, 단가 X)

라운지에서 X:
- "주방 타일 평당 X원에 받았어요" (직접 단가)
- "디자이너 인건비 일당 Y원이 적정" (직접 인건비)
- "특정 회사 견적 가이드 단가 그대로 공유" (단가 정책의 핵심)

### 9-5. 봉기님 결정 요청

이 9장 보고 다음 중 하나로 답해주세요:

1. **"옵션 A 유지"** — 리스크 인지하고 가는 결정. 변호사 자문 1회 권장.
2. **"옵션 B 적용"** — 가이드라인 추가. 작성 모달에 안내 1줄 + 신고 사유에 "단가 직접 명시" 추가 (v1 핫픽스 3시간 작업).
3. **"옵션 C 적용"** — 단가 정책 전면 금지로 정책 변경.

결정 시 이 문서 9-3 표 옆에 ✅ 마킹 + 9-1 현행 정책 갱신.
