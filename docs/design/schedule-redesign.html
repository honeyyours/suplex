일정 화면 리디자인

Suplex의 핵심 wedge 기능. 인테리어 회사의 현장·견적·팀 일정이 모두 같은 캘린더 패턴으로 표시됩니다. 현재 코드(client/src/components/AggregateCalendar.jsx· ScheduleCalendar.jsx· TeamCalendar.jsx)에서 클래스·구조를 그대로 추출했습니다.

개선 의뢰: 캘린더 그리드 가독성 개선 의뢰: 일정 항목 시각 위계 개선 의뢰: 헤더·툴바 정돈 개선 의뢰: 프로젝트 색상 대비 유지: 토큰 시스템(navy·공정 10색) 유지: 격리 정책(현장 정보·금액 X)

① 전체 일정 — 회사의 모든 현장을 한 캘린더에 합산

메뉴: /schedule?tab=all. 현장·견적·전체 3탭 중 가장 자주 보는 화면. 일정 1줄 = 행 단위 (같은 주 같은 프로젝트는 같은 행 정렬).

구리다 평가 ① — 셀이 너무 빽빽하고 정보가 균질해서 “오늘 뭐 중요한가”가 안 보임. 위계 부족.
① 탭 + 헤더 — 현장·견적·전체 3탭. 위에 페이지 제목 별도. 페이지 제목과 탭이 두 줄로 분리되어 공간 낭비. 통합 검토.
② 월 네비게이션 — 좌측 비어있음(flex-1 spacer) · 가운데 ‹ 라벨 › · 우측 액션. 좌측 공간이 비어있어 어색.
③ 캘린더 그리드 — 7×6, 셀 min-h 100px. 일정 1줄당 약 18px. border-r/border-b로 그리드. 다른 달 셀은 bg-gray-50/50.
④ 일정 막대 — border-l-[3px]로 프로젝트 색상 + 공정 chip + 잔여 텍스트. 우측에 ✓ 원형 뱃지(확정). 미확정은 opacity-55% 전체.
구리다 평가 ② — chip + 텍스트가 한 줄에 너무 많은 정보. 색상도 비슷한 톤끼리 옆에 있으면 구분 약함.
⑤ 현장 인덱스 — 캘린더 하단. 색상 + 프로젝트명. 인덱스인지 필터인지 모호.
구리다 평가 ③ — 인덱스가 캘린더 밖에 떨어져 있어 색-프로젝트 매핑을 사용자가 두 번 시선 옮겨 확인해야.

② 프로젝트 일정 — 시작일~종료일만 잘라 보여주는 변형

메뉴: /projects/:id/schedule. 그리드는 동일하나 프로젝트 기간 밖 셀은 bg-gray-100/70로 비활성. 셀 클릭=인라인 입력.

인터랙션 & 행동

· 셀 클릭(데스크탑) — 인라인 입력 활성. Enter=저장 후 같은 셀 머묾, Tab=다음 날, Shift+Tab=이전 날, ↓=다음 주 같은 요일, ↑=이전 주.
· 셀 탭(모바일) — MobileScheduleSheet 바텀시트가 slide-up 180ms로 올라옴.
· 항목 클릭 — 같은 항목 인라인 편집(데스크탑).
· 우클릭 — 일정 컨텍스트 메뉴(삭제·확정 토글·복사·협력업체).
· 확정 토글 — 모바일 시트나 우클릭 메뉴. 확정 시 막대 opacity 100%, 우측 ✓ 원형 뱃지.
· 같은 주 같은 프로젝트 — utils/calendarLane.js의 lane 알고리즘으로 같은 행에 정렬(시각적 연속성).
· PDF 인쇄 — 우상단 📄 버튼 → 새 창에 인쇄용 HTML(별도 디자인 작업 대상).
· 모바일 빽빽 모드 — 4번째 이상 항목은 [&>div:nth-child(n+4)]:hidden으로 숨김 + "+N" 표시.

디자이너 의뢰 사항

1) 위계 부족 해소 — 한 화면에 모든 일정이 균질하게 보입니다. “오늘·내일·이번 주·확정 미정 회의”의 가중치 차이를 시각적으로 표현해주세요. 단순 색상 외 폰트 굵기·여백·그림자 활용 가능.
2) 헤더·툴바 정돈 — 전체일정/팀캘린더에서 페이지 제목 + 탭 + 월 네비게이션 + 액션 버튼이 분산되어 있습니다. 통합/그룹핑 해주세요.
3) 일정 막대 가독성 — chip + 잔여 텍스트 + 확정 ✓가 한 줄에 다 들어가 좁아 보입니다. 우선순위 정리·여백·타이포 weight 조정.
4) 인덱스/필터 일체화 — 현장 인덱스(아래 색상-이름)와 멤버 필터(위 칩)가 분리되어 있는데, “보고 있는 색이 누구의 어떤 현장인가”를 한 곳에서 알 수 있게 해주세요.
5) 모바일 가독성 — 8px 폰트는 임계 아래. 셀 안 표시 정책 자체를 재설계(예: 도트만 + 시트 진입 강조).
6) 토큰 시스템 유지 — Tailwind navy 팔레트, 공정 10색, 프로젝트 hash 색은 회사 자산이라 유지해주세요. 사용 방식만 다듬어주세요.
7) 다크 모드 — 현재 부분만 적용. 캘린더 전체 다크 토큰까지 함께 잡아주시면 좋습니다 (slate 9xx 베이스).
8) 격리 정책 — 캘린더에는 현장 주소·전화·금액 일체 노출 X. PDF 인쇄도 같은 정책. 표시 정보는 “공정·내용·확정 여부·담당자(팀캘린더)”만.
