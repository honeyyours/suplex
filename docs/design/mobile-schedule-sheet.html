모바일 일정 바텀시트

전체일정·프로젝트일정·홈 주간 캘린더에서 모바일 사용자가 날짜 셀을 탭했을 때 올라오는 바텀시트입니다. 현재 구현 화면을 그대로 추출했습니다(코드: client/src/components/MobileScheduleSheet.jsx).

① 헤더 — 좌우 ‹ ›로 날짜 이동(어제/내일). 가운데는 M/D (요일) 한국식. 우측 ✕로 닫기.
② 일정 리스트 — 행 단위. 좌측 체크박스로 확정/미확정 토글. 확정 시 행 배경 emerald-50, 체크 emerald-500. 공정이 매칭되면 좌측 컬러 chip + 잔여 텍스트. 협력업체가 태그되어 있으면 행 아래에 🏢 보라색 라벨.
③ 빠른 공종 칩 — 5열 그리드. 회사가 사용하는 공정만 노출(현재 데모는 10개). 탭 한 번으로 해당 공정 일정이 즉시 추가됩니다. 색상은 공정별 고정 매핑.
④ 직접 입력 — 공정에 매칭되지 않는 메모(예: "사장님 미팅"). 엔터 또는 + 버튼으로 추가.

인터랙션 & 행동

· 진입 — 모바일에서 캘린더의 날짜 셀을 탭. 바텀시트가 slide-up 180ms ease-out로 올라옵니다.
· 닫기 — 시트 밖 어두운 영역 탭, 우측 ✕, 또는 ESC.
· 날짜 이동 — 헤더 좌우 ‹ ›. 첫날/마지막날에서는 비활성(텍스트 gray-300).
· 확정 토글 — 행 좌측 체크박스 탭. 행 배경이 gray-50 → emerald-50으로 전환.
· 빠른 추가 — 빠른 공종 칩 탭 시 같은 이름의 일정이 즉시 추가됨(D-N 룰·체크리스트 자동 시드 연동).
· 직접 입력 — 자유 텍스트. 본문에 공정 키워드가 포함되어 있으면 자동으로 chip이 좌측에 붙음("도배 벽지 도착" → chip "도배" + 잔여 "벽지 도착").
· 협력업체 — onUpdate가 연결되어 있을 때만 🏢 버튼 노출(현재 베타에서는 등록 UI hide, 표시만 사용).
· 시트 높이 — max-h-[85vh]. 헤더·빠른 칩·직접 입력은 고정 영역, 일정 리스트만 스크롤.
