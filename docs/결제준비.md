베타 단계엔 결제 X (시범 고객 무료). 정식 출시 시점에 결제 활성. 사업자등록 + 통장 개설이 가장 큰 선행 작업이며, 이게 끝나야 PG 가맹점 신청 가능. 정식 출시 약 2~3개월 전 준비 시작 권장.

## 1. 행정·법적 준비 (선행 필수)

| 항목 | 비고 |

| 결제 정책 문서 | 환불·해지·구독 갱신·세금계산서 발행 정책 (가격정책.md 6번 참조) |

## 2. PG사 선택 — 추천: 토스페이먼츠

| PG | 평가 |
|---|---|
| **토스페이먼츠** ⭐ | 빌링키(정기결제) API 깔끔, 세금계산서 자동 발행 통합, 개발자 문서·SDK 최고. 수수료 카드 약 2.9% |
| 포트원(PortOne) | 여러 PG 통합 — 추후 갈아끼우기 쉬움. 다만 중계자라 한 단계 추가, 트러블슈팅 복잡 |
| KG이니시스·KCP·나이스페이 | 전통 PG. 정기결제·구독 UX 토스 대비 약함, 개발 부담 큼 |
| 카카오페이·네이버페이 | B2C 간편결제 강점. **B2B SaaS엔 부적합** (사업자 결제 흐름 약함) |

### 토스페이먼츠 핵심 이유
- **빌링키(BillingKey)** — 카드 1번 등록 → 매월 자동 결제. SaaS 구독 패턴 정확히 매칭
- 사업자등록증·통장 사본만 있으면 가맹점 신청 가능
- 세금계산서 자동 발행 (B2B SaaS 필수)
- 가입 흐름: tosspayments.com 신청 → 서류 업로드 → 심사 1~2주 → 카드사 개별 심사 → sandbox → 운영

## 4. 코드·DB 개발 영역

베타 코드에 이미 진화 경로 깔림 (`Company.plan?` 컬럼 자리, `PLAN_FEATURES` 매트릭스, `hasFeature` 게이트). 정식 출시 1~2개월 전 추가:

```prisma
model Company {
  plan        Plan?     // STARTER/PRO/ENTERPRISE (null = 베타 무료)
  trialEndsAt DateTime?
}

model Subscription {
  id            String   @id @default(cuid())
  companyId     String   @unique
  status        SubscriptionStatus // TRIALING/ACTIVE/PAST_DUE/CANCELED
  plan          Plan
  billingKey    String?  // 토스 빌링키
  startedAt     DateTime
  nextBillingAt DateTime?
  canceledAt    DateTime?
}

model Invoice {
  id              String  @id @default(cuid())
  companyId       String
  subscriptionId  String
  amount          Decimal
  vatAmount       Decimal
  status          InvoiceStatus // PENDING/PAID/FAILED/REFUNDED
  paidAt          DateTime?
  taxInvoiceUrl   String?
}

enum Plan { STARTER PRO ENTERPRISE }
```

### UI 신규
- `/settings/billing` — 결제 수단 등록·구독 시작·해지·이력
- 어드민 콘솔 — 회사별 결제 상태 조회·수동 조정·환불

## 5. 운영 체크 (정식 출시 후)

- **결제 실패**: 3일 재시도 → 7일 grace → 자동 정지 + 메일 알림
- **카드 만료** 30일 전 사전 알림 메일
- **세금계산서** 자동 발행 (토스 통합)
- **환불** 어드민 콘솔에서 수동 처리 + Audit log 자동 기록
- **결제 실패율·MRR·해지율** 통계 대시보드 (정식 출시 후 v1.x)
- **용량 초과 알림** — 80% 도달 시 메일, 100% 도달 시 신규 업로드 차단

## 7. 운영 규칙 (영구)

- 베타엔 결제 코드 절대 추가 X — `Company.plan` null 상태 유지로 ENTERPRISE 가정 (`hasFeature` 베타 동작과 동일)
- 사업자등록·통장·통신판매업 진행 상황 변경 시 본 문서 1번 표 갱신
- PG 선택 변경 결정 시 본 문서 2번 결정 사유 기록
- **자격증명 노출 금지**: 토스 가맹점 비밀키·`TOSS_SECRET_KEY` 등 채팅 평문 노출 X. Railway 환경변수로만
- **분쟁 표현 금지**: 약관·환불 정책에서 분쟁·소송·증빙 부정 프레이밍 X
- **회계 영역 분리**: 지출관리(Expense)에 결제 매출 데이터 자동 연동 X — 별도 시스템(Subscription·Invoice)으로 운영
