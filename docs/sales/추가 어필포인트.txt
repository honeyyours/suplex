★ 차별화 코어 컨셉 — 견적 피드백 사이클 (2026-04-30 봉기님 명명)

  견적 작성 → 발주 → 실제 지출 → 공정별 분석 → 정산 메모 → 다음 견적 자동 prefill

  ↑ 이 사이클이 단순 회계 도구가 아닌 "견적이 점점 똑똑해지는 학습 도구"로
  Suplex를 차별화시키는 핵심.

설명:
  - 견적·발주·지출은 따로 돌아가는 모듈이 아니라 한 사이클의 단계
  - 끝나면 자동으로 다음 견적의 시작점으로 흘러감
  - 한 회사가 Suplex를 오래 쓸수록 견적 정확도·공종별 단가·이익률이
    자연스럽게 누적·정교화

영업·소개·매뉴얼 표현 후보:
  - "견적이 점점 똑똑해진다"
  - "지출 데이터가 다시 견적으로 돌아온다"
  - "한 번 짜본 견적은 회사 자산"
  - "끝난 현장이 다음 견적의 출발점"

기능 매핑:
  - 견적 = 간편 견적·상세 견적·견적상담
  - 발주 = 발주 탭 (마감재 확정 → 자동 생성)
  - 지출 = CSV 통장 가져오기·수기 입력 (회계 X, 프로젝트별 비용·손익만)
  - 공정별 분석 = 견적 vs 실지출 자동 비교 화면 (공정별)
  - 정산 메모 = 정산 탭. 공정별 한 줄 메모
  - 다음 견적 prefill = 새 견적 모달이 직전 정산 메모 참고

영업 메시지에 강조 (지출관리·AI비서 컷 자리에 이 사이클 배치):
  - 메인 테마 ⭐⭐⭐ "개인의 능력 → 회사의 능력"과 같은 비중
  - 우리가 회계 도구가 아닌 이유 = 이 사이클이 있기 때문

회계 영역 진입 절대 금지 (관련 정책):
  - 계정과목·복식부기·재무제표·부가세신고 등 일체 X
  - 우리는 "공종 단위 비용·손익·정산"만 다룬다
  - 영업 자료에서 "회계" 표현 금지
